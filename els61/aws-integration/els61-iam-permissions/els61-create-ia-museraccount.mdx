---
title: "Creating a dedicated IAM user account for AnyNet IRIS"
description: "Do not use the AWS account root to set up the required IAM permissions. For more information, see AWS account root user."
---

<p>Do not use the AWS account root to set up the required IAM permissions. For more information, see <a href="https://docs.aws.amazon.com/IAM/latest/UserGuide/id_root-user.html" target="_blank">AWS account root user</a>.</p>

<p>For detailed information about why you need a dedicated IAM user account, see <a href="/iris/iam-permissions/iam-permissions-introduction" target="_blank">About the required AWS IAM user for AnyNet IRIS</a>.</p>

<p>To create an IAM user account:</p>

<ol> <li>Ensure you have signed in to AWS Management Console. </li> <li> Navigate to <span>IAM Services</span> using the following URL:  </li> <li> In the left-hand navigation menu, select <span>Users</span>.  <br /><img src="/images/resources/images/iris/iamdashboardmenu.png" alt="Iamdashboardmenu" />  <br />Any previously created IAM users are listed in the right-hand pane. </li> <li>Select <span>Add User</span>.</li> <li> Type the user name for the new user.  <br />This is the sign-in name for AWS, for example: <span>anynetuser</span>. </li> <li> In the Select AWS access type section, select the <span>Access key – Programmatic access</span> check box only. <br /><img src="/images/resources/images/iris/iamuser.png" alt="Iamuser" />  </li> <li> Select <span>Next: Permissions</span>. <br />We will add permissions later. </li> <li> Select <span>Next: Tags</span>. <br />Add any required tags according to your operational policies. </li> <li>Select <span>Next: Review</span> to review your choices.</li> <li> Select <span>Create user</span> to create the IAM user. <br /><img src="/images/resources/images/iris/iamusercreate.png" alt="Iamusercreate" />  <br />The Access key ID and Secret access key are displayed. AnyNet IRIS will use these security credentials to perform necessary requests against your AWS account. <br />You can view and download the Secret access key once only. If you forget the Secret access key, you will need to regenerate it on your AWS user account. For more information, see: <a href="https://docs.aws.amazon.com/general/latest/gr/aws-sec-cred-types.html#access-keys-and-secret-access-keys">AWS security credentials - programmatic access</a>. </li> <li>Click <span>Download.csv</span> to store the keys locally on your computer.</li> <li>Select <span>Close</span>.</li> </ol>
