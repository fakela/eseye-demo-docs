---

<div> <h3>Before you begin</h3> <ul> <li> Using a terminal emulator, send an <span>AT</span> command to ensure that the smart terminal can receive AT commands.  <span><strong>Note: </strong>For information about connecting a terminal emulator to the modem, see <a href="/els61/els61-rs232-serial-connection">Connecting to the LTE Cat.1-xx click</a>.</span> </li> <li> Send an <span>AT+ETMINFO=version</span> command to verify that the version number is <span>2.0.20</span> or higher. <br />If the version number is not in this range, you will need to update the modem software. For more information, see <a href="/els61/els61-upgrading-software">Updating software – Cinterion® ELS61 Wireless Module</a>. </li> <li> Ensure you know the thing name that you set up in AWS. </li> </ul> </div>

<p>To test that your thing can publish information to AWS:</p>

<ol> <li> Create two publish topics in the Cinterion® ELS61 Wireless Module. <ol> <li> Using a terminal emulator, type:  <br /><span>at+emqpubopen=0,"PublishToCloud0"</span>&lt;return&gt; <br /><span>at+emqpubopen=1,"PublishToCloud1"</span>&lt;return&gt; </li> <li> <div> Check that the first two index numbers are assigned a topic each. Type: <br /><span>at+emqpubopen?</span>&lt;return&gt; </div> <br />A list of index numbers and their assigned topics appears. <br /><img src="/images/resources/images/emqpubopen.png" alt="Emqpubopen" />  </li> </ol> </li> <li> Subscribe to the newly created publish topics in AWS. <ol> <li> Using<span>AWS IoT</span>, in the left hand menu, select <span>Test</span> to open the <span>MQTT test client</span>, then ensure you are on the <span>Subscribe to a topic</span> tab. <br /><img src="/images/resources/images/aws_subscribe1.png" alt="Aws Subscribe1" />  </li> <li> In the<span> Topic filter</span> box, type: <br /><span>+/&lt;<i>ThingName</i>&gt;</span>  <br />This subscribes AWS to all topics related to your thing. </li> <li> Select <span>Subscribe</span>. <br /><img src="/images/resources/images/aws_subscribe2.png" alt="Aws Subscribe2" />  <br />A subscription appears, listed in the Subscriptions panel. </li> </ol> </li> <li> Publish information to the topics you created in the Cinterion® ELS61 Wireless Module. <span><strong>Note: </strong>You can send a maximum payload of 1000 characters to AWS.</span> <ol> <li> Using the terminal emulator, type: <br /><span>at+emqpublish=0,1,"&#123;&#92;"Temperature&#92;": 24&#125;"</span>&lt;return&gt; <br /><span>at+emqpublish=1,1,"&#123;&#92;"BatteryPower&#92;": &#92;"Low&#92;"&#125;"</span>&lt;return&gt; <br />The emqpublish command uses the following syntax: <br /><span>at+emqpublish=<i>IndexNumber</i>,<i>QoS</i>,</span>  <br /><span>"&#123;<i>PublishDataInJSON</i>&#125;"</span>  <br /><img src="/images/resources/images/emqpublish.png" alt="Emqpublish" />  <br />These messages instantly appear in AWS. </li> </ol> </li> <li> View the published information in AWS. <ol> <li> In the <span>Subscriptions</span> panel, select <span>+/&lt;<i>thingName</i>&gt;</span> to view all published messages. <br /><img src="/images/resources/images/subscriptionmessages.png" alt="Subscriptionmessages" />  </li> </ol> </li> </ol>

<Tip>If you can see the messages in AWS, then your thing can successfully publish data into the cloud through the Cinterion® ELS61 Wireless Module.</Tip>

<p>Next, test that the cloud can publish messages to your thing. </p>
