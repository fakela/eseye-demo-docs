title: "CREG – request network registration status"
description: "Verifies the current network registration status of the Cinterion® ELS61 Wireless Module. First set the Mode to determine which information is displayed, and if it's displayed automatically when a change occurs."
---

<p>Verifies the current network registration status of the Cinterion® ELS61 Wireless Module. First set the <span>Mode</span> to determine which information is displayed, and if it's displayed automatically when a change occurs.</p>

<table> <col /> <col /> <col /> <thead> <tr> <th>Type</th> <th>Syntax</th> <th>Returned Result</th> </tr> </thead> <tbody> <tr> <td>Test</td> <td> AT+CREG=? </td> <td> +CREG: (0-2) <br />  <br />OK </td> </tr> <tr> <td>Read</td> <td> AT+CREG? <span><strong>Tip: </strong>Perform the Write command first so that you can set the returned details of the Read command.</span> </td> <td> The response depends on which mode you selected in the write command. <br />The initial number returned is the mode. <br />If the mode = 0, the following response occurs: <br />+CREG: 0,&lt;regStatus&gt; <br />  <br />OK <br />If the mode = 1, the following response occurs: <br />+CREG: 1,&lt;regStatus&gt; <br />  <br />OK <br />If the mode = 2, the following response occurs: <br /><span>+CREG: 2,&lt;regStatus&gt;[,&lt;netLac&gt;,&lt;netCellID&gt;[,&lt;AcT&gt;]]</span>  <br />where: <ul> <li> <span>&lt;regStatus&gt;</span> is the registration status, either: <ul> <li><span>0</span> – Not registered, the device is currently not searching for new operator.</li> <li><span>1</span> – Registered to home network.</li> <li><span>2</span> – Not registered, but the device is currently searching for a new operator.</li> <li><span>3</span> – Registration denied.</li> <li><span>4</span> – Unknown. For example, out of range.</li> <li><span>5</span> – Registered, roaming. The device is registered on a foreign (national or international) network.</li> </ul> </li> </ul> <ul> <li><span>&lt;netLac&gt;</span> is a two byte location area code in hexadecimal format</li> <li><span>&lt;netCellID&gt;</span> is the Cell ID in hexadecimal format, either: <ul><li>16 bit for 2G</li><li>28 bit for 3G or 4G</li></ul></li> <li><span>&lt;AcT&gt;</span> is radio access technology, either:<ul><li><span>0</span> – GSM</li><li><span>2</span> – UTRAN</li><li><span>3</span> – GSM w/EGPRS</li><li><span>4</span> – UTRAN w/HSDPA</li><li><span>5</span> – UTRAN w/HSUPA</li><li><span>6</span> – UTRAN w/HSDPA and w/HSUPA</li><li><span>7</span> – E-UTRAN</li></ul></li> </ul> <br />or <br /><span>ERROR</span> – the command failed. </td> </tr> <tr> <td> Write </td> <td> AT+CREG=&lt;<var>Mode</var>&gt; <br />where <span>&lt;<var>Mode</var>&gt;</span> is either: <ul> <li><span>0</span> – Disable network registration unsolicited result code. You will need to manually check the network registration status.</li> <li> <span>1</span> – Enable network registration unsolicited result code. If there is a change in network registration status, the modem will automatically return a response. </li> <li> <span>2</span> – Enable network registration and location information unsolicited result code. If there is a change in network registration status or at least one of the additional network information elements, the modem will automatically return a response. </li> </ul> </td> <td> <span>OK</span> - the network registration status mode is successfully set <br /><span>ERROR</span> - the command failed </td> </tr> </tbody> </table>

<Note> <p>Factors contributing to SIM failure to register on the network may include: </p> <ul> <li>Missing network coverage</li> <li>Denied network access</li> <li>No valid roaming agreement between the home network and currently available operators</li> </ul> </Note>

<h4>Example</h4>

<p>AT+CREG=2</p>

<p>OK</p>

<p>AT+CREG?</p>

<p>+CREG: 2,5,"54DB","0F6B0578",7</p>

<p>OK</p>
