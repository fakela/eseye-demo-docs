<p>You must attach policies to the dedicated IAM user in order to grant specific permissions, which will allow AnyNet IRIS to function.</p>

<Note>For detailed information about each policy, see Required IAM Managed Policies.</Note>

<ol> <li>Ensure you remain signed into AWS as the root user. </li> <li> Navigate to <span>IAM Services</span> using the following URL:  <br /><a href="https://console.aws.amazon.com/iam/">https://console.aws.amazon.com/iam/</a>  </li> <li> In the left-hand navigation menu, select <span>Users</span>.  <br /><img src="/images/resources/images/iris/iamuserlist.png" alt="Iamuserlist" />  <br />The AnyNet IRIS user you created is listed. </li> <li> Select the AnyNet IRIS IAM user name. <br />For example, select <span>anynetuser</span>. The IAM user Summary appears. </li> <li>On the <span>Permissions</span> tab, select <span>Add permissions</span>.</li> <li>Under <span>Grant permissions</span>, select <span>Attach existing policies directly.</span></li> <li>Using the <span>Search</span> box, search for: <span>AmazonEC2ReadOnlyAccess</span>.</li> <li> Select the check box alongside the returned result. <br /><img src="/images/resources/images/iris/grantpermissions.png" alt="Grantpermissions" />  </li> <li> Search for each of the following policies in turn, ensuring you select the check box alongside each returned listing: <ul> <li><span>AmazonEventBridgeFullAccess</span> </li> <li><span>AmazonS3FullAccess</span> </li> <li><span>AWSCloudFormationFullAccess</span> </li> <li><span>AWSCloudTrail&#95;FullAccess</span> </li> <li><span>AWSIoTFullAccess</span> </li> <li><span>AWSIoTLogging</span> </li> <li><span>IAMFullAccess</span> </li> </ul> <span><strong>Tip: </strong>If you select the wrong policy, clear the check mark alongside it.</span> </li> <li> Select <span>Next: Review</span>. <br />The selected policies are displayed. <br /><img src="/images/resources/images/iris/reviewqsg.png" alt="Reviewqsg" />  </li> <li> Select <span>Add permissions</span>. <br />The updated Summary page appears. </li> <li> Select <span>Add inline policy</span>. <br /><img src="/images/resources/images/iris/summary.png" alt="Summary" />  </li> <li> On the <span>JSON</span> tab, replace the existing text with the following JSON script:<pre>&#123; "Version": "2012-10-17", "Statement": [ &#123; "Sid": "AllowCrossAccountAccess", "Action": [ "sts:AssumeRole" ], "Effect": "Allow", "Resource": "arn:aws:iam::001813207414:role/AnyNetSecure@&lt;customer&#95;id&gt;" &#125;, &#123; "Sid": "DenyAllUserAndOrganizationRelatedActions", "Action": [ "iam:AddUserToGroup", "iam:AttachUserPolicy", "iam:CreateUser", "iam:DeleteUser", "iam:DeleteUserPermissionsBoundary", "iam:DeleteUserPolicy", "iam:DetachUserPolicy", "iam:PutUserPermissionsBoundary", "iam:PutUserPolicy", "iam:RemoveUserFromGroup", "iam:TagUser", "iam:UntagUser", "iam:UpdateUser", "organizations:&#42;" ], "Effect": "Deny", "Resource": "&#42;" &#125; ] &#125; </pre> </li> <li> In the JSON text, replace &lt;<var>customer&#95;id</var>&gt; with your AnyNet Cellular Connectivity for AWS IoT Customer ID. <br />If you cannot remember your Customer ID, see <a href="/pls62-w/aws-integration/pls62-w-view-welcome-page" target="_blank">Viewing the AnyNet IRIS Welcome page</a>. </li> <li> Select <span>Review policy</span>. <br />The Review policy page appears. <br /><img src="/images/resources/images/iris/reviewpolicy.png" alt="Reviewpolicy" />  </li> <li>In the <span>Name</span> field, type a name for the policy, for example: <span>AnyNetSecurePolicy</span>.</li> <li> Select <span>Create policy</span>. <br />The IAM user account Permissions policies updates to include the inline policy. <br /><img src="/images/resources/images/iris/permissionspolicies.png" alt="Permissionspolicies" />  </li> </ol>
