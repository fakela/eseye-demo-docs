<h2>+ETM Unsolicited Response Codes (URCs)</h2>

<p>AnyNet SMARTconnect™ adds the following Eseye URCs to the modem. All ETM URCs are prefixed <span>+ETM:</span>.</p>

<p>You will continue to observe other URCs from the Cinterion® PLS62‑W Wireless Module. </p>

<Note>For information about all other URCs, see the relevant Thales documentation. For more information, contact Thales or their channel partner.</Note>

<table> <col /> <col /> <thead> <tr> <th>URC</th> <th>Description</th> </tr> </thead> <tbody> <tr> <td> +ETM:EMQRDY </td> <td> ETM has entered MQTT mode and is ready to accept any <span>AT+EMQ...</span> commands. Prior to this, only <span>AT+ETM... </span>commands are accepted. </td> </tr> <tr> <td> +ETM:IDLE </td> <td> The modem has started up and is ready for commands. </td> </tr> <tr> <td> +ETM:REBOOTING </td> <td> The modem will shortly restart. Wait for <span>+ETM:</span><span>IDLE</span> before sending commands. Configure automatic rebooting using <span>update&#95;autoreboot</span> in the configuration file. For more information, see <a href="/pls62-w/configuration-file">Using the Cinterion® PLS62‑W Wireless Module configuration file</a>. </td> </tr> <tr> <td> +ETM: UNABLE TO OPEN MQTT </td> <td> The Cinterion® PLS62‑W Wireless Module has the certificates, and has tried and failed to open the data connection. No action required. The system will automatically restart in 30 minutes. </td> </tr> <tr> <td> +ETMSTATE:&lt;StateID&gt; <br />where <span>&lt;StateID&gt;</span> is the current connectivity state of the Cinterion® PLS62‑W Wireless Module. </td> <td> Enable this response using the <span>AT+ETMSTATE</span> write command. For more information, see <a href="/pls62-w/at-commands/etmstate#ETMSTATE">ETMSTATE – check current state</a>. </td> </tr> <tr> <td> +ETM:SYSSTART </td> <td>Indicates that AnyNet SMARTconnect™ application has started.</td> </tr> </tbody> </table>
