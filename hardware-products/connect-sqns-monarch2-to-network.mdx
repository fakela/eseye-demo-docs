<p>You can connect to a Cat-M1 or NB-IOT cellular network using a Sequans Monarch 2 GM02S-NEKTAR EVK (Evaluation Kit) and Eseye eUICC or AnyNet SIM. The following steps will help you through three stages of the network connection process:</p>

<ol> <li>Configuring automatic network operator selection.</li> <li>Checking the network registration status.</li> <li>Ensuring you can send and receive data.</li> </ol>

<div> <h3>Before you begin</h3> <ul> <li>Configure the device and Sequans Monarch 2 GM02S-NEKTAR EVK according to the manufacturer instructions. For example, ensure you have installed the correct drivers.</li> <li>Fully insert the Eseye SIM into your device.</li> <li>Ensure the Eseye SIM is activated on the Infinity Classic portal.</li> <li>Ensure the device is on and the Sequans Monarch 2 GM02S-NEKTAR EVK is powered up.</li> <li>Configure a terminal emulator to send commands to the Sequans Monarch 2 GM02S-NEKTAR EVK.</li> </ul> </div>

<p>To connect a Sequans Monarch 2 GM02S-NEKTAR EVK to a cellular network:</p>

<ol> <li> Using the terminal emulator, verify the device is reading the SIM correctly by requesting the ICCID and IMSI. Type: <br />AT+SQNCCID&lt;CR&gt; <br />AT+CIMI&lt;CR&gt; <span><strong>Note: </strong><span>&lt;CR&gt;</span> is the end-of-line character marking the end of a command line (alias &#92;r <span>– carriage return)</span>.</span> <span><strong>Tip: </strong>Make a note of the returned numbers in the event you need to ask for assistance later.</span> </li> <li> Using the terminal emulator, configure the Sequans Monarch 2 GM02S-NEKTAR EVK for minimum functionality. Type: <br />AT+CFUN=0&lt;CR&gt; </li> <li> Check which <a href="/getting-started/iot-wireless-technologies">LTE IoT technology</a> is allowed according to the installed modem firmware version, and if it is active. Type: <br />AT+SQNSIRSW?&lt;CR&gt; <br />+SQNSIRSW: ("NBIOT","&lt;<var>version</var>&gt;",&lt;mode&gt;),("CatM","&lt;<var>version</var>&gt;",&lt;mode&gt;) <br />where &lt;<var>version</var>&gt; is the firmware version, and &lt;<var>mode</var>&gt; is either: <ul> <li><span>0</span> – not active</li> <li><span>1</span> – active</li> </ul> <br />For example: <span>+SQNSIRSW: ("NBIOT","LR8.1.0.0-55629",1),("CatM","LR8.0.5.10-55042",0)</span> </li> <li> To switch technologies, type the following in order, waiting for each command response before proceeding: <br />If any of these commands returns ERROR, reboot the device and retry the command. You may have to do this several times. If you continue after the command returns ERROR, you may brick the board (render it useless) and it will require re-flashing with the modem firmware. <br />AT^RESET&lt;CR&gt; <br />OK <br />AT+SQNSFACTORYRESET&lt;CR&gt; <br />OK <br />AT+SQNSIRSW="&lt;TechType&gt;"&lt;CR&gt; <br />where &lt;<var>TechType</var>&gt; is either <span>NBIOT</span> or <span>CatM</span> <br />For example: <span>AT+SQNSIRSW="NBIOT"</span> <br />OK </li> <li> Determine which 4G LTE band the modem is allowed to use for different Radio Access Technologies (RATs). Type: <br />AT+SQNBANDSEL?&lt;CR&gt; <br />This will return either:  <ul> <li><span>AT+SQNBANDSEL=0</span> for LTE-M1</li> <li><span>AT+SQNBANDSEL=1</span> for LTE Cat NB1</li> </ul> </li> <li> Configure the device to search the allowed 4G LTE band. Type: <br />AT+SQNBANDSEL=&lt;<var>4GLTEband</var>&gt;,"standard","1,2,3,4,5,8,12,13,17,18,19,20,25,26,28,66,71" <br />where &lt;<var>4GLTEband</var>&gt; is either <span>0</span> or <span>1</span>, depending on the current RAT. <span><strong>Tip: </strong>The advanced user may want to limit the number of bands searched, which is beyond the scope of this topic.</span> </li> <li> Define the PDP context to route data through Eseye's APN. Type: <br />AT+CGDCONT=&lt;cid&gt;,"&lt;PDPtype&gt;","&lt;APN&gt;"&lt;CR&gt; <br />where <span>&lt;cid&gt;</span> is the context ID, <span>&lt;PDPtype&gt;</span> is the IP connection type, and <span>&lt;APN&gt;</span> is the relevant Eseye Access Point Name.  <br />For example: <span>AT+CGDCONT=1,"IP","eseye1"</span> <span><strong>Note: </strong>For information about which Eseye APN to use, speak to your Account Manager. Also see <a href="/connectivity/anynet-apn-list">Current AnyNet APN list</a>. </span> </li> <li> Configure the module for full functionality. Type: <br />AT+CFUN=1 </li> <li> Set the mode to enable <span>+CEREG</span> unsolicited result codes. Type: <br />AT+CEREG=1&lt;CR&gt; <span><strong>Note: </strong>For more information, refer to the Sequans Monarch 2 AT Commands manual for your module.</span> </li> <li> Set the mobile network operator (MNO) selection to <span>automatic</span>, which will force the modem to start searching for networks. Type: <br />AT+COPS=0 <br />If the device has registered on a network, the response will include the current MNO and access technology.  </li> <li> Check the MNO selection. Type:  <br />AT+COPS?&lt;CR&gt; <br />The response includes the operator and the access technology selected. The following example response is for a private network in NB-IoT mode: <br />+COPS: (2,"picoLTE Network","picoLTE","00101",9)(3,"vodafone UK","voda UK","234) <br />where <span>2</span> is the current private network, and <span>3</span> is forbidden. <br />In Cat-M1 mode, the final number is an 8. For example: <br />+COPS: (2,"picoLTE Network","picoLTE","00101",8) </li> <li> Determine the signal strength. Type: <br />AT+CSQ&lt;CR&gt; <br />The Sequans Monarch 2 GM02S-NEKTAR EVK may return any of the following values: <table> <col /> <col /> <thead> <tr> <th>Value</th> <th>Description</th> </tr> </thead> <tbody> <tr> <td>0</td> <td>  -113 dBm or less </td> </tr> <tr> <td>1</td> <td>  -111 dBm </td> </tr> <tr> <td>2-30</td> <td>  -109 to -53 dBm (when the value decreases by 1, the dBm increases by 2) </td> </tr> <tr> <td>31</td> <td>  -51 dBm or greater </td> </tr> <tr> <td>99</td> <td>Not known or not detectable</td> </tr> </tbody> </table> <br />For example: <br />+CSQ: 14,99 <br />where <span>14</span> relates to the RSSI ~ -85dBm <br />and <span>99</span> is the bit error rate <br />For information about the response, see the Monarch 2 LR 8.1 AT Command Reference Manual. </li> <li> Check the network registration state per network type. Type: <br />AT+CEREG?&lt;CR&gt; <br />The response includes the mode you set earlier and the registration status, which is either: <ul> <li><span>0</span> – Not registered, the device is currently not searching for new operator.</li> <li><span>1</span> – Registered to home network.</li> <li><span>2</span> – Not registered, but the device is currently searching for a new operator.</li> <li><span>3</span> – Registration denied.</li> <li><span>4</span> – Unknown. For example, out of range.</li> <li><span>5</span> – Registered, roaming. The device is registered on a foreign (national or international) network.</li> </ul> <br />For example: <br />+CEREG: 1,5 <div> Factors contributing to SIM failure to register on the network may include:  <ul> <li>Missing network coverage</li> <li>Denied network access</li> <li>No valid roaming agreement between the home network and currently available operators</li> </ul> </div> </li> <li> Open a data context. Type: <br />AT+CGATT=1&lt;CR&gt; <br />If the response is <span>OK</span>, the device can send and receive data. If <span>ERROR</span> or <span>+CME ERROR</span> is returned, the attempt to open a data context has failed. For more information, see the Sequans Monarch 2 AT Commands manual for your module. </li> <li> Check the allocated IP address. If you registered to the correct APN, this matches the IP address on the RADIUS log in Infinity Classic. Type: <br />AT+CGPADDR&lt;CR&gt; </li> <li> Use the AnyNet Ping service to ensure you can send and receive data packets. Type: <br />AT+PING="192.168.109.2"&lt;CR&gt; <br />A successful response is similar to the following: <br />+PING: &lt;responseNumber&gt;,&lt;IP address&gt;,&lt;Latency&gt;,&lt;TTL&gt; <br />where Latency is in milliseconds, and TTL is <a href="/glossary/ttl">TTL (Time To Live)</a>. <br />For example: <br />+PING: 1,192.168.109.2,1590,252 <br />+PING: 2,192.168.109.2,840,252 <br />+PING: 3,192.168.109.2,1410,252 <br />+PING: 4,192.168.109.2,7460,252 <br />OK </li> </ol>
