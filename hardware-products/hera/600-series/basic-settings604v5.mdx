<h2 id="ConfiguringtheRouter">Configuring the Router</h2>

<p>Describes general router information. </p>

<h3>Status</h3>

<p>For more information, see<a href="/hardware-products/hera/600-series/system-status604v5#Router">Router</a>.</p>

<h3>Name and location</h3>

<p>These user-defined settings help describe which Hera you are remotely logged into, so that you can verify you are viewing and modifying the correct router.</p>

<p>You can edit some basic router settings here:</p>

<p><span>Basic Settings</span> &gt; <span>Router</span> &gt; <span>Name and location</span></p>

<table> <col /> <col /> <thead> <tr> <th>Property</th> <th>Description</th> </tr> </thead> <tbody> <tr> <td><span>Site name</span> </td> <td>The physical site name where the Hera is situated.</td> </tr> <tr> <td><span>Site location</span> </td> <td>The site address.</td> </tr> <tr> <td><span>GPS</span> </td> <td> Select the slider to <span>Enable</span> or <span>Disable</span> GPS, if the device hardware supports this function. </td> </tr> <tr> <td><span>GPS location</span> </td> <td>Only visible if you <span>Enable</span> GPS. If the device hardware supports this function, the current GPS coordinates are displayed in latitude and longitude.</td> </tr> <tr> <td><span>Notes</span> </td> <td>Type any useful notes about the current Hera. For example, we recommend you use this parameter to note the filename of the factory configuration file stored in subversion.</td> </tr> </tbody> </table>

<h3>DHCP fixed hosts</h3>

<p>If you have configured DHCP address assignment, this ensures that the Hera always assigns the same unique IP address to a unique MAC address for those devices on your LAN that must have static addresses.</p>

<p>You can edit the DHCP fixed host settings here:</p>

<p><span>Basic Settings</span> &gt; <span>Local Area Network (LAN)</span> &gt; <span>DHCP fixed hosts</span></p>

<table> <col /> <col /> <thead> <tr> <th>Property</th> <th>Description</th> </tr> </thead> <tbody> <tr> <td><span>Name</span> </td> <td>Connection name to help identify the DHCP fixed host.</td> </tr> <tr> <td><span>IP address</span> </td> <td>The IPv4 IP address you want to assign to the MAC address.</td> </tr> <tr> <td><span>MAC address</span> </td> <td>The unique MAC address of the device network interface controller. Use <span>:</span> to separate the digit pairs.</td> </tr> </tbody> </table>

<p>To create a new table entry:</p>

<ol> <li> Under the table, type the new entry name into the text box. <br /><img src="/images/resources/images/hera/createnewentry.png" alt="Createnewentry" />  </li> <li> Select <span>Create</span> to add the entry to the table. </li> <li> Complete the entry details, as described in the table above. </li> <li> Select <span>Save</span> to commit your changes to the Hera. </li> </ol>

<p>You can revert local changes to the last save.</p>

<p>To revert your local changes:</p>

<ul> <li> Select <span>Reset</span>. <br />All local changes are removed and the previous settings are displayed. </li> </ul>

<p>To delete a table entry:</p>

<ol> <li> Alongside the item you want to delete, select <span>Remove</span>. </li> <li> If required, select <span>Reinstate</span> alongside the item to undo the deletion. </li> <li> Select <span>Save</span> to commit your changes. <span><strong>Note: </strong>After you have saved, you can no longer reinstate a deleted item.</span> </li> </ol>

<p>You can sort the table entries into a logical list.</p>

<p>To sort a table entry in the list:</p>

<ol> <li> Alongside the item you want to promote to the top of the list, select <span>Promote</span>. <br />The item moves up one row. </li> <li> When you have sorted the items, select <span>Save</span> to save your changes. </li> </ol>

<h2 id="ConfiguringtheMobileNetwork">Configuring the Mobile Network</h2>

<h3>Status (read only)</h3>

<p>For more information, see<a href="/hardware-products/hera/600-series/system-status604v5#Mobile">Mobile network</a> </p>

<h3>Configuring the mobile connection</h3>

<p>The Hera is equipped for mobile connection, which you can enable or disable. It has multiple antennas for antenna diversity, to improve the quality and reliability of a wireless link.</p>

<h4>Configuring the Hera to use a SIM</h4>

<p>You can configure up to three SIMs simultaneously – a ChipSim (embedded in the Hera) and two SIM cards. You must insert the SIM cards before you physically install the Hera. </p>

<p>For information about installing the Hera, see the <a href="https://docs.eseye.com/Content/Resources/Files/8435-Hera-604v5-Series-Router-Quick-Start-Guide.pdf" target="_blank">8435 Hera 604v5 Series Router Quick Start Guide (PDF)</a>.</p>

<Note>Eseye preconfigures the Hera cellular connection for you. </Note>

<p>To configure SIMs:</p>

<ol> <li> Using the Hera user interface, navigate to <span>Basic Settings</span> &gt; <span>Mobile Network</span> &gt; <span>Connection</span>. <br /><img src="/images/resources/images/hera/basicsettingsconnection.png" alt="Basicsettingsconnection" />  </li> <li>Enable the <span>Mobile connection</span>.</li> <li> Only for Hera models <span>H604xx-5...</span> or <span>H604xx-4...</span>. If required, enable the <span>Receive diversity</span> antenna.  <br />For more information, see<a href="/hardware-products/hera/600-series/end-panel-connectors604v5#top" target="_blank">Top panel functions</a>. </li> <li> Leave <span>Metric</span> as the default (2) unless you are planning to configure multiple WAN connections, such as Ethernet protected by cellular. <br />For information about the Health Monitor metric shift, see <a href="/hardware-products/hera/600-series/health-monitor604v5#General" target="_blank">Configuring general settings</a>. <span><strong>Tip: </strong>Contact Support if you would like to set up more complex multi-WAN interface configurations.</span> </li> <li> If your system is not preconfigured and you need to create a cellular profile, type a <span>Profile name</span> in the text box, then select <b>Create</b>. <br /><img src="/images/resources/images/hera/createcellularprofile.png" alt="Createcellularprofile" />  <br />The profile appears listed in the table. You must add connection details to the row. </li> <li> Type the mandatory <b>APN</b> name for the profile.  <br />Eseye may preconfigure multiple APNs to link to an MNO. Always use the APN specified in your contract. <br />For information about APNs, see <a href="/connectivity/anynet-apn-list">Current AnyNet APN list</a>. </li> <li>Type the APN <b>User name</b> to enable Hera access to the APN.</li> <li>Type the APN <b>Password</b>.</li> <li> If the SIM is a locked SIM, type the four digit SIM <span>PIN</span>. <br />This gives the Hera read access to the SIM and enables SIM access to the mobile operator's network. </li> <li> Select which <span>SIM</span> is linked to this profile, either: <ul> <li><span>CHIP</span> – the embedded ChipSIM</li> <li><span>SIM 1</span> – the SIM in slot SIM1</li> <li><span>SIM 2</span> – the SIM in slot SIM2</li> </ul> </li> <li>Select the <span>Active</span> option if this is the active profile that the Hera must use.</li> <li>At the top right, select <span>Save</span>.</li> </ol>

<Note>Ensure the Health Monitor configuration is correct, so that the Health Monitor does not fail its ping test following mobile connection changes. For more information, see<a href="/hardware-products/hera/600-series/health-monitor604v5" target="_blank">Configuring the Health Monitor</a>.</Note>
