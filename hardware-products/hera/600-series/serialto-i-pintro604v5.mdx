<p>The Hera router Serial to IP agent enables you to transfer RS-232 serial data over an IP link. You can use the agent to:</p>

<ul> <li>Enable remote connection to serial-only devices</li> <li>Configure either inbound only, or inbound and outbound connection to TCP, Telnet, or UDP</li> <li>Run executable applications or scripts stored in the Hera file system from locally connected serial devices</li> <li>Run device-originated queries to gather information from the Hera that is sent back to the device</li> <li>Extend an RS-232 link over a long distance to log trace data from a device to a remote database</li> </ul>

<p>The Serial to IP agent links a serial port to any of the following:</p>

<ul> <li>TCP stream</li> <li>Telnet stream</li> <li>Bi-directional UDP datagrams collection</li> </ul>

<Note>Relative timing data of sent and received characters is not preserved. </Note>

<p>You can set up to two serial to IP ports, <span>serial1</span> and <span>serial2</span>. You need to configure these individually. For more information, see <a href="#Using">Using the Serial to IP agent configuration file</a>.</p>

<p>You can use delimiters to control when data is sent. The Serial to IP agent can buffer messages as groups of characters. For more information, see <a href="#Message">Message termination and delimitation</a>.</p>

<h2 id="IPconnectivity">IP connectivity</h2>

<p>Connection modes are either:</p>

<ul> <li> <span>Inbound only</span> – configured by setting <span>listenport</span> </li> <li> <span>Inbound and outbound</span> – configured and enabled by setting <span>ipaddress</span> and <span>port</span> </li> </ul>

<p>For more information, see <a href="#Using">Using the Serial to IP agent configuration file</a>.</p>

<Note>For Telnet operation only. Telnet signalling information and control characters, such as character echo, are not sent. The Serial to IP agent adds escaped Interpret as Command characters (IACs) from serial to IP, and strips IACs from IP to serial.</Note>

<h2 id="Messageterminationanddelimitation">Message termination and delimitation</h2>

<p>You can set one or more of the following delimiters using the Serial to IP agent configuration file. </p>

<Tip>For more information, see <a href="#Using">Using the Serial to IP agent configuration file</a>.</Tip>

<table> <col /> <col /> <thead> <tr> <th>Delimiter</th> <th>Description</th> </tr> </thead> <tbody> <tr> <td> <span>Termination characters</span>  </td> <td> Use <span>fwdcheckchars</span> to configure up to 10 individual message termination characters per serial stream, which indicate the end of the message. If any of these characters is matched from the incoming stream, the Serial to IP agent assumes the current buffer is a complete message, which it then forwards to IP.  </td> </tr> <tr> <td> <span>Maximum string length</span>  </td> <td> Use <span>fwdnumchars</span> to configure message forwarding after the serial port receives a specified number of characters.  </td> </tr> <tr> <td> <span>Termination string</span>  </td> <td> Use <span>fwdstring</span> to configure a single termination string to indicate the end of a message. When the string is matched in the incoming character stream, the Serial to IP agent forwards the current buffer as a complete message. </td> </tr> <tr> <td> <span>Timeout termination</span>  </td> <td> Use <span>fwdtimer</span> to configure a rest time interval, during which no further characters are received. <h2 id="Connectionstateindication">Connection state indication</h2>

<p>The Serial to IP agent can indicate the TCP or Telnet host connection state to the serial device by transmitting connect and disconnect messages from the serial port to the device. When the connection is not available, the agent buffers the characters received from the serial port and transmits them to the TCP or Telnet stream after connection is re-established.</p>

<p>For more information, see <span>fwdtimer</span> and <span>usdelimiter</span> in <a href="#Using">Using the Serial to IP agent configuration file</a>.</p>

<h2 id="Sendingserialcommandsorqueries">Sending serial commands or queries</h2>

<p>You can configure the Serial to IP agent to process commands and queries on the host that are sent by a locally connected serial device. Such commands and queries are not forwarded to IP, but use the same serial connection as traffic that is passed to IP. You can use the commands to run executable applications or scripts stored in the Hera file system. You can use the queries to gather information from the Hera that is sent back to the device. For example, the device can use a script to query the Hera for the current time.</p>

<p>To make use of this feature, you must set a command prefix (<span>cmdprefix</span>) in the configuration file, to demarcate messages that the Serial to IP agent processes locally as a command or query. Subsequent messages without the prefix are processed for IP. </p>

<p>The Serial to IP agent sends all standard output (stdout) characters from the running application or script back to the serial port, prefixed with the <span>cmdprefix</span>. </p>

<p>Commands are defined globally within the configuration file (not per serial port). </p>

<p>For more information, see <a href="#Using">Using the Serial to IP agent configuration file</a>.</p>

<h2 id="UsingtheSerialtoIPagentconfigurationfile">Using the Serial to IP agent <span>configuration file</span></h2>

<p>You can find the Serial to IP agent configuration file in the Hera file system: <span>/etc/config/serial2ip</span></p>

<p>Alternatively, use the Hera web interface to edit the file:</p>

<p><span>Advanced Settings</span> &gt; <span>File Editor</span> &gt; <span>General Configuration</span> &gt; <span>Contents</span> &gt; <span>serial2ip</span> &gt; <span>Edit</span></p>

<p><img src="/images/resources/images/hera/advancedconfigfilesserial2ip.png" alt="Advancedconfigfilesserial2Ip" /> </p>

<h3>Formatting the Serial to IP agent <span>configuration file</span></h3>

<p>You must individually configure the serial ports you want to use (up to two), and any commands or queries, in the following format:</p>

<p>config serial2ip &lt;<var>serialport</var>&gt;</p>

<p>option device '&lt;<var>serialPortPath</var>&gt;'</p>

<p>option protocol '&lt;<var>protocol</var>&gt;'</p>

<p>option enabled '&lt;<var>messageForwardingOnOrOff</var>&gt;'</p>

<p>...</p>

<p>config serial2ip &lt;<var>serialport</var>&gt;</p>

<p>...</p>

<p>config command</p>

<p>option command '<var>&lt;commandName&gt;</var>'</p>

<p>option execfile '/<var>&lt;absolutePath&gt;</var>'</p>

<p>config command</p>

<p>...</p>

<p>Configuration file parameters are written in the format: <span>option &lt;<var>parameter</var>&gt; '&lt;<var>value</var>&gt;'</span>, where <span>&lt;<var>parameter</var>&gt;</span> is the setting you want to configure, and <span>&lt;<var>value</var>&gt;</span> defines that setting. </p>

<p>For example: <span>option baud '115200'</span></p>

<Note>You can disable a parameter using <span>''</span> (single quotation marks with no space between them). For example: <span>option connectmsg ''</span></Note>

<h4>Formatting commands in the Serial to IP agent configuration file</h4>

<p>You can add multiple commands to the Serial to IP agent command list. They consist of the following:</p>

<ul> <li>A section header to mark that what follows is a command or query</li> <li>A command name string for identifying the command</li> <li>The absolute path of a script or application on the Hera file system, which runs when the command is issued </li> </ul>

<p>For example:</p>

<p>config command</p>

<p>option command 'uptime'</p>

<p>option execfile '/bin/uptime'</p>

<h4>Example configuration</h4>

<p>In the below example, the <span>serial1</span> configuration means that the Serial to IP agent will collate the characters received from the device until there are 10 characters to send. </p>

<p>The agent will also monitor for a time gap that is greater than 2000mS, when all previously received characters are considered a complete message and forwarded. </p>

<p>This means that if the agent receives eight characters, and then receives a further five characters after a 1500mS time gap, then 10 characters are immediately sent. The remaining three characters are held until the agent receives seven more characters (or the timeout occurs).</p>

<p>config serial2ip serial1</p>

<p>option device '/dev/eser0'</p>

<p>option protocol 'tcp'</p>

<p>option enabled 'true'</p>

<p>option fwdnumchars '10'</p>

<p>option fwdtimer '2000'</p>

<p>option </p>

<p>config serial2ip serial2</p>

<p>option device '/dev/eser1'</p>

<p>option protocol 'udp'</p>

<p>option enabled 'false'</p>

<p>config command</p>

<p>option command 'uptime'</p>

<p>option execfile '/bin/uptime'</p>

<p>config command</p>

<p>option command 'chktime'</p>

<p>option execfile '/bin/timechecker'</p>

<p>If the agent receives a message containing the <span>cmdprefix</span>, followed by the <var>commandName</var> string, then it will run the associated script or application on the Hera. For example:</p>

<p><span>+++chktime</span> will run the <span>timechecker</span> script in the Hera <span>/bin/</span> location.</p>
