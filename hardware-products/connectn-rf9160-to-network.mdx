<p>You can connect to a cellular network using a Nordic nRF9160 modem and Eseye eUICC or AnyNet SIM. The following steps will help you through three stages of the network connection process:</p>

<ol> <li>Configuring automatic network operator selection.</li> <li>Checking the network registration status.</li> <li>Ensuring you can send and receive data.</li> </ol>

<div> <h3>Before you begin</h3> <ul> <li>Configure the Nordic nRF9160 modem according to the manufacturer instructions. For example, ensure you have installed the correct drivers and requisite firmware.</li> <li>Fully insert the Eseye SIM into your device.</li> <li>Ensure the Eseye SIM is activated on the Infinity Classic portal.</li> <li>Ensure the device is on and the Nordic nRF9160 modem is powered up.</li> <li>Configure a terminal emulator to send commands to the Nordic nRF9160 modem.</li> </ul> </div>

<p>To connect a Nordic nRF9160 modem to a cellular network:</p>

<ol> <li> Using the terminal emulator, configure the module for full functionality. Type: <br />AT+CFUN=1&lt;CR&gt; <span><strong>Note: </strong><span>&lt;CR&gt;</span> is the end-of-line character marking the end of a command line (alias &#92;r <span>– carriage return)</span>.</span> </li> <li> Verify the device is reading the SIM correctly by requesting the ICCID and IMSI. Type: <br />AT+CRSM=176,12258,0,0,10&lt;CR&gt; <br />The response is similar to the following: <br />+CRSM: 144,0,"&lt;<var>ICCID</var>&gt;" <br />where ICCID is in binary coded decimal format, with switched digits. So the returned response <span>2143658709</span> actually means <span>1234567890</span>. <br />To clarify: <br />2<b>1</b> 4<b>3</b> 6<b>5</b> 8<b>7</b> 0<b>9</b> <br />Switch places: <br /><b>1</b>2 <b>3</b>4 <b>5</b>6 <b>7</b>8 <b>9</b>0 <br />Type:  <br />AT+CIMI&lt;CR&gt; <span><strong>Tip: </strong>Make a note of the returned numbers in the event you need to ask for assistance later.</span> </li> <li> Configure the Nordic nRF9160 modem for minimum functionality. To ensure the radio is off, type: <br />AT+CFUN=0&lt;CR&gt; </li> <li> Configure the supported system modes for both LTE-M (Cat M1) and NB-IoT, with a preference of LTE-M. Type: <br />AT%XSYSTEMMODE=1,1,0,1&lt;CR&gt; <span><strong>Note: </strong>The Nordic nRF9160 modem does not support other RAT types.</span> <span><strong>Note: </strong>System modes are region and network specific, and may not exist where your device is connected. For example, in India you can only access NB-IoT. </span> </li> <li> Configure the device to search all available frequency bands. Type: <br />AT%XBANDLOCK=0&lt;CR&gt; <span><strong>Tip: </strong>The advanced user may want to limit the number of bands searched, which is beyond the scope of this topic.</span> </li> <li> Set the mobile network operator selection to <span>automatic</span>. Type: <br />AT+COPS=0&lt;CR&gt; </li> <li> Define the PDP context to route data through Eseye's APN. Type: <br />AT+CGDCONT=&lt;cid&gt;,"&lt;PDPtype&gt;","&lt;APN&gt;"&lt;CR&gt; <br />where <span>&lt;cid&gt;</span> is the context ID, <span>&lt;PDPtype&gt;</span> is the IP connection type, and <span>&lt;APN&gt;</span> is the relevant Eseye Access Point Name.  <br />For example: <span>AT+CGDCONT=0,"IP","eseye1"</span> <span><strong>Note: </strong>For information about which Eseye APN to use, speak to your Account Manager. Also see <a href="/connectivity/anynet-apn-list">Current AnyNet APN list</a>. </span> </li> <li> Configure the module for full functionality. Type: <br />AT+CFUN=1&lt;CR&gt; </li> <li> Set the mode to enable <span>+CEREG</span> unsolicited result codes. Type: <br />AT+CEREG=1&lt;CR&gt; <span><strong>Note: </strong>For more information, see the Nordic nRF9160 modem AT Commands documentation.</span> </li> <li> Check the network operator selection. Type:  <br />AT+COPS?&lt;CR&gt; <br />The response includes the operator and the access technology selected. The following example response is for Verizon in LTE Cat M1 mode: <br />+cops: 0,0,"Verizon ",8 <br />The following example response is for T-Mobile in NB-IoT mode: <br />+cops: 0,0,"T-Mobile USA",9 </li> <li> Determine the signal strength. Type: <br />AT+CSQ&lt;CR&gt; </li> <li> Check the network registration state. Type: <br />AT+CEREG?&lt;CR&gt; <br />The response includes the mode you set earlier and the registration status, which is either: <ul> <li><span>0</span> – Not registered, the device is currently not searching for new operator.</li> <li><span>1</span> – Registered to home network.</li> <li><span>2</span> – Not registered, but the device is currently searching for a new operator.</li> <li><span>3</span> – Registration denied.</li> <li><span>4</span> – Unknown. For example, out of range.</li> <li><span>5</span> – Registered, roaming. The device is registered on a foreign (national or international) network.</li> </ul> <br />Example response: <br />+CEREG: 1,5 <br /> <div> Factors contributing to SIM failure to register on the network may include:  <ul> <li>Missing network coverage</li> <li>Denied network access</li> <li>No valid roaming agreement between the home network and currently available operators</li> </ul> </div> </li> <li> Open a data context. Type: <br />AT+CGATT=1&lt;CR&gt; <br />If the response is <span>OK</span>, the device can send and receive data. If <span>ERROR</span> or <span>+CME ERROR</span> is returned, the attempt to open a data context has failed. Refer to the Nordic nRF9160 modem AT Commands manual for more information. </li> <li> Check the allocated IP address. If you registered to the correct APN, this matches the IP address on the RADIUS log in Infinity Classic. Type: <br />AT+CGPADDR&lt;CR&gt; </li> <li> Log into Infinity Classic and check the SIM status is <span>Active</span> on the SIM list page to ensure the SIM is connected. </li> <li> If you have integrated the Nordic nRF9160 modem into the IP stack of another device, such as a Raspberry Pi, you can check the connectivity by ensuring you can send and receive data packets via the modem. We recommend pinging the AnyNet Ping service: <br />ping 192.168.109.2&lt;CR&gt; <br />A successful response is similar to the following: <br />OK <br />+PING: 0,"192.168.109.2",32,744,255 <br />+PING: 0,"192.168.109.2",32,257,255 <br />+PING: 0,"192.168.109.2",32,234,255 <br />+PING: 0,"192.168.109.2",32,238,255 <br />+PING: 0,4,4,0,234,744,368 <span><strong>Note: </strong>If the Nordic nRF9160 modem is not integrated into the IP stack of another device, you cannot use the ping function. For more information, see <a href="https://devzone.nordicsemi.com/f/nordic-q-a/50537/ping-tcp-ip-server-with-at-command" target="_blank">Ping TCP/IP server with AT Command (external link)</a>.</span> </li> </ol>
