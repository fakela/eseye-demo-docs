<p>Eseye generates Rated Data Record Exit Files (REFs) to record your precise network usage, for billing purposes. These records are only generated if a device is active on a network. They are initially unrated raw data (Call Data Records), which are then enriched with billing information to become rated records. The REFs are combined to calculate your monthly invoice. They contain information such as start time, duration, and source and destination details. The REFs are updated every half an hour. Eseye holds the files for 90 days.</p>

<p>You can request the REFs for your own use, such as to monitor your network usage for rogue devices that have breached in bundle agreements and are causing network charges to escalate. </p>

<p>You can pull the records from the Eseye AWS server. Alternatively, Eseye can securely push the REFs using the SFTP protocol. The default frequency for pushing the files is every half an hour, but you can request different timing (for example, every 24 hours). You must set up a folder where you want to receive the REFs, and share credentials for this folder with Eseye. </p>

<p><img src="/images/resources/images/billing/networkbillingarchitecture.png" alt="Networkbillingarchitecture" /> </p>

<Note>Speak to your Account Manager for more information.</Note>

<p> The filename is saved with the following format:</p>

<p><span>ref-&lt;<span>yyyy-mm-dd&gt;-&lt;REFstarttime&gt;-&lt;REFstoptime</span>&gt;-&lt;<span>portfolioId</span>&#95;&lt;filenumber&gt;</span> </p>

<p>where:</p>

<ul> <li> <var>yyyy-mm-dd&gt;</var> is the date when the REF was generated </li> <li> <var>REFstarttime</var> is the time period when the REF started processing data (in hh and mm, for example 0025 is 00h25) </li> <li> <var>REFstoptime</var> is the time period when the REF stopped processing data (in hh and mm, for example 1418 is 14h18)  </li> <li>  <var>portfolioId</var> is the portfolio the REF file refers to </li> <li> <var>filenumber</var> is incremented if there is more than one REF for that time window </li> </ul>

<h2 id="VerifyingtheREFsusingthePackageItemsnapshotfile">Verifying the REFs using the Package Item snapshot file</h2>

<p>Every 24 hours Eseye also generates a <span>PackageItem&#95;snapshot</span> file per portfolio. The PackageItem&#95;snapshot lists all the <a href="/glossary/package-item">package items</a> in each of your <a href="/glossary/package">packages</a>, and their related costs. You can use this information to verify the costs applied in the REFs. </p>

<p>The filename is saved with the following format: </p>

<p><span>PACKAGEITEM&#95;SNAPSHOT-&lt;<span>yyyy-mm-dd</span>&gt;-&lt;<span>fileId</span>&gt;-PORTFOLIO-&lt;<span>portfolioId</span>&gt;</span> </p>

<p>where: </p>

<ul> <li> <var>yyyy-mm-dd</var> is the date the information was captured </li> <li> <var>fileId</var> is the unique identifier for the PackageItem&#95;snapshot </li> <li> <var>portfolioId</var> is the portfolio the PackageItem&#95;snapshot refers to </li> </ul>
