<Note>
  For devices that have never previously connected, the modem will attempt to connect according to the IMSI order that was configured during SIM manufacture. For devices that have previously connected to a network, the modem will attempt to connect to the last successfully connected network.
</Note>

<Note>
  Different companies configure their devices to implement the rotation timers in different ways. For example, some customer devices use a timer to count the IMSI rotation time, whereas other customer devices count the number of status commands.
</Note>

## How the limited service fallback timer behaves

The fallback timer is effectively a **reset**, not a pause/resume. It only runs during a _continuous_ limited service condition. If the device transitions into another state — such as PLMN search or normal service — the timer is reset and only starts again when limited service is re-entered.

### Example 1: Timer resets, fallback does not trigger

1. The device enters **limited service**.
2. The device transitions into **PLMN search** and subsequently into **normal service**.
3. Because the limited service condition was broken, the timer does not run continuously for 3 minutes, so fallback does not trigger at the expected time.

<Note>
  A PLMN search effectively breaks the limited service condition and resets the fallback timer.
</Note>

### Example 2: Timer runs continuously, fallback triggers

1. The device enters **limited service**.
2. The device briefly transitions, but returns to limited service **without** reaching normal service.
3. The timer runs continuously, and fallback occurs at ~3 minutes as expected.
