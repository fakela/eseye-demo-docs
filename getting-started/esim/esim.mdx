## What is an eSIM?

An **eSIM** (embedded SIM) is a small chip permanently soldered onto a device's circuit board. Unlike traditional removable SIM cards, an eSIM's contents are rewritable over the air, allowing you to:

- Provision a device onto a mobile network without inserting a physical SIM.
- Switch between mobile network operators (MNOs) remotely.
- Manage connectivity for devices deployed in the field, sealed enclosures, or harsh environments.

<Note>
  The GSMA defines different eSIM architectures for machine-to-machine (M2M) and consumer device use cases. The Eseye eSIM solution uses the M2M eSIM architecture.
</Note>

## Why use eSIM for IoT?

eSIM technology is purpose-built for IoT deployments where physical SIM access is impractical or impossible:

| Benefit | Description |
| --- | --- |
| **Remote provisioning** | Download and activate operator profiles over the air, anywhere in the world. |
| **Network flexibility** | Change MNOs to optimise coverage, performance, or cost without recalling devices. |
| **Durability** | Soldered form factor (MFF2) withstands shock, vibration, temperature, and humidity. |
| **Tamper resistance** | Cannot be removed or swapped, reducing theft and fraud risk. |
| **Single SKU** | Manufacture once, deploy anywhere — localise connectivity at the destination. |

## Key components

An eSIM solution is made up of several interrelated components:

- **eSIM** — the physical embedded chip on the device.
- **eUICC** — the software (embedded Universal Integrated Circuit Card) that runs on the chip and stores multiple network profiles. See [eUICC overview](/getting-started/esim/euicc).
- **Profile** — operator credentials (IMSI, ICCID, keys) that allow the device to authenticate on a specific mobile network. See [Profiles](/getting-started/esim/profiles).
- **Remote SIM Provisioning (RSP)** — the GSMA-defined framework for downloading, enabling, disabling, and deleting profiles over the air. See [Remote SIM Provisioning overview](/getting-started/esim/remote-sim-provisioning-overview).

## eSIM vs. traditional SIM

|  | Traditional SIM | eSIM |
| --- | --- | --- |
| Form factor | Removable (2FF, 3FF, 4FF) | Soldered (MFF2) |
| Operator change | Physical swap | Remote download |
| Profiles stored | One | Multiple |
| Suitable for IoT | Limited | Yes |
| Lifecycle management | Manual | Automated via RSP |
