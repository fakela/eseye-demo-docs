<p>To access packages using Infinity:</p>

<ol> <li> On the left menu, select <span>Finance</span>. <br /> <img src="/images/resources/images/infinity/financeicon-only_80x78.png" alt="Financeicon Only 80X78" />  </li> <li> Select <span>Packages</span> to view the packages list.  </li> </ol>

<h2 id="Viewingallpackages">Viewing all packages</h2>

<p>The package list displays all packages associated with the portfolio assigned to the current user.</p>

<p>The Infinity package list displays the following fields:</p>

<table> <col /> <col /> <thead> <tr> <th>Field</th> <th>Description</th> </tr> </thead> <tbody> <tr> <td><span>Package ID</span> </td> <td> The unique identifier for the package. </td> </tr> <tr> <td><span>Name</span> </td> <td> The package name, which contains enough detail to distinguish the package, such as customer name, amount of bundled data, geographies, and so on. </td> </tr> <tr> <td><span>Status</span> </td> <td> The package status, either: <ul> <li>[CDATA[ ]]<span>Active</span> – approved and ready for use.</li> <li>[CDATA[ ]]<span>Legacy</span> – a redundant package that is required for auditing purposes on past transactions, but is not available for use.</li> <li><span>Pending</span> – awaiting approval to change into another state. Not available for use.</li> <li><span>Suspended</span> – available for use but you cannot continue to assign SIMs to it. However, you can manage the lifecycle of the SIMs that already exist within this package. </li> <li><span>Unsigned</span> – awaiting the customer's signature and Eseye Ltd final approval. Not available for use.</li> </ul> </td> </tr> </tbody> </table>

<h3>Filtering the displayed packages</h3>

<p>Refine the displayed <span>Packages</span> list using the <span>Status</span> filter.</p>

<p><img src="/images/resources/images/infinity/packagesstatusfilter.png" alt="Packagesstatusfilter" /> </p>

<p>To filter the packages list by status:</p>

<ol> <li>On the <span>Packages</span> page, select the <span>Status</span> drop-down list to view the available package statuses.</li> <li> Select the relevant status, or select <span>All</span> to view all packages. <br />The list immediately filters. </li> </ol>

<h2 id="ViewingSIMsassociatedwithapackage">Viewing SIMs associated with a package</h2>

<p>To view SIMs associated with a package:</p>

<ol> <li>On the <span>Packages</span> page, in the packages list, select the package to view its associated SIMs.</li> <li> Select <span>Actions</span> &gt; <span>View SIMs</span> (top-right). <br />The SIM and Device Search page appears, displaying a list of associated SIMs for the selected package. </li> </ol>

<h2 id="Changingthepackagestatus">Changing the package status</h2>

<p>If you have the relevant permissions, you can use <span>Infinity</span> to change a package status. For information about the package statuses, see <a href="#Package">Viewing all packages</a>.</p>

<p>You cannot change the package status if the current status is <span>Legacy</span>, <span>Suspended</span>, or <span>Unsigned</span>. </p>

<Tip>If you require help to change a package status, contact Support.</Tip>

<p>To change the package status:</p>

<ol> <li>On the <span>Packages</span> page, in the packages list, select the package that requires a status change.</li> <li> Select <span>Actions</span> &gt; <span>Change Status</span> (top-right). <span><strong>Note: </strong>You will only see a list of available statuses that you have permission to use. </span> </li> <li> In the <span>Status change</span> dialog, select the new status. <span><strong>Note: </strong>Changing a package status may affect your ability to activate SIMs associated with that package.</span> </li> <li> Select <span>Confirm</span> to initiate the change. </li> </ol>
