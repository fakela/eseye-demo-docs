<p>To manage packages using Integra:</p>

<ul> <li> On the left menu, select <span>Packages</span> to view the Packages page. <br /> <img src="/images/resources/images/integra/packages-icon.png" alt="Packages Icon" />  <br /><img src="/images/resources/images/integra/packages-new-menu.png" alt="Packages New Menu" />  </li> </ul>

<h2 id="Viewingallpackages">Viewing all packages</h2>

<p>The package list displays all packages associated with the portfolio assigned to the current user.</p>

<p>The Integra package list displays the following fields:</p>

<table> <col /> <col /> <thead> <tr> <th>Field</th> <th>Description</th> </tr> </thead> <tbody> <tr> <td><span>Package ID</span> </td> <td> The unique identifier for the package. </td> </tr> <tr> <td><span>Name</span> </td> <td> The package name, which contains enough detail to distinguish the package, such as customer name, amount of bundled data, geographies, and so on. </td> </tr> <tr> <td><span>Description</span> </td> <td> Extra notes about the package for internal use only, such as activation details or customer-related information. For example:  <ul> <li>Activate SIMs before shipping </li> <li>Enable MT SMS, disable MO SMS </li> <li>1 month proof of concept contract </li> </ul> </td> </tr> </tbody> </table>

<h3>Filtering the displayed packages</h3>

<p>Refine the displayed <span>Packages</span> list using any of the supplied filters.</p>

<p>To filter the Packages list:</p>

<ol> <li> On the <span>Packages</span> page, use any of the following filters to constrain the available list and find a specific package: <ul> <li><span>Package ID</span> – type the full package ID.</li> <li><span>Package Name</span> – type the full package name.</li> <li><span>Package Status</span> – select the required status from the drop-down list.</li> </ul> </li> <li> Select <span>Search</span> to filter the displayed packages. </li> <li>Alternatively, select <span>Reset</span> to clear the filter constraints.</li> </ol>

<h2 id="Creatingpackages">Creating packages</h2>

<p>To create a new package:</p>

<ol> <li> On the <span>Packages</span> page, select <span>Actions</span> &gt; <span>Create Package</span> (top-right) to display the <span>Create new package</span> dialog.  <br /><img src="/images/resources/images/integra/createnewpackagedialog.png" alt="Createnewpackagedialog" />  </li> <li> Provide the following information: <table> <col /> <col /> <thead> <tr> <th>Field</th> <th>Description</th> </tr> </thead> <tbody> <tr> <td><span>Package Name</span> </td> <td> The package name, which contains enough detail to distinguish the package, such as customer name, amount of bundled data, geographies, and so on. </td> </tr> <tr> <td><span>Title</span> </td> <td> The short unique label (no spaces) for identifying the package. </td> </tr> <tr> <td><span>Category</span> </td> <td> The unique package category identifier. A package category may describe a physical hardware item, delivery charge, or a service item or tariff.[CDATA[ ]] </td> </tr> <tr> <td><span>Currency</span> </td> <td> The trading currency name. </td> </tr> <tr> <td><span>Price</span> </td> <td> The periodic service charge per SIM, including service charge and bundled usage charges. </td> </tr> <tr> <td><span>Billing Period</span> </td> <td> The number of months covered by the price. Consumer invoices are generated monthly. </td> </tr> <tr> <td><span>Default Mobile Country Code</span> </td> <td> The default mobile country code identifier used by Billing for real time rating and alerting if the device location is not immediately identified. When the location is identified, the charges are re-rated if required.  </td> </tr> <tr> <td><span>Default Package ID</span> </td> <td>The default package identifier that is used to process how much a SIM is charged for a specific event if a package item is not defined for that event on the current package. For example, if a SIM associated with the current package receives an SMS, and MT SMS is not defined as a package item on the package, then the system will use the MT SMS package item values that are defined in the default package instead. <span><strong>Tip: </strong>The default value is <span>1</span>.</span></td> </tr> <tr> <td><span>Portal</span> </td> <td> The unique portal identifier. Portals controls user permissions and determines what they can access. </td> </tr> </tbody> </table> </li> <li> Select <span>Save</span> to create the package. <br />A new package appears, with a Status of Setup. You must update the package to contain more details. </li> <li> Add contract information to define pricing and testing arrangements. <br />For more information, see <a href="/packages/manage-contract-information" target="_blank">Managing contract information</a>. </li> <li> Add package items to define individual service costs. <br />For more information, see <a href="/packages/manage-package-items" target="_blank">Managing package items</a>. </li> <li> Add CommsProfile details. <br />For more information, see <a href="/packages/manage-comms-profile" target="_blank">Managing the package communications profile</a>. </li> <li> Associate at least one portfolio with the package. <br />For more information, see <a href="/packages/manage-package-portfolios" target="_blank">Managing the package portfolios</a>. </li> </ol>

<h2 id="Updatingapackage">Updating a package</h2>

<p>You can only update packages if you have the permission to do so.</p>

<p>To change the details of an existing package:</p>

<ol> <li> On the <span>Packages</span> page, in the Packages list, select the <span>ID</span> of the package you want to update. </li> <li> Select <span>Edit</span> alongside the information you want to update. <span><strong>Tip: </strong>You can update the Contract details, associated Package Items, and the CommsProfile.</span> </li> <li>Make your changes.</li> <li> Select <span>Save</span> to apply the changes. <br />If the package is updated successfully, a green notification appears: "The package has been updated". <br />If you encounter errors, contact Support. </li> </ol>

<h2 id="Changingapackagestatus">Changing a package status</h2>

<p>To change a package status:</p>

<ol> <li> View the specific package with the status you want to change. <br />For more information, see <a href="#Viewing">Viewing a specific package</a>. </li> <li> On the <span>Contract</span> tab, alongside <span>Status</span>, select <img src="/images/resources/images/infinity/iconedit_24x24.png" alt="Iconedit 24X24" />. <br />A dialog appears, detailing the package name. <br /><img src="/images/resources/images/integra/edit-package status.png" alt="Edit Package Status" />  </li> <li> Select the new status. <span><strong>Tip: </strong>Select <span>Active</span> to activate the package.</span> </li> <li> Select <span>Update</span> to apply the new status to the package. <span><strong>Note: </strong>If the package is updated successfully, a green notification appears: "The package has been updated". If you encounter errors, contact Support.</span> </li> </ol>

<h2 id="Preventacustomerfromusingapackage">Prevent a customer from using a package</h2>

<p>To prevent a customer from accessing a package: </p>

<ol> <li>  On the <span>Package</span> &gt; <span>Portfolios</span> tab, select <span>Edit</span> alongside the relevant portfolio. </li> <li> Select the <span>AdminSuspended</span> checkbox. <br /><img src="/images/resources/images/integra/status-portfolio.png" alt="Status Portfolio" />  </li> <li> Select <span>Update</span> to suspend the portfolio from using the package. </li> </ol>

<h2 id="Allowacustomertouseapackage">Allow a customer to use a package</h2>

<p>To allow a customer to use a package: </p>

<ol> <li>  On the <span>Package</span> &gt; <span>Portfolios</span> tab, select <span>Edit</span> alongside the relevant portfolio. </li> <li> Clear the <span>AdminSuspended</span> checkbox. </li> <li> Select <span>Update</span> to unsuspend the portfolio and enable the customer to use the package. </li> </ol>
