<p>A package item is a specific charge listed on an invoice as part of a service package, like data usage, voice calls, or service fees. It details individual costs, making it clear what each charge is for. Together, the package items form part of a complete package, helping customers see the breakdown of their SIM estate costs.</p>

<p>The package items define how much is charged for an event that takes place on the network on any SIMs that are associated with the current package. For example, you can configure how much is charged for Mobile Originated SMS for SIMs that are registered on a network in a particular country.</p>

<Note>If a SIM event occurs that is not defined by the listed package items, a default set of values is used instead to process the cost of that event. For more information, see <a href="/packages/manage-packages" target="_blank">About managing packages using Integra</a>, where <span>defaultPackageId</span> defines which package is the default.</Note>

<h2 id="Viewingpackageitemsforaspecificpackage">Viewing package items for a specific package </h2>

<p>To view package items for a specific package:</p>

<ol> <li> Using Integra, on the left menu, select <span>Packages</span>. <br /> <img src="/images/resources/images/integra/packages-icon_80x78.png" alt="Packages Icon 80X78" />  </li> <li> On the <span>Packages</span> page, in the Packages list, select the package <span>ID</span> where you want to view, add, update or delete package items. </li> <li> Select the <span>Items</span> tab. <br /><img src="/images/resources/images/integra/items-package.png" alt="Items Package" />  <br />The package items appear as separate cards in a list. </li> </ol>

<h2 id="Creatingapackageitem">Creating a package item</h2>

<p>To create a new package item for a specific package:</p>

<ol> <li>View the package <span>Items</span> tab.</li> <li> Select <span>Add</span>. <br /><img src="/images/resources/images/integra/createpackageitem.png" alt="Createpackageitem" />  <br />The New Package Item dialog opens. </li> <li> Enter the following information, where &#42; indicates a mandatory parameter: <table> <col /> <col /> <thead> <tr> <th>Field</th> <th>Description</th> </tr> </thead> <tbody> <tr> <td><span>Name</span>&#42;</td> <td> The package item name, which contains details about the service.  </td> </tr> <tr> <td><span>Title</span>&#42;</td> <td> The short unique package item title (no spaces) for identifying the package item. </td> </tr> <tr> <td><span>Description</span> </td> <td> Extra notes about the package item for internal use only, such as pooling details or customer-related information. For example:  <ul> <li>For Brazil SIMs only.</li> <li>Support charge per hour.</li> <li>Brazil and USA 1 SMS pooled. Must be 12 digits for routing in Brazil.</li> </ul> </td> </tr> <tr> <td><span>Package Item Category</span>&#42;</td> <td> The package item category name. A package item category may describe a service item, API, tariff, or some form of rebate or discount.  </td> </tr> <tr> <td><span>Package Category</span>&#42;</td> <td>  The package category name. A package category may describe a physical hardware item, delivery charge, or a service item or tariff. <span><strong>Tip: </strong>You can only use a specific package category on one package item per package.</span> </td> </tr> <tr> <td><span>Transmission Type</span> </td> <td> The unique event that this package item defines (such as MO SMS). </td> </tr> <tr> <td><span>Transmission Type Group</span>&#42;</td> <td> The group that the transmission type belongs to (such as SMS services). </td> </tr> <tr> <td><span>Price</span> </td> <td>The periodic service charge per SIM, including service charge and bundled usage charges.<span><strong>Tip: </strong>The default value is set to <span>1</span>.</span></td> </tr> <tr> <td><span>Bundle Amount</span> </td> <td> The package item in-bundle amount, which is the number of bytes, SMS units, or seconds per SIM that are included in the package item price. The selected package item transmission type, such as Data or SMS, determines what is measured. For example, a bundleAmount of `60` is either: <ul> <li>60 bytes (Data)</li> <li>60 units (SMS)</li> <li>60 seconds (Voice)</li> </ul> </td> </tr> <tr> <td><span>Bundled Rounding</span> </td> <td> The in-bundle rounding amount. Each in-bundle data session is rounded up to the nearest specified amount. Depending on the package item transmission type, the default is either: <ul> <li>Data – 1024 (bytes)</li> <li>SMS – 1 (unit)</li> <li>Voice – 60 (seconds)</li> </ul> </td> </tr> <tr> <td><span>Out-of-bundle Price</span> </td> <td> The usage price charged for the event, when the agreed bundled usage is exceeded. </td> </tr> </tbody> </table> </li> <li> Select either <span>Yes</span> or <span>No</span> for the following preconfigured attribute: <table> <col /> <col /> <thead> <tr> <th>Field</th> <th>Description</th> </tr> </thead> <tbody> <tr> <td><span>doNotpool</span> </td> <td> Indicates whether this package item should not be included in the package pooling, where a set amount of service usage is shared across multiple SIM bundles. Either: <ul> <li><span>yes</span> – Do not include this package item in the package pooling.</li> <li><span>no</span> – Include this package item in the package pooling.</li> </ul> <br />For more information, see <a href="/billing/billing-pooling">Understanding pooling</a>. </td> </tr> </tbody> </table> </li> <li> Select a checkbox alongside one or more <span>Countries</span>, to define the region where this package item applies. </li> <li> Select <span>Save</span> to commit the Package Item to the package. </li> </ol>

<h2 id="Updatingapackageitem">Updating a package item</h2>

<p>To update a package item:</p>

<ol> <li>View the package <span>Items</span> tab.</li> <li> On the package item card you want to update, select <span>Edit</span>. <br /><img src="/images/resources/images/integra/editpackageitem.png" alt="Editpackageitem" />  <br />The package item opens. </li> <li> Make your changes. <br />For information about each field, see <a href="#Creating">Creating a package item</a>. </li> <li> Select <span>Update</span> to commit your changes to the package item. <br />If the package item is updated successfully, a green notification appears: "The package item has been updated". <br />If you encounter errors, contact Support. </li> </ol>

<h2 id="Deletingapackageitem">Deleting a package item</h2>

<p>To delete a package item:</p>

<ol> <li>View the package <span>Items</span> tab.</li> <li> On the package item card you want to delete, select <span>Delete</span>. <br /><img src="/images/resources/images/integra/deletepackageitem.png" alt="Deletepackageitem" />  <br />A confirmation box appears. </li> <li> Select <span>Delete</span> to remove the package item from the package. </li> </ol>
