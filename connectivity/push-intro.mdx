<p>Using the <span>Push API</span>, you can stream live raw network data from various sources to a web-based endpoint, for the purposes of monitoring and administering network usage. </p>

<p>You can subscribe to one or more of the following network data stream services:</p>

<ul> <li>NetFlow Raw Push</li> <li>RADIUS Authentication</li> <li>RADIUS Accounting</li> </ul>

<Accordion title="About NetFlow Raw Push">
 <p>NetFlow is a networking protocol that collects active IP network traffic data as it flows in or out of an interface, such as an IoT device. The collected data includes:</p> <ul> <li>Traffic source and destination IP addresses and port numbers</li> <li>The number of bytes and IP packets observed in the traffic flow</li> </ul> <p>Eseye captures real-time NetFlow data across all its data centres, for every device. This high volume data is processed through Eseye's cloud infrastructure using Amazon Kinesis, then enriched with valid SIM ID and IMSI information. Eseye presents NetFlow data to subscribed customers in a constant, aggregated stream via the Push API. Each NetFlow payload that is sent to a customer is linked to a specific SIM in the customer's Infinity Classic account. For information about receiving NetFlow messages, see the <a href="https://docs.eseye.com/Content/Resources/Files/8725-Push-API-Developer-Guide.pdf">8725 Push API Developer Guide (PDF)</a>.</p> <p>Network administrators can analyse the data to understand the traffic flow and volume across their SIM estate network, which may inform business decisions. For example, a business may need to purchase new hardware in areas of high usage to ease traffic flow congestion, or retire a thing that is never used.</p> 
</Accordion>

<Accordion title="About RADIUS Authentication">
 <p>RADIUS authentication controls a user's network resource access. Users are first verified and then authorized to access specific resources. </p> <p>The user requests access from the Network Access Server (NAS), which in turn sends a RADIUS <span>Access-Request</span> to the RADIUS server. The Eseye RADIUS server returns either of the following responses to the NAS:</p> <ul> <li> <span>Access-Accept</span>  <br />The user is granted access to network resources. After authenticating the user, the RADIUS server regularly checks that the user is authorized to use a requested network service. For example, a user may have access to the wireless network, but not the VPN service. This information is either stored locally on the RADIUS server, or in an external source, such as LDAP or Active Directory. </li> <li> <span>Access-Reject</span>  <br />The user is denied access to network resources. Reasons may include:  <ul> <li>Failure to provide proof of identification</li> <li>Unknown or inactive user account</li> </ul> </li> </ul> <p>Eseye captures real-time RADIUS authentication data across all its data centres, for every device. This high volume data is processed through Eseye's cloud infrastructure using Amazon Kinesis, then enriched with valid SIM ID and IMSI information. Eseye presents this data to subscribed customers in a constant, aggregated stream via the Push API. Each RADIUS authentication payload that is sent to a customer is linked to a specific SIM in the customer's Infinity Classic account. For information about receiving RADIUS authentication messages, see the <a href="https://docs.eseye.com/Content/Resources/Files/8725-Push-API-Developer-Guide.pdf">8725 Push API Developer Guide (PDF)</a>.</p> <p>Customers can use the data to track network and device usage, and analyse networking issues. For example, they can track which devices are disconnected from the network for a long period of time, or how often a device authenticates.</p> 
</Accordion>

<Accordion title="About RADIUS Accounting">
 <p>RADIUS accounting collects data that accurately measures a RADIUS network user's start and stop times and traffic usage. This information is used for billing purposes and general network monitoring.</p> <p>RADIUS accounting has the following flow:</p> <table> <col /> <col /> <thead> <tr> <th>Accounting type</th> <th>Description</th> </tr> </thead> <tbody> <tr> <td><span>Accounting-Start</span> </td> <td>The RADIUS accounting session starts after the user is authorised to access the RADIUS server. At this point, the Network Access Server (NAS) sends a RADIUS Accounting Request packet to the RADIUS server. This packet signifies that the user's session on the network has begun, and normally contains the user ID, point of access, and network address, as well as a unique session identifier.</td> </tr> <tr> <td><span>Accounting-Interim</span> </td> <td>Depending on the session length, the NAS may also periodically send one or more <span>Accounting-Interim</span> updates, which contain the session duration and current data usage. The first update occurs 30 minutes after <span>Accounting-Start</span>.</td> </tr> <tr> <td><span>Accounting-Stop</span> </td> <td> The RADIUS accounting session terminates when the SIM disconnects from the network. Reasons for termination may include:  <ul> <li>The user manually signs off</li> <li>The user is denied access because of role-level restrictions</li> <li>An error occurs on the network, such as a port error</li> <li>Session time out occurs because of inactivity, or the maximum session length is exceeded</li> <li>An administrator manually stops the session</li> </ul> </td> </tr> </tbody> </table> <p>Eseye captures real-time RADIUS accounting data across all its data centres, for every device. This high volume data is processed through Eseye's cloud infrastructure using Amazon Kinesis, then enriched with valid SIM ID and IMSI information. Eseye presents this data to subscribed customers in a constant, aggregated stream via the Push API. Each RADIUS accounting payload that is sent to a customer is linked to a specific SIM in the customer's Infinity Classic account. For information about receiving RADIUS accounting messages, see the <a href="https://docs.eseye.com/Content/Resources/Files/8725-Push-API-Developer-Guide.pdf">8725 Push API Developer Guide (PDF)</a>.</p> <p>Customers can use the data to generate network usage and billing reports.</p> 
</Accordion>
