<h2 id="AbouttheAnyNetsolution">About the AnyNet solution</h2>

<p>The AnyNet solution provides access to mobile networks across the world. Devices connect on whichever mobile network provides the best connectivity for their location and requirements. The AnyNet Connectivity Management Platform (CMP) can use <a href="/getting-started/roaming#Steering" target="_blank">steering of roaming</a> and <a href="/getting-started/multi-imsi#ImsiSwitching" target="_blank">IMSI switching</a> to connect to different mobile networks in order to maintain optimum connectivity between a device and the customer network.</p>

<h2 id="Understandingdatamovement">Understanding data movement</h2>

<p>Data needs a direction of flow, and as such it needs to know where it has come from (the source) and where it is going to (the destination). Direction of flow is determined on a device level (for example, the device needs to know which mobile network it can connect to) and network level (for example, the mobile network needs to know which TCP/IP network to connect with).</p>

<h2 id="Configuringadeviceforthemobilenetwork">Configuring a device for the mobile network</h2>

<ul> <li>Each device connects to the mobile network through a modem across a radio access network. For more information, see <a href="https://en.wikipedia.org/wiki/Radio_access_network" target="_blank">About Radio Access Networks</a>.</li> <li>Devices use SIMs to connect to a network. For more information, see <a href="/hardware-products/sims-intro" target="_blank">AnyNet SIMs</a>.</li> <li>Devices can use IMSI technology to ensure optimum connectivity. For more information, see <a href="/getting-started/multi-imsi" target="_blank">Understanding multi-IMSI functionality</a>.</li> </ul>

<h2 id="Configuringadevicetoaccessthecorrectnetwork">Configuring a device to access the correct network</h2>

<p>Before an IoT device can send data, it must first get permission to use a mobile network.</p>

<p>All SIMs contain unique numbers that the network uses to identify the SIM, the subscriber, and the home network. The modem also contains a unique identifier.</p>

<p>Every customer device must also contain the Eseye network Access Point Name (<a href="/glossary/apn">APN</a>) details. The APN that the customer must use is detailed in the customer contract. For more information, see <a href="/connectivity/anynet-apn-list" target="_blank">Current AnyNet APN list</a>.</p>

<p>When the device requests to connect to a mobile network, the network operator uses the SIM and device unique identifiers, as well as the APN details, to authenticate the device via a Packet Gateway (PGW). </p>

<Note>For the sake of simplicity, the image below depicts the Packet Gateway (PGW) between the Eseye and MNO networks. However, the PGW will actually exist within either network, depending on the agreed configuration between Eseye and the MNO. </Note>

<p><img src="/images/resources/images/connectivity/networkauth.png" alt="Networkauth" /> </p>

<p>The current IMSI determines which mobile network the device uses. This means that data sent from a device may ingress the Eseye network at different PoPs, depending on the current IMSI (and therefore the MNO). For more information, see <a href="/connectivity/understanding-pops">About Eseye PoPs</a>.</p>

<p><img src="/images/resources/images/connectivity/popconnection.png" alt="Popconnection" /> </p>

<p>If the device is roaming, the data will ingress the Eseye network using the home mobile network. For more information, see <a href="/getting-started/roaming" target="_blank">Understanding roaming</a>.</p>
