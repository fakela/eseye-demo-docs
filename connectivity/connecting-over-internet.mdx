<p>This section of topics unpacks the third part of the data journey between a device and the customer network, where data egresses the MPLS network to the internet, and into the customer network or third party network (such as a cloud).</p>

<p><img src="/images/resources/images/connectivity/networkoverviewsection3.png" alt="Networkoverviewsection3" /> </p>

<p>For an overview on Connectivity, see <a href="/connectivity/connectivity-intro" target="_blank">Eseye connectivity overview</a>.</p>

<h2 id="Aboutsendingdatasecurelyovertheinternet">About sending data securely over the internet</h2>

<p>Eseye works with each customer to configure secure methods for sending device data over the internet to the customer network or third party network (such as a cloud).</p>

<p>This includes:</p>

<ul> <li>Using Network Address Translation (NAT). For more information, see <a href="/connectivity/nat-overview">About Network Address Translation (NAT)</a>.</li> <li>Configuring VPN where required. For more information, see <a href="/connectivity/anynet-vpns#Configur">Configuring AnyNet VPNs</a>.</li> <li>Using an Access Control List (ACL) to restrict internet access. For more information, see <a href="/connectivity/non-vpn-network-traffic">Routing non-VPN network traffic</a>.</li> </ul>

<Note>Customers may also request that the data egresses onto the internet for onward routing to its destination, with no restrictions. Data can bypass the Eseye MPLS network altogether. </Note>

<Tip>Speak to your Account Manager to help Eseye understand what your company needs, and to provide a bespoke solution to deliver device data securely and quickly.</Tip>
