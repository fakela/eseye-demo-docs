<p>Do not create thing s until you have verified the advisory and alert email addresses that you supplied during AnyNet IRIS configuration.For more information, see <a href="/iris/installing-iris#Emails">Configuring AnyNet IRIS</a>. Receiving a verification link may take up to 30 minutes. If you have not received an email, contact Support: <a href="mailto:support@eseye.com">support@eseye.com</a>.</p>

<div> <h3>Before you begin</h3> <ul> <li> Ensure that the AnyNet Secure Cellular Connectivity configuration process is complete on your AWS account. </li> <li> Eseye connects your thing to AWS over a cellular network. To purchase the requisite SIM cards, search for <a href="https://www.amazon.com/s?k=anynet+sim&ref=nb_sb_noss">AnyNet SIM on Amazon.com</a>. </li> <li> You need a AnyNet Secure SIM number, which is the unique serial number printed on the back of the SIM card. <br /><img src="/images/resources/images/sim_iccid_qsg.jpg" alt="Sim Iccid Qsg" />  </li> </ul> </div>

<p>To create a thing in AWS IoT Core:</p>

<ol> <li> Sign in to the AWS Management Console: <a href="http://aws.amazon.com/console">aws.amazon.com/console</a> </li> <li> In the <span>AWS Services</span> section, find the <span>IoT Core</span> service. <div> AWS has updated the AWS IoT Console. To use the original console that is in line with these instructions, in the bottom left hand corner of the AWS IoT Console browser window is the <span>New console experience</span> switch. Slide this switch off. <br /><img src="/images/resources/images/awsnewconsoleexperience.png" alt="Awsnewconsoleexperience" />  </div> </li> <li> If this is your first time to sign into the AWS Management Console, select <span>Get Started</span>. </li> <li> In the left-hand <span>AWS IoT</span> menu, select <span>Manage</span> &gt; <span>Things</span>. <br /><img src="/images/resources/images/managethings.png" alt="Managethings" />  </li> </ol>

<ol> <li> In the top right corner, select the correct AWS<span>Region</span> you want to use. <br /><img src="/images/resources/images/region.png" alt="Region" />  </li> <li> If this is your first time creating thing s, select <span>Register a thing</span>, otherwise select <span>Create</span>. <br />The Creating AWS IoT things page appears. </li> <li> Select <span>Create a single thing</span>. <br />The Add your device to the thing registry page appears. </li> <li> Type a <span>Name</span> for your thing. <span><strong>Note: </strong>The AWS thingName maximum length is 128 characters, and can contain the following alphanumeric and special letters: [a-zA-Z0-9:&#95;-]+</span> </li> <li> In the <span>Thing Type</span> drop-down box, select <span>AnyNetThingType</span>. <br /><img src="/images/resources/images/anynetthingtype.png" alt="Anynetthingtype" />  </li> <li> If required, add the thing to a group. </li> <li> In the <span>Set searchable thing attributes</span> section, leave the <span>ActionRequest Value</span> field blank. </li> <li>Leave the <span>PolicySelector Value</span> field blank.</li> <li> In the <span>SimId Value</span> field, type the SIM number (described above). <br /><img src="/images/resources/images/simid.png" alt="Simid" />  </li> <li> Select <span>Next</span>. <br />The Add a certificate for your thing page appears. Eseye manages the certificate process so you don't have to. </li> <li> Select <span>Create thing without certificate</span>. <span><strong>Note: </strong>When you use AWS IoT in your AWS account for the first time, an AWS error may appear because Device Gateway endpoint provisioning is not complete on your account. If this occurs, allow AWS services 5-10 minutes to complete, and try again.</span> <br />A Successfully created thing message appears, and the new thing appears on the Things page </li> </ol>
