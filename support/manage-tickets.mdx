<p>You can log Support tickets for several reasons, primarily to report technical issues, track issues, seek help, or request services.</p>

<p>To access <span>Tickets</span>:</p>

<ul> <li> Using Infinity, on the left menu, select <span>Tickets</span> to display the <span>Support Tickets</span> page. <br /> <img src="/images/resources/images/infinity/ticketsicon_80x78.png" alt="Ticketsicon 80X78" />  <br />All <span>Open</span> tickets for the current user are listed. <br /><img src="/images/resources/images/infinity/supportticketspage.png" alt="Supportticketspage" />  </li> </ul>

<h2 id="Viewingtickets">Viewing tickets</h2>

<p>The <span>Support Tickets</span> list contains the following fields:</p>

<table> <col /> <col /> <thead> <tr> <th>Field</th> <th>Description</th> </tr> </thead> <tbody> <tr> <td><span>Ticket no.</span> </td> <td>The Zendesk ticket number.<br />Select to display the <span>Support ticket details</span> for your ticket. </td> </tr> <tr> <td><span>Date created</span> </td> <td>The ticket start date.</td> </tr> <tr> <td><span>Title</span> </td> <td>Brief summary of the issue.</td> </tr> <tr> <td><span>Status</span> </td> <td> Support assigns the following statuses to the ticket: <ul> <li><span>New</span>: An initial response email sent as acknowledgement to the customer.</li> <li><span>Open</span>: Support are currently working on the ticket.</li> <li><span>Pending</span>: Waiting for a response from the customer. </li> <li><span>On-Hold</span>: Waiting for an action from a third party or internal team.</li> <li><span>Solved</span>: Resolved by the customer support team with no further action required. </li> <li><span>Closed</span>: Solved tickets are <span>Closed</span> after 7 days. To open a closed ticket, a follow-up ticket is created by the Support.</li> </ul> </td> </tr> <tr> <td><span>Priority</span> </td> <td> Support triages the ticket based on the following priorities: <ul> <li><span>Low/ P4</span>: Less impact on business, or informational request.</li> <li> <span>Normal/ P3</span>: Minor impact to service affecting a small proportion of your estate, while other functions can operate normally. <span><strong>Tip: </strong>By default, <span>Normal</span> is assigned to the new ticket.</span> </li> <li><span>High/ P2</span>: Major impact to service affecting a large proportion of your estate, but not a complete outage.</li> <li><span>Critical/ P1</span>: Severe impact affecting your entire estate, for example. a complete outage of service or a single critical system down.</li> </ul> <br />To escalate tickets categorised as P1/P2 incident, phone Support. </td> </tr> <tr> <td><span>Raised by</span> </td> <td>The Infinity user that created the ticket.</td> </tr> </tbody> </table>

<h3>Filtering the displayed tickets</h3>

<p>To display tickets raised within a specified time period:</p>

<ol> <li>On the <span>Support Tickets</span> page, use any of the following filters to constrain the available list and find a specific ticket: <ul><li><span>Start Date</span> – date when the tickets were first raised (initial search range date)</li><li><span>End Date</span> – date before which, the tickets were first raised (final search range date)</li><li><span>Status</span> – the current ticket status<span><strong>Note: </strong>By default, the <span>Status</span> value is <span>Open</span>. For more information, see <a href="#Viewing">Viewing tickets</a>.</span></li></ul></li> <li> Select <span>Filter</span> to display all tickets matching the specified parameters. </li> <li>Select <span>Reset</span> to clear the values and display only <span>Open</span> tickets. </li> </ol>

<h3>Refreshing the list of tickets</h3>

<p>To refresh the tickets list:</p>

<ul> <li>On the <span>Support Tickets</span> page, select <span>Actions</span> &gt; <span>Refresh the Tickets</span>.</li> </ul>

<h2 id="Creatingaticket">Creating a ticket</h2>

<p>To raise a ticket:</p>

<ol> <li>On the left menu, select <span>Tickets</span>.</li> <li>On the <span>Support Tickets</span> page, select <span>Actions</span> &gt; <span>Create a ticket</span> to display the <span>Create a Ticket</span> dialog.<br /><img src="/images/resources/images/infinity/createaticketdialog.png" alt="Createaticketdialog" /></li> <li>Specify a <span>Title</span> and a detailed <span>Message</span> explaining the issue.</li> <li> Select <span>Ticket Type</span> as: <ul> <li><span>Incident</span> – If there is an outage or network issue</li> <li><span>Problem</span> – If there is an issue with the SIM or device</li> <li><span>Question</span> – If you have any doubt or query that requires clarification</li> <li><span>Task</span> – If you are unable to perform any procedure using the available menus</li> </ul> </li> <li>Select <span>Save</span> to log the issue as a ticket.</li> </ol>

<h2 id="Updatingtickets">Updating tickets</h2>

<p> Use the <span>Support Ticket Details</span> page to:</p>

<ul> <li>Add comments and attach files to the raised ticket. </li> <li>View Support updates.</li> <li> Close the ticket <span><strong>Note: </strong> Depending on their role and permissions, some customers can close their tickets.</span> </li> <li>Return to the <span>Support tickets</span> page</li> </ul>

<p><img src="/images/resources/images/infinity/supportticketsdetailspage.png" alt="Supportticketsdetailspage" /> </p>

<p>To update an open ticket:</p>

<ol> <li>On the left menu, select <span>Tickets</span> to list all the tickets raised that match the filter parameters.</li> <li>In the ticket list, select the ticket number to display its details.</li> <li> To update the ticket, select <span>Add Comment</span> to display the <span>Add Comment</span> dialog. <br /><img src="/images/resources/images/infinity/addcomment.png" alt="Addcomment" />  </li> <li>In the <span>New Comment</span> text box, describe the update.</li> <li> To attach files or screenshots as additional information, select <span>BROWSE</span> to navigate to the file location, select it and select <span>Add</span>.  <span><strong>Tip: </strong>Attachments assist the Support to replicate the issue for quicker resolution.</span> </li> <li> Below <span>Updates</span>, you can view a list of additional comments about the support ticket. Each update includes the: <ul> <li><span>Updates</span> creation date and time</li> <li>Email address of the user that added the update</li> <li> <span>Updates</span> description text</li> <li>Attachment(if any) in.png or.jpeg formats</li> </ul> </li> <li> If you are satisfied with the resolution, select <span>Close Ticket</span>. <span><strong>Note: </strong>If your ticket status is updated by Support, you will receive an email notification.</span> </li> </ol>
