<h2 id="Viewingalluserpermissionsforaspecificuser">Viewing all user permissions for a specific user</h2>

<p>To view all users:</p>

<ul> <li> Call <span>GET /api/v2/portfolio/&#123;<var>portfolioId</var>&#125;/user/&#123;<var>userId</var>&#125;/userPermission/</span>, where &#123;<var>portfolioId</var>&#125; identifies the parent portfolio, and &#123;<var>userId</var>&#125; identifies the user whose user permissions you want to view. <span><strong>Tip: </strong>You can apply filters in the request path to limit the results, using one or more parameters. The API returns exactly matching records only. Alternatively, remove all filters from the path to return all records. For example, adding <span><span>?status=active</span></span> to the end of the path will return <span>active</span> objects only.</span> <br />The API returns all the user permissions. </li> </ul>

<h4>Example</h4>

<p><span>GET https://portfolio.api.anynetiot.com/api/v2/portfolio/a628ebc659f7dac01f9fdcbd1116307a/user/8d9cde45a21f3f6cb52a368e7fc70b9c/userPermission/</span> </p>

<p>or</p>

<p><span>GET https://portfolio.api.anynetiot.com/api/v2/portfolio/a628ebc659f7dac01f9fdcbd1116307a/user/8d9cde45a21f3f6cb52a368e7fc70b9c/userPermission/?status=active</span> </p>

<h3>Response schema application/json: </h3>

<p><span>userPermissionId</span>; <span>name</span>; <span>title</span>; <span>permissionId</span>; <span>userId</span>; <span>portfolioId</span>; <span>status</span></p>

<Note>For more information, see <a href="#PackageP" target="_blank">About user permission endpoints</a>.</Note>

<h2 id="Viewingaspecificuserpermission">Viewing a specific user permission</h2>

<p>To view details for a specific user permission:</p>

<ul> <li> Call <span>GET /api/v2/portfolio/&#123;<var>portfolioId</var>&#125;/user/&#123;<var>userId</var>&#125;/userPermission/&#123;<var>userPermissionId</var>&#125;</span> <br /> where &#123;<var>portfolioId</var>&#125; identifies the parent portfolio, &#123;<var>userId</var>&#125; identifies the user, and &#123;<var>userPermissionId</var>&#125; identifies which user permission you want to view. <br /><span>Path parameters:</span> <span>portfolioId</span><b>&#42;</b>; <span>userId</span><b>&#42;</b>; <span>userPermissionId</span><b>&#42;</b> <span><strong>Note: </strong>JSON keys marked <b>&#42;</b> are mandatory.</span> <br />The API returns the specified user permission details. </li> </ul>

<h4>Example</h4>

<p><span>GET https://portfolio.api.anynetiot.com/api/v2/portfolio/a628ebc659f7dac01f9fdcbd1116307a/user/8d9cde45a21f3f6cb52a368e7fc70b9c/userPermission/748627</span> </p>

<p><span>userPermissionId</span>; <span>name</span>; <span>title</span>; <span>permissionId</span>; <span>userId</span>; <span>portfolioId</span>; <span>status</span></p>

<h2 id="Creatinganewuserpermission">Creating a new user permission</h2>

<p>To create a new user permission:</p>

<ul> <li> Call <span>POST /api/v2/portfolio/&#123;<var>portfolioId</var>&#125;/user/&#123;<var>userId</var>&#125;/userPermission/</span> <br /> where &#123;<var>portfolioId</var>&#125; identifies the parent portfolio and <var>userId</var> identifies the user who you want to grant a new user permission. <br /><span>Path parameters:</span> <span>portfolioId</span><b>&#42;</b>; <span>userId</span><b>&#42;</b> <br /><span>Body parameters:</span> <span>name</span><b>&#42;</b>; <span>title</span><b>&#42;</b>; <span>permissionId</span><b>&#42;</b> <span><strong>Note: </strong> JSON keys marked <b>&#42;</b> are mandatory. <br />Do not use <span>null</span> or <span>""</span> (empty) for mandatory parameters. <br />The <span>permissionId</span> must already exist in the system. </span> <br />The API creates a new user permission. </li> </ul>

<h2 id="Updatingaspecificuserpermission">Updating a specific user permission</h2>

<p>To update a specific user permission:</p>

<ul> <li> Call <span>PATCH /api/v2/portfolio/&#123;<var>portfolioId</var>&#125;/user/&#123;<var>userId</var>&#125;/userPermission/&#123;<var>userPermissionId</var>&#125;/</span>, where &#123;<var>portfolioId</var>&#125; identifies the parent portfolio, &#123;<var>userId</var>&#125; identifies the user with the user permission, and &#123;<var>userPermissionId</var>&#125; identifies which user permission you want to update. <br /><span>Path parameters:</span> <span>portfolioId</span><b>&#42;</b>; <span>userId</span><b>&#42;</b>; <span>userPermissionId</span><b>&#42;</b>;  <br /><span>Body:</span> <span>name</span>; <span>title</span>; <span>permissionId</span>; <span>status</span> <span><strong>Note: </strong> JSON keys marked <b>&#42;</b> are mandatory. <br />Do not use <span>null</span> or <span>""</span> (empty) for mandatory fields. <br />The <span>permissionId</span> must already exist in the system. <br />If required, you can update more than one optional parameter. </span> <br />The API updates the specified user permission parameters with the new details. </li> </ul>

<h4>Example</h4>

<p>PATCH https://portfolio.api.anynetiot.com/api/v2/portfolio/a628ebc659f7dac01f9fdcbd1116307a/user/8d9cde45a21f3f6cb52a368e7fc70b9c/userPermission/748627</p>

<p>/&#42; header information &#42;/</p>

<p>&#123;</p>

<p>"status": "active"</p>
