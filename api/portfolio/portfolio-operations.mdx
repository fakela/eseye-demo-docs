<p>The Portfolio API enables you to: </p>

<ul> <li>View details for <a href="#Viewing2">single</a> or <a href="#Viewing">multiple</a> portfolios</li> <li><a href="#Creating">Create</a>, <a href="#Updating">update</a>, and <a href="#Deleting">delete</a> portfolios</li> </ul>

<p>You can also:</p>

<ul> <li>Create, update, delete and view <a href="#Viewing" target="_blank">portfolio attributes</a>.</li> <li>Create, update, delete and view <a href="/api/portfolio/portfolio-address-operations#Viewing" target="_blank">portfolio addresses</a>.</li> <li>Create, update, delete and view <a href="/api/portfolio/portfolio-address-operations#Viewing" target="_blank">portfolio address attributes</a>.</li> </ul>

<h2 id="Authentication">Authentication</h2>

<ul> <li>User required to login</li> <li>Bearer token required</li> </ul>

<Note>For more information, see <a href="/api/portfolio/portfolio-api-intro" target="_blank">Portfolio API Introduction</a>.</Note>

<h2 id="Viewingallportfolios">Viewing all portfolios</h2>

<p>To view all portfolios:</p>

<ul> <li> Call <span>GET /api/v2/portfolio/</span> <span><strong>Tip: </strong>You can apply filters in the request path to limit the results, using one or more parameters. The API returns exactly matching records only. Alternatively, remove all filters from the path to return all records. For example, adding <span><span>?status=active</span></span> to the end of the path will return <span>active</span> objects only.</span> <br />The API returns all the portfolios. </li> </ul>

<h4>Example</h4>

<p><span>GET https://portfolio.api.anynetiot.com/api/v2/portfolio/</span> </p>

<p>or</p>

<p><span>GET https://portfolio.api.anynetiot.com/api/v2/portfolio?subnetRangeType=Sim4VPN;status=active</span> – this will return all portfolios with a <span>Sim4VPN</span> subnetRangetype and <span>active</span> status.</p>

<h2 id="Viewingaspecificportfolio">Viewing a specific portfolio</h2>

<p>To view details for a specific portfolio:</p>

<ul> <li> Call <span>GET /api/v2/portfolio/&#123;<var>portfolioId</var>&#125;</span>, where &#123;<var>portfolioId</var>&#125; identifies which portfolio you want to view. <br /><span>Path parameters:</span> <span>portfolioId</span><b>&#42;</b> <span><strong>Note: </strong>JSON keys marked <b>&#42;</b> are mandatory.</span> <br />The API returns the specified portfolio details. </li> </ul>

<h4>Example</h4>

<p><span>GET https://portfolio.api.anynetiot.com/api/v2/portfolio/a628ebc659f7dac01f9fdcbd1116307a</span> </p>

<h2 id="Creatinganewportfolio">Creating a new portfolio</h2>

<p>To create a new portfolio:</p>

<ul> <li> Call <span>POST /api/v2/portfolio/</span>. <br /><span>Body parameters:</span> <span>name</span><b>&#42;</b>; <span>title</span><b>&#42;</b>; <span>businessCodeId</span><b>&#42;</b>; <span>taxCodeId</span><b>&#42;</b>; <span>currencyId</span><b>&#42;</b>; <span>paymentMethod</span><b>&#42;</b>; <span>invoicingEntityId</span><b>&#42;</b>; <span>billingDayOfMonth</span><b>&#42;</b>; <span>parentPortfolioId</span>; <span>portalId</span> <span><strong>Note: </strong> JSON keys marked <b>&#42;</b> are mandatory. <br />Do not use <span>null</span> or <span>""</span> (empty) for mandatory parameters. <br />You can use one or more optional parameters. </span> <br />The API creates a new portfolio. </li> </ul>

<Note>Before you can use the portfolio, you need to create an <span>active</span> billing portfolio address, and change the portfolio status to <span>active</span>. For more information, see <a href="/api/portfolio/portfolio-address-operations">About portfolio address endpoints</a> and <a href="/api/portfolio/portfolio-api-reference#Portfoli">Portfolio API reference tables</a>.</Note>

<h2 id="Updatingaspecificportfolio">Updating a specific portfolio</h2>

<p>To update a specific portfolio:</p>

<ul> <li> Call <span>PATCH /api/v2/portfolio/&#123;<var>portfolioId</var>&#125;</span>, where &#123;<var>portfolioId</var>&#125; identifies which portfolio you want to update. <br /><span>Path parameters:</span> <span>portfolioId</span><b>&#42;</b> <br /><span>Body:</span> <span>name</span>; <span>title</span>; <span>parentPortfolioId</span>; <span>portalId</span>; <span>businessCodeId</span>; <span>taxCodeId</span>; <span>currencyId</span>; <span>invoicingEntityId</span>; <span>paymentMethodId</span>; <span>billingDayOfMonth</span>; <span>status</span> <span><strong>Note: </strong> JSON keys marked <b>&#42;</b> are mandatory. <br />Do not use <span>null</span> or <span>""</span> (empty) for mandatory parameters. <br />If required, you can update more than one optional parameter. <br />If you are updating any of the following: <span>parentPortfolioId</span>, <span>portalId</span>, <span>businessCodeId</span>, <span>taxCodeId</span>, <span>currencyId</span>, <span>invoicingEntityId</span>, or <span>paymentMethodId</span> then that value must already exist in the system. </span> <br />The API updates the specified portfolio parameters with the new details. </li> </ul>

<h4>Example</h4>

<p>PATCH https://portfolio.api.anynetiot.com/api/v2/portfolio/13579</p>

<p>/&#42; header information &#42;/</p>

<p>&#123;</p>

<p>"status": "active"</p>
