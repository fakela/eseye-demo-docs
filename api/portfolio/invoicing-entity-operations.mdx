<p>The Portfolio API enables you to: </p>

<ul> <li>Create, update, delete and view <a href="/api/portfolio/invoicing-entity-attribute-operations#Viewing" target="_blank">invoicing entity attributes</a>.</li> </ul>

<h2 id="Viewingallinvoicingentities">Viewing all invoicing entities</h2>

<p>To view all invoicing entities:</p>

<ul> <li> Call <span>GET /api/v2/invoicingEntity/</span> <span><strong>Tip: </strong>You can apply filters in the request path to limit the results, using one or more parameters. The API returns exactly matching records only. Alternatively, remove all filters from the path to return all records. For example, adding <span><span>?status=active</span></span> to the end of the path will return <span>active</span> objects only.</span> <br />The API returns all the invoicing entities. </li> </ul>

<h4>Example</h4>

<p><span>GET https://portfolio.api.anynetiot.com/api/v2/invoicingEntity/</span> </p>

<p>or</p>

<p><span>GET https://portfolio.api.anynetiot.com/api/v2/invoicingEntity?countryCodeId=703;status=active</span> – this will return all active UK invoicing entities.</p>

<h2 id="Viewingaspecificinvoicingentity">Viewing a specific invoicing entity</h2>

<p>To view details for a specific invoicing entity:</p>

<ul> <li> Call <span>GET /api/v2/&#123;<var>invoicingEntityId</var>&#125;</span>, where &#123;<var>invoicingEntityId</var>&#125; identifies which invoicing entity you want to view. <br /><span>Path parameters:</span> <span>invoicingEntityId</span><b>&#42;</b> <span><strong>Note: </strong>JSON keys marked <b>&#42;</b> are mandatory.</span> <br />The API returns the specified invoicing entity details. </li> </ul>

<h4>Example</h4>

<p><span>GET https://portfolio.api.anynetiot.com/api/v2/invoicingEntity/80</span> </p>

<h3>Response schema application/json: </h3>

<p><span>invoicingEntityId</span>; <span>name</span>; <span>title</span>; <span>countryCodeId</span>; <span>invoiceNumber</span>; <span>grossBilling</span>; <span>portfolioId</span>; <span>status</span></p>

<Note>For more information, see <a href="/api/portfolio/portfolio-api-reference#Invoicin" target="_blank">Invoicing entity definition</a>.</Note>

<p>If an <span>invoicingEntity</span> contains attributes, these are returned in the <span>attributes</span> array. Each returned attribute includes:</p>

<p><span>invoicingEntityAttributeId</span>; <span>invoicingEntityId</span>; <span>attribute</span>; <span>value</span>; <span>status</span></p>

<Note>For more information, see <a href="/api/portfolio/portfolio-api-reference#Invoicin2" target="_blank">Invoicing entity attribute definition</a>.</Note>

<h2 id="Creatinganewinvoicingentity">Creating a new invoicing entity</h2>

<p>To create a new invoicing entity:</p>

<ul> <li> Call <span>POST /api/v2/invoicingEntity/</span>. <br /><span>Body parameters:</span> <span>name</span><b>&#42;</b>; <span>title</span><b>&#42;</b>; <span>grossBilling</span><b>&#42;</b>; <span>portfolioId</span><b>&#42;</b>; <span>countryCodeId</span>; <span>invoiceNumber</span> <span><strong>Note: </strong> JSON keys marked <b>&#42;</b> are mandatory. <br />Do not use <span>null</span> or <span>""</span> (empty) for mandatory parameters. <br />You can use one or more optional parameters. </span> <br />The API creates a new invoicing entity. </li> </ul>

<h2 id="Updatingaspecificinvoicingentity">Updating a specific invoicing entity</h2>

<p>To update a specific invoicing entity:</p>

<ul> <li> Call <span>PATCH /api/v2/invoicingEntity/&#123;<var>invoicingEntityId</var>&#125;</span>, where &#123;<var>invoicingEntityId</var>&#125; identifies which invoicing entity you want to update. <br /><span>Path parameters:</span> <span>invoicingEntityId</span><b>&#42;</b> <br /><span>Body:</span> <span>name</span>; <span>title</span>; <span>grossBilling</span>; <span>portfolioId</span>; <span>countryCodeId</span>; <span>invoiceNumber</span>; <span>status</span> <span><strong>Note: </strong> JSON keys marked <b>&#42;</b> are mandatory. <br />Do not use <span>null</span> or <span>""</span> (empty) for mandatory parameters. <br />If required, you can update more than one optional parameter. <br />If you are updating the following: <span>portfolioId</span>, <span>countryCodeId</span>, then that value must already exist in the system. </span> <br />The API updates the specified invoicing entity parameters with the new details. </li> </ul>

<h4>Example</h4>

<p>PATCH https://portfolio.api.anynetiot.com/api/v2/invoicingEntity/80</p>

<p>/&#42; header information &#42;/</p>

<p>&#123;</p>

<p>"status": "active"</p>
