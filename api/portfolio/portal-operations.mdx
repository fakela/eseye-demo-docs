<p>The Portfolio API enables you to: </p>

<ul> <li>View details for <a href="#Viewing2">single</a> or <a href="#Viewing">multiple</a> portals</li> <li><a href="#Creating">Create</a> and <a href="#Updating">update</a> portals</li> </ul>

<p>You can also:</p>

<ul> <li>Create, update, delete and view <a href="/api/portfolio/portal-attribute-operations#Viewing" target="_blank">portal attributes</a>.</li> </ul>

<h2 id="Viewingallportals">Viewing all portals</h2>

<p>To view all portals:</p>

<ul> <li> Call <span>GET /api/v2/portal/</span> <span><strong>Tip: </strong>You can apply filters in the request path to limit the results, using one or more parameters. The API returns exactly matching records only. Alternatively, remove all filters from the path to return all records. For example, adding <span><span>?status=active</span></span> to the end of the path will return <span>active</span> objects only.</span> <br />The API returns all the portals. </li> </ul>

<h4>Example</h4>

<p><span>GET https://portfolio.api.anynetiot.com/api/v2/portal/</span> </p>

<p>or</p>

<p><span>GET https://portfolio.api.anynetiot.com/api/v2/portal?status=active</span> </p>

<h2 id="Viewingaspecificportal">Viewing a specific portal</h2>

<p>To view details for a specific portal:</p>

<ul> <li> Call <span>GET /api/v2/&#123;<var>portalId</var>&#125;</span>, where &#123;<var>portalId</var>&#125; identifies which portal you want to view. <br /><span>Path parameters:</span> <span>portalId</span><b>&#42;</b> <span><strong>Note: </strong>JSON keys marked <b>&#42;</b> are mandatory.</span> <br />The API returns the specified portal details. </li> </ul>

<h4>Example</h4>

<p><span>GET https://portfolio.api.anynetiot.com/api/v2/portal/f430d1738a6c2c34be504e1d3a9a6fc</span> </p>

<h3>Response schema application/json: </h3>

<p><span>portalId</span>; <span>name</span>; <span>title</span>; <span>portfolioId</span>; <span>defaultHost</span>; <span>subdomain</span>; <span>status</span></p>

<Note>For more information, see <a href="/api/portfolio/portfolio-api-reference#Portal" target="_blank">Portal definition</a>.</Note>

<h2 id="Creatinganewportal">Creating a new portal</h2>

<p>To create a new portal:</p>

<ul> <li> Call <span>POST /api/v2/portal/</span>. <br /><span>Body parameters:</span> <span>name</span><b>&#42;</b>; <span>title</span><b>&#42;</b>; <span>portfolioId</span><b>&#42;</b>; <span>defaultHost</span><b>&#42;</b>; <span>subdomain</span> <span><strong>Note: </strong> JSON keys marked <b>&#42;</b> are mandatory. <br />Do not use <span>null</span> or <span>""</span> (empty) for mandatory parameters. <br />You can use one or more optional parameters. </span> <br />The API creates a new portal. </li> </ul>

<h2 id="Updatingaspecificportal">Updating a specific portal</h2>

<p>To update a specific portal:</p>

<ul> <li> Call <span>PATCH /api/v2/portal/&#123;<var>portalId</var>&#125;</span>, where &#123;<var>portalId</var>&#125; identifies which portal you want to update. <br /><span>Path parameters:</span> <span>portalId</span><b>&#42;</b> <br /><span>Body:</span> <span>name</span>; <span>title</span>; <span>portfolioId</span>; <span>defaultHost</span>; <span>subdomain</span>; <span>status</span> <span><strong>Note: </strong> JSON keys marked <b>&#42;</b> are mandatory. <br />Do not use <span>null</span> or <span>""</span> (empty) for mandatory parameters. <br />If required, you can update more than one parameter. <br />If you are updating the following: <span>portfolioId</span>, then that value must already exist in the system. </span> <br />The API updates the specified portal parameters with the new details. </li> </ul>

<h4>Example</h4>

<p>PATCH https://portfolio.api.anynetiot.com/api/v2/portal/f430d1738a6c2c34be504e1d3a9a6fc</p>

<p>/&#42; header information &#42;/</p>

<p>&#123;</p>

<p>"status": "active"</p>

<p>&#125;</p>
