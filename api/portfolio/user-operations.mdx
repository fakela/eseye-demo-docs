<h2 id="Viewingallusers">Viewing all users</h2>

<p>To view all users:</p>

<ul> <li> Call <span>GET /api/v2/portfolio/&#123;<var>portfolioId</var>&#125;/user/</span>, where &#123;<var>portfolioId</var>&#125; identifies the parent portfolio containing the users you want to view. <span><strong>Tip: </strong>You can apply filters in the request path to limit the results, using one or more parameters. The API returns exactly matching records only. Alternatively, remove all filters from the path to return all records. For example, adding <span><span>?status=active</span></span> to the end of the path will return <span>active</span> objects only.</span> <br />The API returns all the users. </li> </ul>

<h4>Example</h4>

<p><span>GET https://portfolio.api.anynetiot.com/api/v2/portfolio/a628ebc659f7dac01f9fdcbd1116307a/user/</span> </p>

<p>or</p>

<p><span>GET https://portfolio.api.anynetiot.com/api/v2/portfolio/a628ebc659f7dac01f9fdcbd1116307a/user/?userTypeId=service;status=active</span> – this will return all active <span>service</span> users.</p>

<h2 id="Viewingaspecificuser">Viewing a specific user</h2>

<p>To view details for a specific user:</p>

<ul> <li> Call <span>GET /api/v2/portfolio/&#123;<var>portfolioId</var>&#125;/user/&#123;<var>userId</var>&#125;</span> <br /> where &#123;<var>portfolioId</var>&#125; identifies the parent portfolio and &#123;<var>userId</var>&#125; identifies which user you want to view. <br /><span>Path parameters:</span> <span>portfolioId</span><b>&#42;</b>; <span>userId</span><b>&#42;</b> <span><strong>Note: </strong>JSON keys marked <b>&#42;</b> are mandatory.</span> <br />The API returns the specified user details. </li> </ul>

<h4>Example</h4>

<p><span>GET https://portfolio.api.anynetiot.com/api/v2/portfolio/a628ebc659f7dac01f9fdcbd1116307a/user/8d9cde45a21f3f6cb52a368e7fc70b9c</span> </p>

<h2 id="Creatinganewuser">Creating a new user</h2>

<p>To create a new user:</p>

<ul> <li> Call <span>POST /api/v2/portfolio/&#123;<var>portfolioId</var>&#125;/user/</span> <br /> where &#123;<var>portfolioId</var>&#125; identifies the portfolio where you want to create the new user. <br /><span>Path parameters:</span> <span>portfolioId</span><b>&#42;</b> <br /><span>Body parameters:</span> <span>firstName</span><b>&#42;</b>; <span>lastName</span><b>&#42;</b>; <span>emailAddress</span><b>&#42;</b>; <span>username</span><b>&#42;</b>; <span>mobileNumber</span><b>&#42;</b>; <span>userTypeId</span><b>&#42;</b>; <span>landLineNumber</span>; <span>jobTitleId</span>; <span>status</span> <span><strong>Note: </strong> JSON keys marked <b>&#42;</b> are mandatory. <br />Do not use <span>null</span> or <span>""</span> (empty) for mandatory parameters. <br />If required, you can use more than one optional parameter. </span> <br />The API creates a new user. </li> </ul>

<h2 id="Updatingaspecificuser">Updating a specific user</h2>

<p>To update a specific user:</p>

<ul> <li> Call <span>PATCH /api/v2/portfolio/&#123;<var>portfolioId</var>&#125;/user/&#123;<var>userId</var>&#125;</span>, where &#123;<var>portfolioId</var>&#125; identifies the parent portfolio, and &#123;<var>userId</var>&#125; identifies which user you want to update. <br /><span>Path parameters:</span> <span>portfolioId</span><b>&#42;</b>; <span>userId</span><b>&#42;</b>;  <br /><span>Body:</span> <span>portfolioId</span>; <span>firstName</span>; <span>lastName</span>; <span>emailAddress</span>; <span>username</span>; <span>mobileNumber</span>; <span>userTypeId</span>; <span>landLineNumber</span>; <span>jobTitleId</span>; <span>status</span> <span><strong>Note: </strong> JSON keys marked <b>&#42;</b> are mandatory. <br />Do not use <span>null</span> or <span>""</span> (empty) for mandatory fields. <br />The <span>portfolioId</span>, <span>userTypeId</span> and <span>jobTitleId</span> values must already exist in the system. <br />If required, you can update more than one optional parameter. </span> <br />The API updates the specified user parameters with the new details. </li> </ul>

<h4>Example</h4>

<p>PATCH https://portfolio.api.anynetiot.com/api/v2/portfolio/a628ebc659f7dac01f9fdcbd1116307a/user/8d9cde45a21f3f6cb52a368e7fc70b9c</p>

<p>/&#42; header information &#42;/</p>

<p>&#123;</p>

<p>"status": "active"</p>
