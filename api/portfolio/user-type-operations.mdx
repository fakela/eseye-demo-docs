<h2 id="Viewingallusertypes">Viewing all user types</h2>

<p>To view all users types:</p>

<ul> <li> Call <span>GET /api/v2/userType/</span> <span><strong>Tip: </strong>You can apply filters in the request path to limit the results, using one or more parameters. The API returns exactly matching records only. Alternatively, remove all filters from the path to return all records. For example, adding <span><span>?status=active</span></span> to the end of the path will return <span>active</span> objects only.</span> <br />The API returns all the user types. </li> </ul>

<h4>Example</h4>

<p><span>GET https://portfolio.api.anynetiot.com/api/userType/</span> </p>

<p>or</p>

<p><span>GET https://portfolio.api.anynetiot.com/api/v2/userType/?status=active</span> </p>

<h2 id="Viewingaspecificusertype">Viewing a specific user type</h2>

<p>To view details for a specific user type:</p>

<ul> <li> Call <span>GET /api/v2/userType/&#123;<var>userTypeId</var>&#125;</span> <br /> where &#123;<var>userTypeId</var>&#125; identifies which user type you want to view. <br /><span>Path parameters:</span> <span>userTypeId</span><b>&#42;</b> <span><strong>Note: </strong>JSON keys marked <b>&#42;</b> are mandatory.</span> <br />The API returns the specified user type details. </li> </ul>

<h4>Example</h4>

<p><span>GET https://portfolio.api.anynetiot.com/api/v2/userType/4</span> </p>

<h3>Response schema application/json: </h3>

<p><span>userTypeId</span>; <span>name</span>; <span>title</span>; <span>status</span></p>

<Note>For more information, see <a href="/api/portfolio/portfolio-api-reference#Package4" target="_blank">Portfolio API reference tables</a>.</Note>

<h4>Example</h4>

<p>&#123;</p>

<p>"userTypeId": 4,</p>

<p>"name": "service",</p>

<p>"title": "service",</p>

<p>"status": "setup"</p>

<p>&#125;</p>

<h2 id="Creatinganewusertype">Creating a new user type</h2>

<p>To create a new user type:</p>

<ul> <li> Call <span>POST /api/v2/userType/</span> <br /><span>Body parameters:</span> <span>name</span><b>&#42;</b>; <span>title</span><b>&#42;</b> <span><strong>Note: </strong> JSON keys marked <b>&#42;</b> are mandatory. <br />Do not use <span>null</span> or <span>""</span> (empty) for mandatory parameters. </span> <br />The API creates a new user type. </li> </ul>

<h2 id="Updatingaspecificusertype">Updating a specific user type</h2>

<p>To update a specific user type:</p>

<ul> <li> Call <span>PATCH /api/v2/userType/&#123;<var>userTypeId</var>&#125;/</span>, where &#123;<var>userTypeId</var>&#125; identifies which user type you want to update. <br /><span>Path parameters:</span>[CDATA[ ]]<span>userTypeId</span><b>&#42;</b>;  <br /><span>Body:</span> <span>name</span>; <span>title</span>; <span>status</span> <span><strong>Note: </strong> JSON keys marked <b>&#42;</b> are mandatory. <br />Do not use <span>null</span> or <span>""</span> (empty) for mandatory fields. <br />If required, you can update more than one optional parameter. </span> <br />The API updates the specified user type parameters with the new details. </li> </ul>

<h4>Example</h4>

<p>PATCH https://portfolio.api.anynetiot.com/api/v2/userType/4</p>

<p>/&#42; header information &#42;/</p>

<p>&#123;</p>

<p>"status": "active"</p>

<p>&#125;</p>
