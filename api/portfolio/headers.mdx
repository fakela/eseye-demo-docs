<p>Headers accompany the body of a request, containing information about the request and how to process it. </p>

<h2 id="Headers">Headers</h2>

<Note>If you are using Postman, the following headers are automatically generated. The endpoint type influences what Postman generates – view the Portfolio API Postman collection endpoint headers to see which ones are included. For more information, see <a href="/api/portfolio/portfolio-api-intro#Using2" target="_blank">Using the supplied Postman collection and environment</a>.</Note>

<h3>Schema</h3>

<table> <col /> <col /> <col /> <thead> <tr> <th>Object name</th> <th>Default value (Postman)</th> <th>Description</th> </tr> </thead> <tbody> <tr> <td>Accept</td> <td>&#42;/&#42;</td> <td> If you are using Postman, we recommend using this header. The "Accept: &#42;/&#42;" header tells the server that Postman can understand and process all forms of response content types. </td> </tr> <tr> <td>Accept-Encoding</td> <td>gzip, deflate, br</td> <td> <div> If you are using Postman, we recommend using this header, for parsing purposes. Indicates to the server that the Postman HTTP client supports a defined list of content-encoding or compression algorithms as response. </div> </td> </tr> <tr> <td>Authorization</td> <td> 'Authorization': '••••••' </td> <td>Automatically added. '••••••' is the Bearer Token set in the environment in the supplied Postman collection and environment.</td> </tr> <tr> <td>Connection</td> <td>keep-alive</td> <td> <div> We recommend using this header. Indicates to the server to keep the underlying network connection open after receiving the current response. This enables you to reuse the same connection for faster response times in subsequent requests to the same server. </div> </td> </tr> <tr> <td>Content-Length</td> <td>&lt;calculated when request is sent&gt;</td> <td> For parsing purposes. Indicates the size of the current request body to the server. </td> </tr> <tr> <td>Content-Type</td> <td>application/json</td> <td> Helps the server identify the media type of the current request body. </td> </tr> <tr> <td>Host</td> <td>&lt;calculated when request is sent&gt;</td> <td> Helps the server identify the domain name where the request is sent. </td> </tr> <tr> <td>Postman-Token</td> <td>&lt;calculated when request is sent&gt;</td> <td> If you are using Postman, we recommend this header, which appends a random UUID to every outgoing request. This aids debugging, by ensuring separate requests are distinct in the receiving server. </td> </tr> <tr> <td>User-Agent</td> <td>PostmanRuntime/&lt;PostmanVersion&gt;</td> <td> If you are using Postman, we recommend using this header. Helps the server identify Postman as the HTTP requesting application or client. </td> </tr> </tbody> </table>
