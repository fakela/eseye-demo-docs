<p>The Portfolio API enables you to: </p>

<ul> <li>View details for <a href="#Viewing2">single</a> or <a href="#Viewing">multiple</a> attributes on a specific portal</li> <li><a href="#Creating">Create</a>, <a href="#Updating">update</a>, and <a href="#Deleting">delete</a> portal attributes on a specific portal</li> </ul>

<p>You can also:</p>

<ul> <li>Create, update, delete and view <a href="/api/portfolio/portal-operations" target="_blank">portals</a>.</li> </ul>

<h2 id="Authentication">Authentication</h2>

<ul> <li>User required to login</li> <li>Bearer token required</li> </ul>

<Note>For more information, see <a href="/api/portfolio/portfolio-api-intro" target="_blank">Portfolio API Introduction</a>.</Note>

<h2 id="Viewingallportalattributes">Viewing all portal attributes</h2>

<p>To view all attributes for a specific portal:</p>

<ul> <li> Call <span>GET /api/v2/portal/&#123;<var>portalId</var>&#125;/attribute/</span> <br />where &#123;<var>portalId</var>&#125; identifies the portal containing the attributes you want to view. <br /><span>Path parameters:</span> <span>portalId</span><b>&#42;</b> <span><strong>Note: </strong>JSON keys marked <b>&#42;</b> are mandatory.</span> <span><strong>Tip: </strong>You can apply filters in the request path to limit the results, using one or more parameters. The API returns exactly matching records only. Alternatively, remove all filters from the path to return all records. For example, adding <span><span>?status=active</span></span> to the end of the path will return <span>active</span> objects only.</span> <br />The API returns all the portal attribute details for the specified portal. </li> </ul>

<p><span>GET https://portfolio.api.anynetiot.com/api/v2/portal/f430d1738a6c2c34be504e1d3a9a6fc/attribute/</span> </p>

<h2 id="Viewingaspecificportalattribute">Viewing a specific portal attribute</h2>

<p>To view details for a specific portal attribute:</p>

<ul> <li> Call <span>GET /api/v2/portal/&#123;<var>portalId</var>&#125;/attribute/&#123;<var>portalAttributeId</var>&#125;</span>, where &#123;<var>portalId</var>&#125; identifies the parent portal, and &#123;<var>portalAttributeId</var>&#125; identifies which attribute you want to view. <br /><span>Path parameters:</span> <span>portalId</span><b>&#42;</b>; <span>portalAttributeId</span><b>&#42;</b> <span><strong>Note: </strong>JSON keys marked <b>&#42;</b> are mandatory.</span> <br />The API returns the specified portal attribute details. </li> </ul>

<h4>Example</h4>

<p><span>GET https://portfolio.api.anynetiot.com/api/v2/portal/f430d1738a6c2c34be504e1d3a9a6fc/attribute/142</span> </p>

<h3>Response schema application/json: </h3>

<p><span>portalAttributeId</span>; <span>portalId</span>; <span>attribute</span>; <span>value</span>; <span>status</span></p>

<Note>For more information, see <a href="/api/portfolio/portfolio-api-reference#Portal2" target="_blank">Portal attribute definition</a>.</Note>

<h4>Example</h4>

<p>&#123;</p>

<p>"portalAttributeId": 142,</p>

<p>"portalId": f430d1738a6c2c34be504e1d3a9a6fc,</p>

<p>"attribute": "defaultLocale",</p>

<p>"value": "en&#95;gb"</p>

<p>"status": "active"</p>

<p>&#125;</p>

<h2 id="Creatinganewportalattribute">Creating a new portal attribute</h2>

<p>To create a new portal attribute:</p>

<ul> <li> Call <span>POST /api/v2/portal/&#123;<var>portalId</var>&#125;/attribute/</span>, where &#123;<var>portalId</var>&#125; identifies the parent portal where you want to create the new attribute. <br /><span>Path parameters:</span> <span>portalId</span><b>&#42;</b> <br /><span>Body parameters:</span> <span>attribute</span><b>&#42;</b>; <span>value</span> <span><strong>Note: </strong>JSON keys marked <b>&#42;</b> are mandatory. You can leave <span>value</span> empty <span>""</span>, but do not use <span>null</span>. </span> <br />The API creates a new portal attribute. </li> </ul>

<h4>Example</h4>

<p>POST https://portfolio.api.anynetiot.com/api/v2/portal/f430d1738a6c2c34be504e1d3a9a6fc/attribute/</p>

<p>/&#42; header information &#42;/</p>

<p>&#123;</p>

<p>"attribute":"dynamicsCompanyName"</p>

<p>"value":"Eseye"</p>

<p>&#125;</p>

<h3>Response schema application/json: </h3>

<p><span>portalAttributeId</span>; <span>portalId</span>; <span>attribute</span>; <span>value</span>; <span>status</span></p>

<Note>For more information, see <a href="/api/portfolio/portfolio-api-reference#Portal2" target="_blank">Portal attribute definition</a>.</Note>

<h4>Example</h4>

<p>&#123;</p>

<p>"portalAttributeId": 153,</p>

<p>"portalId": f430d1738a6c2c34be504e1d3a9a6fc,</p>

<p>"attribute": "orderEmail"</p>

<p>"value": "billing@eseye.com"</p>

<p>"status": "active"</p>

<p>&#125;</p>

<h2 id="Updatingaspecificportalattribute">Updating a specific portal attribute</h2>

<p>To update a specific portal attribute:</p>

<ul> <li> Call <span>PATCH /api/v2/portal/&#123;<var>portalId</var>&#125;/attribute/&#123;<var>portalAttributeId</var>&#125;</span>, where &#123;<var>portalId</var>&#125; identifies the parent portal, and &#123;<var>portalAttributeId</var>&#125; identifies which attribute you want to update. <br /><span>Path parameters:</span> <span>portalId</span><b>&#42;</b>; <span>portalAttributeId</span><b>&#42;</b> <br /> <span>Body:</span> <span>value</span><b>&#42;</b> <span><strong>Note: </strong>JSON keys marked <b>&#42;</b> are mandatory. You can leave <span>value</span> empty <span>""</span>, but do not use <span>null</span>. </span> <br />The API updates the specified portal attribute <span>value</span> with the new details. </li> </ul>

<h4>Example</h4>

<p>PATCH https://portfolio.api.anynetiot.com/api/v2/portal/f430d1738a6c2c34be504e1d3a9a6fc/attribute/153</p>

<p>/&#42; header information &#42;/</p>

<p>&#123;</p>

<p>"value": "orders@eseye.com"</p>

<p>&#125;</p>

<h3>Response schema application/json: </h3>

<p><span>portalAttributeId</span>; <span>portalId</span>; <span>attribute</span>; <span>value</span>; <span>status</span></p>

<Note>For more information, see <a href="/api/portfolio/portfolio-api-reference#Portal2" target="_blank">Portal attribute definition</a>.</Note>

<h4>Example</h4>

<p>&#123;</p>

<p>"portalAttributeId": 153,</p>

<p>"portalId": f430d1738a6c2c34be504e1d3a9a6fc,</p>

<p>"attribute": "orderEmail"</p>

<p>"value": "orders@eseye.com"</p>

<p>"status": "active"</p>
