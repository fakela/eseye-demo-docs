<h2 id="Viewingalladdresstypes">Viewing all address types</h2>

<p>To view all address types:</p>

<ul> <li> Call <span>GET /api/v2/addressType/</span> <span><strong>Tip: </strong>You can apply filters in the request path to limit the results, using one or more parameters. The API returns exactly matching records only. Alternatively, remove all filters from the path to return all records. For example, adding <span><span>?status=active</span></span> to the end of the path will return <span>active</span> objects only.</span> <br />The API returns all the address types. </li> </ul>

<h4>Example</h4>

<p><span>GET https://portfolio.api.anynetiot.com/api/v2/addressType/</span> </p>

<p>or</p>

<p><span>GET https://portfolio.api.anynetiot.com/api/v2/addressType/?status=active</span> </p>

<h2 id="Viewingaspecificaddresstype">Viewing a specific address type</h2>

<p>To view details for a specific address type:</p>

<ul> <li> Call <span>GET /api/v2/addressType/&#123;<var>addressTypeId</var>&#125;</span> <br /> where &#123;<var>addressTypeId</var>&#125; identifies which address type you want to view. <br /><span>Path parameters:</span>[CDATA[ ]]<span>addressTypeId</span><b>&#42;</b> <span><strong>Note: </strong>JSON keys marked <b>&#42;</b> are mandatory.</span> <br />The API returns the specified address type details. </li> </ul>

<h4>Example</h4>

<p><span>GET https://portfolio.api.anynetiot.com/api/v2/addressType/7</span> </p>

<h3>Response schema application/json: </h3>

<p><span>addressTypeId</span>; <span>name</span>; <span>title</span>; <span>status</span></p>

<Note>For more information, see <a href="/api/portfolio/portfolio-api-reference#Address" target="_blank">Address type definition</a>.</Note>

<h4>Example</h4>

<p>&#123;</p>

<p>"addressTypeId": "7",</p>

<p>"name": "Where to send invoices",</p>

<p>"title": "billing",</p>

<p>"status": "active"</p>

<p>&#125;</p>

<h2 id="Creatinganewaddresstype">Creating a new address type</h2>

<p>To create a new address type:</p>

<ul> <li> Call <span>POST /api/v2/addressType/</span> <br /><span>Body parameters:</span> You can use one or more of the following parameters: <span>name</span><b>&#42;</b>; <span>title</span><b>&#42;</b>; <span>status</span> <span><strong>Note: </strong> JSON keys marked <b>&#42;</b> are mandatory. <br />Do not use <span>null</span> or <span>""</span> (empty) for mandatory fields. </span> <br />The API creates a new address type. </li> </ul>

<h2 id="Updatingaspecificaddresstype">Updating a specific address type</h2>

<p>To update a specific address type:</p>

<ul> <li> Call <span>PATCH /api/v2/addressType/&#123;<var>addressTypeId</var>&#125;</span>, where &#123;<var>addressTypeId</var>&#125; identifies which address type you want to update. <br /><span>Path parameters:</span> <span>addressTypeId</span><b>&#42;</b>;  <br /><span>Body:</span> You can update one or more of the following parameters: <span>name</span>; <span>title</span>; <span>status</span> <span><strong>Note: </strong> JSON keys marked <b>&#42;</b> are mandatory. <br />Do not use <span>null</span> or <span>""</span> (empty) for mandatory fields. <br />The <span>countryCodeId</span> value must already exist in the system. </span> <br />The API updates the specified address type parameters with the new details. </li> </ul>

<h4>Example</h4>

<p>PATCH https://portfolio.api.anynetiot.com/api/v2/addressType/1</p>

<p>/&#42; header information &#42;/</p>

<p>&#123;</p>

<p>"status": "setup"</p>
