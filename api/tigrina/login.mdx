<p>Logs in the user using the supplied credentials. The session times out after 1440 seconds of inactivity. If you attempt to use a session cookie after that session has timed out, you will need to log in again.</p>

<h2 id="Requestkeys">Request keys</h2>

<table> <col /> <col /> <col /> <thead> <tr> <th> JSON key </th> <th> Data type </th> <th> Description </th> </tr> </thead> <tbody> <tr> <td> username </td> <td> String </td> <td> End-user name, can contain from 6 to 255 characters </td> </tr> <tr> <td> password </td> <td> String </td> <td> End-user password, can contain from 6 to 255 characters </td> </tr> <tr> <td> portfolioID </td> <td> 32 digit hex </td> <td> End-user Portfolio ID. Must contain exactly 32 characters. </td> </tr> </tbody> </table>

<h3>Request example</h3>

<p>&#123;</p>

<p>"username": "Example@Username.com",</p>

<p>"password": "examplepassword",</p>

<p>"portfolioID": "CustomerPortfolioID"</p>

<p>&#125;</p>

<h2 id="Responsekeys">Response keys</h2>

<p>The response is a status structure, with a cookie value and account permissions on successful authentication.</p>

<table> <col /> <col /> <thead> <tr> <th> JSON key </th> <th> Data type </th> <th> Returned value </th> </tr> </thead> <tbody> <tr> <td> Cookie </td> <td> .&#123;0,36&#125; </td> <td> Session cookie ID </td> </tr> <tr> <td> Status </td> <td> Status object </td> <td> Request status </td> </tr> </tbody> </table>
