<p>A <span>package</span> contains the information required to supply, provision and operate SIMs for a customer. Each package is a specific bundle of products and services, tailored to meet particular customer needs. It defines commercial data (such as the tariff and billing terms), contract terms (such as contract length and agreed testing length), the available SIM services (such as data, voice, SMS), and the technical provisioning data required to activate and operate SIMs. The products and services available in a package are itemised as separate <span>package items</span> on the customer invoice.</p>

<p>The Package API enables you to manage your packages and package items.</p>

<h2 id="Gettingstarted">Getting started</h2>

<p>To get started with our Package API, you need an existing Infinity account, and a Package API service account user. Eseye will create the service account for you.</p>

<p>To request access to the Package API:</p>

<ol> <li> Select <span>Request Package API</span> (below) to display a pre-formatted email that you can use to initiate Package API services. <h2 id="UsingaserviceaccounttoaccesstheAPI">Using a service account to access the API</h2>

<p>Service accounts enable machine-to-machine (m2m) API access. Use the supplied <span>base64String</span> to retrieve an API token.</p>

<p>To retrieve a token using a service account:</p>

<ol> <li> Using an API calling application like <span>Postman</span>, target the following URL: <br />POST https://eseye-idp-prod.auth.eu-west-1.amazoncognito.com/oauth2/token <br />where <span>https://eseye-idp-prod.auth.eu-west-1.amazoncognito.com</span> is the URL for the authorisation server. </li> <li> Ensure the headers include the following: <br />"Content-Type": "application/x-www-form-urlencoded" <br />"Authorization": "Basic" &lt;<span>Base64string</span>&gt; <br />where <span>&lt;<var>Base64string</var>&gt;</span> is the string you generated earlier. For example: <br />Basic NDhoNHY5dmNzdHN1dWlqdjl0OGg1Mzkdbmc6MWlpMnMydmt2M21jOWgwZnJqdGxqbmZyZGs2Mmw5ODN2ZTJ1bHByZXU1N2NydWZhODZsdg </li> <li> Ensure the body includes the following: <br />grant&#95;type=client&#95;credentials <br />scope=api/api. </li> <li> Send the token request. <br />The Package API returns an access token in the following format: <br />&#123; <br />"access&#95;token": "&lt;<var>accessToken</var>&gt;", <br />"expires&#95;in": 3600, <br />"token&#95;type": "Bearer" <br />&#125; <br />where <span>&lt;<var>accessToken</var>&gt;</span> is the JWT token used to authorise API requests, <span>expires&#95;in</span> is the amount of time in seconds that the key remains valid before expiring (the default is 1 hour), and <span>token&#95;type</span> is the <span>Authorization</span> type. </li> <li> Use the <span>&lt;<var>accessToken</var>&gt;</span> to authorise Package API calls, as a <span>Bearer</span> token in the <span>Authorization</span> header, with the following format: <br />"Authorization": "Bearer" &lt;<var>accessToken</var>&gt; <br />For example: <br />Authorization: Bearer eyJraWQiOiJENmZrVU5tMlhpemZ...A <span><strong>Note: </strong>Only endpoints that are authorised by the relevant registered service account permissions will return a valid response.</span> </li> </ol>
