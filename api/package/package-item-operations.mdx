<p>The Package API enables you to: </p>

<ul> <li>View details for <a href="#Viewing2">single</a> or <a href="#Viewing">multiple</a> package items on a specific package</li> <li><a href="#Creating">Create</a>, <a href="#Deleting">delete</a>, and <a href="#Updating">update</a> package items on a specific package</li> </ul>

<p>You can also:</p>

<ul> <li>Create, update, delete and view <a href="/api/package/package-item-attributes-operations" target="_blank">package item attributes</a>.</li> <li>Create, update, delete and view <a href="/api/package/package-operations" target="_blank">packages</a>.</li> <li>Create, update, delete and view <a href="/api/package/package-attribute-operations" target="_blank">package attributes</a>.</li> </ul>

<h2 id="Viewingallpackageitemsforapackage">Viewing all package items for a package</h2>

<p>To view all package items for a specific package:</p>

<ul> <li> Call <span>GET /api/v2/package/&#123;<var>packageId</var>&#125;/packageItem/</span> <br />where &#123;<var>packageId</var>&#125; identifies the parent package containing the package items you want to view. <br /><span>Path parameters:</span> <span>packageId</span><b>&#42;</b> <span><strong>Note: </strong>JSON keys marked <b>&#42;</b> are mandatory.</span> <span><strong>Tip: </strong>You can apply filters in the request path to limit the results, using one or more parameters. The API returns exactly matching records only. Alternatively, remove all filters from the path to return all records. For example, adding <span><span>?status=active</span></span> to the end of the path will return <span>active</span> objects only.</span> <br />The API returns all the package items for the specified package. </li> </ul>

<h2 id="Viewingaspecificpackageitem">Viewing a specific package item</h2>

<p>To view details for a specific package item:</p>

<ul> <li> Call <span>GET /api/v2/package/&#123;<var>packageId</var>&#125;/packageItem/&#123;<var>packageItemId</var>&#125;</span>, where &#123;<var>packageId</var>&#125; identifies the parent package, and &#123;<var>packageItemId</var>&#125; identifies which package item you want to view. <br /><span>Path parameters:</span> <span>packageId</span><b>&#42;</b>; <span>packageItemId</span><b>&#42;</b> <span><strong>Note: </strong>JSON keys marked <b>&#42;</b> are mandatory.</span> <br />The API returns the specified package item details. </li> </ul>

<p><span>GET https://package.api.anynetiot.com/api/v2/package/12345/packageItem/206647</span> </p>

<h2 id="Creatinganewpackageitem">Creating a new package item</h2>

<p>To create a new package item:</p>

<ul> <li> Call <span>POST /api/v2/package/&#123;<var>packageId</var>&#125;/packageItem/</span>, where &#123;<var>packageId</var>&#125; identifies the parent package where you want to create the new package item. <br /><span>Path parameters:</span> <span>packageId</span><b>&#42;</b> <br /><span>Body parameters:</span> <span>name</span><b>&#42;</b>; <span>title</span><b>&#42;</b>; <span>packageItemCategoryId</span><b>&#42;</b>; <span>packageCategoryId</span><b>&#42;</b>; <span>description</span>; <span>transmissionTypeId</span>; <span>transmissionTypeGroupId</span>; <span>inBundleRounding</span>; <span>outOfBundleRounding</span>; <span>testBundleAmount</span>; <span>bundleAmount</span>; <span>outOfBundlePrice</span>; <span>price</span> <span><strong>Note: </strong> JSON keys marked <b>&#42;</b> are mandatory. <br />Do not use <span>null</span> or <span>""</span> (empty) for mandatory parameters. <br />The <span>packageItemCategoryId</span> and <span>packageCategoryId</span> values must already exist in the system. <br />If required, you can use more than one optional parameter. </span> <br />The API creates a new package item. </li> </ul>

<h4>Example</h4>

<p>POST https://package.api.anynetiot.com/api/v2/package/12345/packageItem/</p>

<p>/&#42; header information &#42;/</p>

<p>&#123;</p>

<p>"name": "Zone 1 SMS MO",</p>

<p>"title": "Zone1SMSMO",</p>

<p>"description": "Zone 1 SMS MO",</p>

<p>"inBundleRounding": 1,</p>

<p>"outOfBundleRounding": 1,</p>

<p>"outOfBundlePrice": 1.00,</p>

<p>"price": 0.10,</p>

<h2 id="Updatingaspecificpackageitem">Updating a specific package item</h2>

<p>To update a specific package item:</p>

<ul> <li> Call <span>PATCH /api/v2/package/&#123;<var>packageId</var>&#125;/packageItem/&#123;<var>packageItemId</var>&#125;</span>, where &#123;<var>packageId</var>&#125; identifies the parent package, and &#123;<var>packageItemId</var>&#125; identifies which package item you want to update. <br /><span>Path parameters:</span> <span>packageId</span><b>&#42;</b>; <span>packageItemId</span><b>&#42;</b> <br /><span>Body:</span> You can use one or more of the following parameters: <span>name</span>; <span>title</span>; <span>description</span>; <span>inBundleRounding</span>; <span>outOfBundleRounding</span>; <span>transmissionTypeId</span>; <span>transmissionTypeGroupId</span>; <span>testBundleAmount</span>; <span>bundleAmount</span>; <span>outOfBundlePrice</span>; <span>price</span>; <span>packageItemCategoryId</span>; <span>packageCategoryId</span>; <span>status</span> <span><strong>Note: </strong> JSON keys marked <b>&#42;</b> are mandatory. <br />The <span>packageItemCategoryId</span> and <span>packageCategoryId</span> values must already exist in the system. <br />If required, you can update more than one optional parameter. </span> <br />The API updates the specified package item parameters with the new details. </li> </ul>

<h4>Example</h4>

<p>PATCH https://package.api.anynetiot.com/api/v2/package/12345/packageItem/216913</p>

<p>/&#42; header information &#42;/</p>

<p>&#123;</p>

<p>"status": "active"</p>

<p>&#125;</p>

<h3>Response schema application/json: </h3>

<p><span>packageItemId</span>; <span>name</span>; <span>title</span>; <span>description</span>; <span>packageId</span>; <span>inBundleRounding</span>; <span>outOfBundleRounding</span>; <span>transmissionTypeId</span>; <span>transmissionTypeGroupId</span>; <span>testBundleAmount</span>; <span>bundleAmount</span>; <span>outOfBundlePrice</span>; <span>price</span>; <span>packageItemCategoryId</span>; <span>packageCategoryId</span>; <span>status</span></p>

<p>Each <span>packageItem</span> may also include the <span>mcc</span> array, which describes where the package item applies. Each returned <span>mcc</span> includes:</p>
