<p>The Package API enables you to: </p>

<ul> <li>View details for <a href="#Viewing2">single</a> or <a href="#Viewing">multiple</a> attributes for a specific package</li> <li><a href="#Creating">Create</a>, <a href="#Updating">update</a>, and <a href="#Deleting">delete</a> package attributes</li> </ul>

<p>You can also:</p>

<ul> <li>Create, update, delete and view <a href="/api/package/package-operations" target="_blank">packages</a>.</li> <li>Create, update, delete and view <a href="/api/package/package-item-operations" target="_blank">package items</a>.</li> <li>Create, update, delete and view <a href="/api/package/package-item-attributes-operations" target="_blank">package item attributes</a>.</li> </ul>

<h2 id="Viewingallpackageattributesforapackage">Viewing all package attributes for a package</h2>

<p>To view all attributes for a specific package:</p>

<ul> <li> Call <span>GET /api/v2/package/&#123;<var>packageId</var>&#125;/attribute/</span> <br />where &#123;<var>packageId</var>&#125; identifies the parent package containing the attributes you want to view. <br /><span>Path parameters:</span> <span>packageId</span><b>&#42;</b> <span><strong>Note: </strong>JSON keys marked <b>&#42;</b> are mandatory.</span> <span><strong>Tip: </strong>You can apply filters in the request path to limit the results, using one or more parameters. The API returns exactly matching records only. Alternatively, remove all filters from the path to return all records. For example, adding <span><span>?status=active</span></span> to the end of the path will return <span>active</span> objects only.</span> <br />The API returns all the package attribute details for the specified package. </li> </ul>

<h3>Example</h3>

<p><span>GET /api/v2/package/12345/attribute/</span> </p>

<h2 id="Viewingaspecificpackageattribute">Viewing a specific package attribute</h2>

<p>To view details for a specific package attribute:</p>

<ul> <li> Call <span>GET /api/v2/package/&#123;<var>packageId</var>&#125;/attribute/&#123;<var>packageAttributeId</var>&#125;</span>, where &#123;<var>packageId</var>&#125; identifies the parent package, and &#123;<var>packageAttributeId</var>&#125; identifies which attribute you want to view. <br /><span>Path parameters:</span> <span>packageId</span><b>&#42;</b>; <span>packageAttributeId</span><b>&#42;</b> <span><strong>Note: </strong>JSON keys marked <b>&#42;</b> are mandatory.</span> <br />The API returns the specified package details. </li> </ul>

<h4>Example</h4>

<p><span>GET https://package.api.anynetiot.com/api/v2/package/12345/attribute/21345</span> </p>

<h3>Response schema application/json: </h3>

<p><span>packageAttributeId</span>; <span>packageId</span>; <span>attribute</span>; <span>value</span>; <span>status</span></p>

<Note>For more information, see <a href="/api/package/package-api-reference#Package2" target="_blank">Package attribute definition</a>.</Note>

<h4>Example</h4>

<p>&#123;</p>

<p>"packageAttributeId": 21345,</p>

<p>"packageId": 12345,</p>

<p>"attribute": "autoProvision",</p>

<p>"value": "no"</p>

<p>"status": "active"</p>

<p>&#125;</p>

<h2 id="Creatinganewpackageattribute">Creating a new package attribute</h2>

<p>To create a new package attribute:</p>

<ul> <li> Call <span>POST /api/v2/package/&#123;<var>packageId</var>&#125;/attribute/</span>, where &#123;<var>packageId</var>&#125; identifies the parent package where you want to create the new attribute. <br /><span>Path parameters:</span> <span>packageId</span><b>&#42;</b> <br /><span>Body parameters:</span><span>attribute</span><b>&#42;</b>; <span>value</span> <span><strong>Note: </strong>JSON keys marked <b>&#42;</b> are mandatory. You can leave <span>value</span> empty <span>""</span>, but do not use <span>null</span>. </span> <br />The API creates a new package attribute. </li> </ul>

<h4>Example</h4>

<p>POST https://package.api.anynetiot.com/api/v2/package/12345/attribute/</p>

<p>/&#42; header information &#42;/</p>

<p>&#123;</p>

<p>"attribute":"MSazure"</p>

<p>"value":"no"</p>

<p>&#125;</p>

<h3>Response schema application/json: </h3>

<p><span>packageAttributeId</span>; <span>packageId</span>; <span>attribute</span>; <span>value</span>; <span>status</span></p>

<Note>For more information, see <a href="/api/package/package-api-reference#Package2" target="_blank">Package attribute definition</a>.</Note>

<h4>Example</h4>

<p>&#123;</p>

<p>"packageAttributeId": 70123,</p>

<p>"packageId": 12345,</p>

<p>"attribute": "MSazure"</p>

<p>"value": "no"</p>

<p>"status": "active"</p>

<p>&#125;</p>

<h2 id="Updatingaspecificpackageattribute">Updating a specific package attribute</h2>

<p>To update a specific package attribute:</p>

<ul> <li> Call <span>PATCH /api/v2/package/&#123;<var>packageId</var>&#125;/attribute/&#123;<var>packageAttributeId</var>&#125;</span>, where &#123;<var>packageId</var>&#125; identifies the parent package, and &#123;<var>packageAttributeId</var>&#125; identifies which attribute you want to update. <br /><span>Path parameters:</span> <span>packageId</span><b>&#42;</b>; <span>packageAttributeId</span><b>&#42;</b> <br /> <span>Body:</span> <span>value</span> <span><strong>Note: </strong>JSON keys marked <b>&#42;</b> are mandatory. You can leave <span>value</span> empty <span>""</span>, but do not use <span>null</span>. </span> <br />The API updates the specified package attribute <span>value</span> with the new details. </li> </ul>

<h4>Example</h4>

<p>PATCH https://package.api.anynetiot.com/api/v2/package/12345/attribute/70123</p>

<p>/&#42; header information &#42;/</p>

<p>&#123;</p>

<p>"value": "yes"</p>

<p>&#125;</p>

<h3>Response schema application/json: </h3>

<p><span>packageAttributeId</span>; <span>packageId</span>; <span>attribute</span>; <span>value</span>; <span>status</span></p>

<Note>For more information, see <a href="/api/package/package-api-reference#Package2" target="_blank">Package attribute definition</a>.</Note>

<h4>Example</h4>

<p>&#123;</p>

<p>"packageAttributeId": 70123,</p>

<p>"packageId": 12345,</p>

<p>"attribute": "MSazure"</p>

<p>"value": "yes"</p>

<p>"status": "active"</p>

<p>&#125;</p>

<h2 id="Deletingaspecificpackageattribute">Deleting a specific package attribute</h2>

<p>To delete a specific package attribute:</p>

<ul> <li> Call <span>DELETE <span>/api/v2/package/&#123;<var>packageId</var>&#125;/attribute/&#123;<var>packageAttributeId</var>&#125;</span></span>, where &#123;<var>packageId</var>&#125; identifies the parent package, and &#123;<var>packageAttributeId</var>&#125; identifies which attribute you want to delete. <br /><span>Path parameters:</span> <span>packageId</span><b>&#42;</b>; <span>packageAttributeId</span><b>&#42;</b> <span><strong>Note: </strong>JSON keys marked <b>&#42;</b> are mandatory.</span> <br />The API removes the specified package attribute. </li> </ul>

<h4>Example</h4>

<p>PATCH https://package.api.api.anynetiot.com/api/v2/package/12345/attribute/21345</p>

<h3>Response schema application/json: </h3>

<p>&#123;</p>

<p>"PACKAGE&#95;ATTRIBUTE": 1</p>
