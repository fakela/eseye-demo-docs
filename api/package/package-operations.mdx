<h2 id="Viewingallpackages">Viewing all packages</h2>

<p>To view all packages:</p>

<ul> <li> Call <span>GET /api/v2/package/</span> <span><strong>Tip: </strong>You can apply filters in the request path to limit the results, using one or more parameters. The API returns exactly matching records only. Alternatively, remove all filters from the path to return all records. For example, adding <span><span>?status=active</span></span> to the end of the path will return <span>active</span> objects only.</span> <br />The API returns all the packages. </li> </ul>

<h4>Example</h4>

<p><span>GET https://package.api.anynetiot.com/api/v2/package/</span> </p>

<p>or</p>

<p><span>GET https://package.api.anynetiot.com/api/v2/package/?subnetRangeType=Sim4VPN;status=active</span> – this will return all packages with a <span>Sim4VPN</span> subnetRangetype and <span>active</span> status.</p>

<h2 id="Viewingaspecificpackage">Viewing a specific package</h2>

<p>To view details for a specific package:</p>

<ul> <li> Call <span>GET /api/v2/package/&#123;<var>packageId</var>&#125;</span>, where &#123;<var>packageId</var>&#125; identifies which package you want to view. <br /><span>Path parameters:</span> <span>packageId</span><b>&#42;</b> <span><strong>Note: </strong>JSON keys marked <b>&#42;</b> are mandatory.</span> <br />The API returns the specified package details. </li> </ul>

<h4>Example</h4>

<p><span>GET https://package.api.anynetiot.com/api/v2/package/12345</span> </p>

<h3>Response schema application/json: </h3>

<p><span>ratingSourceId</span>; <span>name</span>; <span>title</span>; <span>description</span>; <span>portalId</span>; <span>currencyId</span>; <span>price</span>; <span>billingPeriod</span>; <span>simPrice</span>; <span>escapePrice</span>; <span>unusedRebatePrice</span>; <span>contractPeriod</span>; <span>activationPrice</span>; <span>terminationPrice</span>; <span>categoryId</span>; <span>freeActivationDay</span>; <span>freeFirstUseDay</span>; <span>testingPrice</span>; <span>residualContractPrice</span>; <span>subnetRangeType</span>; <span>defaultMccId</span>;; <span>status</span></p>

<Note>For more information, see <a href="/api/package/package-api-reference#Package" target="_blank">Package definition</a>.</Note>

<p>If a <span>package</span> contains attributes, these are returned in the <span>attributes</span> array. Each returned attribute includes:</p>

<p><span>packageAttributeId</span>; <span>packageId</span>; <span>attribute</span>; <span>value</span>; <span>status</span></p>

<Note>For more information, see <a href="/api/package/package-api-reference#Package2" target="_blank">Package attribute definition</a>.</Note>

<p>Each <span>package</span> may also include the <span>portfolios</span> array, which describes which customers use this package. Each returned <span>portfolio</span> includes:</p>

<p><span>packagePortfolioId</span>, <span>packageId</span>, <span>portfolioId</span>; <span>adminSuspend</span>; <span>customerSuspend</span>; <span>status</span></p>

<Note>For more information, see <a href="/api/package/package-api-reference#PackageP" target="_blank">PackagePortfolio definition</a>.</Note>

<p>Each <span>package</span> may also include the <span>commsProfile</span> array, which describes which network settings are available on this package. Each returned <span>commsProfile</span> includes:</p>

<p><span>commsProfileId</span> </p>

<Note>For more information, see <a href="/api/package/package-api-reference#Comms" target="_blank">Comms profile definition</a>.</Note>

<h2 id="Creatinganewpackage">Creating a new package</h2>

<p>To create a new package:</p>

<ul> <li> Call <span>POST /api/v2/package/</span>. <br /><span>Body parameters:</span> <span>name</span><b>&#42;</b>; <span>title</span><b>&#42;</b>; <span>billingPeriod</span><b>&#42;</b>; <span>currencyId</span><b>&#42;</b>; <span>defaultMccId</span><b>&#42;</b>; <span>price</span><b>&#42;</b>; <span>description</span>;; <span>simPrice</span>; <span>escapePrice</span>; <span>unusedRebatePrice</span>; <span>contractPeriod</span>; <span>activationPrice</span>; <span>terminationPrice</span>; <span>freeActivationDay</span>; <span>freeFirstUseDay</span>; <span>testingPrice</span>; <span>residualContractPrice</span>; <span>subnetRangeType</span>; <span>packageCategoryId</span> <span><strong>Note: </strong> JSON keys marked <b>&#42;</b> are mandatory. <br />Do not use <span>null</span> or <span>""</span> (empty) for mandatory parameters. <br />The <span>packageCategoryId</span> value must already exist in the system. <br />If required, you can use more than one optional parameter. </span> <br />The API creates a new package. </li> </ul>

<h2 id="Updatingaspecificpackage">Updating a specific package</h2>

<p>To update a specific package:</p>

<ul> <li> Call <span>PATCH /api/v2/package/&#123;<var>packageId</var>&#125;</span>, where &#123;<var>packageId</var>&#125; identifies which package you want to update. <br /><span>Path parameters:</span> <span>packageId</span><b>&#42;</b> <br /><span>Body:</span> <span>name</span>; <span>title</span>; <span>billingPeriod</span>; <span>currencyId</span>; <span>defaultMccId</span>; <span>price</span>; <span>description</span>; <span>simPrice</span>; <span>escapePrice</span>; <span>unusedRebatePrice</span>; <span>contractPeriod</span>; <span>activationPrice</span>; <span>terminationPrice</span>; <span>freeActivationDay</span>; <span>freeFirstUseDay</span>; <span>testingPrice</span>; <span>residualContractPrice</span>; <span>subnetRangeType</span>; <span>packageCategoryId</span>; <span>status</span> <span><strong>Note: </strong> JSON keys marked <b>&#42;</b> are mandatory. <br />The <span>packageCategoryId</span> value must already exist in the system. <br />If required, you can update more than one optional parameter. </span> <br />The API updates the specified package parameters with the new details. </li> </ul>

<h4>Example</h4>

<p>PATCH https://package.api.anynetiot.com/api/v2/package/13579</p>

<p>/&#42; header information &#42;/</p>

<p>&#123;</p>

<p>"status": "active"</p>
