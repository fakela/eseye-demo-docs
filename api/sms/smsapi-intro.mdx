<p>The SMS API enables you to send and receive text and short messages to and from SMS-capable devices over internet and cellular networks. The SMS API ensures the message is compatible for network delivery. It also notifies the sender about the message status, so that you can track if the message was delivered or not.</p>

<p> <img src="/images/resources/images/apis/sms/smsapioverview.svg" alt="Smsapioverview" /> </p>

<p>You can send the following requests to the SMS API:</p>

<ul> <li> <a href="/api/sms/mtsms">MT SMS</a> – sends a mobile terminated (MT) SMS messages to an IoT device. <br />You send SMS messages to both AnyNet SIMs and third party SIMs. However, SMS messages sent to and received from third party SIMs will incur extra costs. For information about SMS API billing, see <a href="/billing/billing-sm-scharges" target="_blank">Understanding SMS charges on the CSV invoice</a>. </li> <li> <a href="/api/sms/mt-cancel">Cancel SMS</a> – cancels a previously send MT SMS message.  </li> <li> <a href="/api/sms/mt-status">Status update</a> – requests the delivery status of an SMS message. </li> </ul>

<p>You can receive the following messages from the SMS API:</p>

<ul> <li><a href="/api/sms/mosms">MO SMS</a> – the AnyNet Messaging Service intercepts and routes mobile originated (MO) SMS messages (sent to a specific API MSISDN) to the customer's SMS application.</li> <li><a href="/api/sms/delivery-receipt">Delivery receipts</a> – returned to the customer's SMS application or alternatively to a different application.</li> </ul>
