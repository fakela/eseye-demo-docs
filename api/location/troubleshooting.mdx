<p>When an error occurs, an error code and message are generated in the returned result:</p>

<p>&#123;</p>

<p>"status": &#123;</p>

<p>"status": "Error",</p>

<p>"errorCode": "&lt;ErrorCodeNumber&gt;", </p>

<p>"errorMessage": "&lt;ErrorCodeMessage&gt;"</p>

<p>&#125;,</p>

<p>The following table can help you diagnose the error:</p>

Ensure the JSON text follows the format described.  <span><strong>Note: </strong>You may receive this error if you copied and pasted text from Microsoft Word. Some MS Word characters are slightly different from ASCII text. To avoid this, either type the text directly into Postman, or configure Microsoft Word to use basic ASCII text.</span> </td> </tr> <tr> <td>1 – The number of cells is 0</td> <td>The mast, power, or ta values are absent or entered incorrectly.</td> </tr> <tr> <td> 601 – Missing parameter ICCID/MSISDN </td> <td> The ICCID or MSISDN is absent or entered incorrectly, or the "s" in the https section of the request URL was not entered. </td> </tr> <tr> <td>602 – Unauthorised Access, Invalid Account, or Invalid Password</td> <td> This may indicate: <ul> <li>The username or password is incorrect.</li> <li>The account does not exist in Infinity Classic. Speak to your Account Manager for more information.</li> <li>The account is not authorised to use the Location API. Speak to your Account Manager for more information.</li> </ul> </td> </tr> <tr> <td> 603 – Parameter &lt;parameter&gt; value missing <br />where &lt;parameter&gt; is the missing parameter </td> <td> The reported parameter field name or value is absent or entered incorrectly.  <br />If the parameter is one of two parameters that you can use in the request, then the error refers to either parameter.
