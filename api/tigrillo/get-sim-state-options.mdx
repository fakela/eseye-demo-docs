<p>Returns the state options you can apply to the current SIM, based on the current state: </p>

<ul> <li>If the SIM is active (provisioned), then you can suspend or terminate (deprovision) it. </li> <li>If the SIM is suspended, then you can reactivate or deprovision it.</li> <li>If the SIM is unprovisioned (terminated), then you can reactivate it.</li> </ul>

<p>Eseye SIMs have the following lifecycle in the API:</p>

<p><img src="/images/resources/images/management/simlifecycleapi.png" alt="Simlifecycleapi" /> </p>

<p>For more information, see <a href="/billing/billing-sim-lifecycle" target="_blank">Understanding the SIM lifecycle</a>.</p>

<p>The Tigrillo API fetches the following statuses:</p>

<ul> <li><span>null</span> (default) – returns all values, regardless of state</li> </ul>

<ul> <li><span>available</span> </li> <li><span>pendingProvision</span> </li> <li><span>provisioned</span> </li> <li><span>suspended</span> </li> <li><span>pendingDeprovision</span> </li> <li><span>unprovisioned</span> </li> </ul>
