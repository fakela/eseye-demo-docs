<p>When an error occurs, an error code and message are generated in the returned result. The error format depends on where the error originated.</p>

<h2 id="Generalandsystemerrors">General and system errors</h2>

<p><span>Errors in the URL:</span> E0002, E0003</p>

<p><span>System errors:</span> E9998</p>

<p>&#123;</p>

<p>"status": &#123;</p>

<p>"status": ".&#123;0,36&#125;",</p>

<p>"errorCode": "E[0-9]&#123;4&#125;", </p>

<p>"errorMessage": ".&#123;0,255&#125;"</p>

<p>&#125;,</p>

<p>&#125;</p>

<h2 id="Singleoperationerrors">Single operation errors</h2>

<p>Errors for single operations (such as login, or getSIMStateOptions for a single SIM) return the following format:</p>

<p>&#123;</p>

<p>"status": &#123;</p>

<p>"status": ".&#123;0,36&#125;",</p>

<p>"code": "E[0-9]&#123;4&#125;", </p>

<p>"message": ".&#123;0,255&#125;"</p>

<p>&#125;,</p>

<p>&#125;</p>

<p><span>Affected errors:</span> E0000, E0001, E0100, E0004, E0005, E0006, E0137 (for single SIM endpoints: getSIMStateOptions, getSIMLastActivity, setSIMDetail), E0140, and F0000 </p>

<h2 id="BulkSIMoperationerrors">Bulk SIM operation errors</h2>

<p>For bulk SIM operations, where an endpoint manipulating the SIM status is applied to more than one SIM, the following format occurs (for one or more SIMs with an error):</p>

<p>&#123;</p>

<p>"status": &#123;</p>

<p>"status": "OK",</p>

<p>"errorCode": "", </p>

<p>"errorMessage": ""</p>

<p>&#125;,</p>

<p>"simStatuses": &#123;</p>

<p>"&lt;<var>ICCID</var>&gt;": "&lt;<var>errorCode</var>&gt;&lt;<var>errorMessage</var>&gt;",</p>

<p>"&lt;<var>ICCID</var>&gt;": "&lt;<var>errorCode</var>&gt;&lt;<var>errorMessage</var>&gt;",</p>

<p>...</p>

<p>&#125;</p>

<p>&#125;</p>

<p><span>Affected errors:</span> E0137 (for bulk SIM endpoints: activateSIMs, suspendSIMs, or terminateSIMs), E0144, E0145</p>

<p>For bulk operations, if one SIM presents an error, the API will not process the other SIMs. The error will show for one or more invalid SIMs only, and the status on the valid SIMs will not change.</p>

<p>For example, for <span>suspendSIMs</span>, where 8955170000202030123 does not exist on the account, the following is sent:</p>

<p>&#123;</p>

<p>"sims": [</p>

<p>"8955170000202030996",</p>

<p>"8955170000202030123"</p>

<p>]</p>

<p>&#125;</p>

<p>The following code is returned, and neither SIM is suspended:</p>

<p>&#123;</p>

<p>"status": &#123;</p>

<p>"status": "OK",</p>

<p>"errorCode": "",</p>

<p>"errorMessage": ""</p>

<p>&#125;,</p>

<p>"simStatuses": &#123;</p>

<p>"8955170000202030123": "E0137:Invalid SIM no. provided"</p>

<p>&#125;</p>

<p>&#125;</p>
