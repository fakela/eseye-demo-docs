<p>Eseye sends customer-specific Radius Authentication data as raw push messages to subscribed customers. Each record appears in the format described below.</p>

<h2 id="Rawpushmessagejson">Raw push message/json</h2>

<h3>Schema</h3>

<table> <col /> <col /> <col /> <thead> <tr> <th>Object name</th> <th>Type</th> <th>Description</th> </tr> </thead> <tbody> <tr> <td><span>apn</span> </td> <td>string</td> <td>The Access Point Name the device used to access the IP network.</td> </tr> <tr> <td><span>authDateTime</span> </td> <td>DateTime</td> <td> Date and time the data capture started. <br />UTC format: YYYY-MM-DDThh:mm:ss.ssssss, where T is the time separator. </td> </tr> <tr> <td><span>authState</span> </td> <td>string</td> <td> The authentication state, either: <br /><span>Access-Accept</span> – the user is allowed access to the system <br /><span>Access-Reject</span> – the user is not allowed access to the system <span><strong>Note: </strong>You can subscribe to one or both authentication states. </span> </td> </tr> <tr> <td><span>eventDateTime</span> </td> <td>DateTime</td> <td> Date and time the data capture started. <br />UTC format: YYYY-MM-DDThh:mm:ss.ssssss, where T is the time separator. </td> </tr> <tr> <td><span>expiryDateTime</span> </td> <td>DateTime</td> <td> The timeout value configured for the Radius Authentication cache by the network. <br />UTC format: YYYY-MM-DDThh:mm:ss.ssssss, where T is the time separator. </td> </tr> <tr> <td><span>iccId</span> </td> <td>string</td> <td>The unique Integrated Circuit Card Identifier of the specific SIM used by the device.</td> </tr> <tr> <td><span>imei</span> </td> <td>string</td> <td> International Mobile Equipment Identity. The unique 15-digit International Mobile Equipment Identity serial number that identifies the device on the cellular network. </td> </tr> <tr> <td><span>imsi</span> </td> <td>string</td> <td>  International Mobile Subscriber Identity number. Identifies a cellular network subscriber. The network operator that provided the IMSI uses it to authenticate a SIM (and the IoT device containing the SIM) on the network.  <span><strong>Note: </strong>Contains between 15 and 16 numeric digits.</span> </td> </tr> <tr> <td><span>ip</span> </td> <td>string</td> <td>The assigned IP address.</td> </tr> <tr> <td><span>laiCI</span> </td> <td>string</td> <td> The LAI and CI, concatenated: <br />Location Area Identity – the unique identifier of a location area, in hexadecimal format. Tracks the physical location of the SIM within the network. <br />Cell Identification – identifies the cell tower in a location area, in hexadecimal format. </td> </tr> <tr> <td><span>mccmnc</span> </td> <td>string</td> <td> The MCC and MNC, concatenated:  <br />Mobile Country Code – identifies the country of the cellular tower to which the SIM last connected, in three-digit numeric format. For more information, see <a href="https://en.wikipedia.org/wiki/Mobile_country_code">Mobile country codes</a>.  <br />Mobile Network Code – identifies the network operator of the network to which the SIM last subscribed, in two-digit numeric format. For more information, see: <a href="https://en.wikipedia.org/wiki/Mobile_country_code">Mobile network codes listed by country</a>. </td> </tr> <tr> <td><span>messageSequenceNumber</span> </td> <td>string</td> <td>A network-generated unique reference for the payload.</td> </tr> <tr> <td><span>mnoId</span> </td> <td>string</td> <td>The Mobile Network Operator Identifier.</td> </tr> <tr> <td><span>msisdn</span> </td> <td>string</td> <td>The unique primary SIM phone number (Mobile Station International ISDN Number).</td> </tr> <tr> <td><span>notificationClass</span> </td> <td>string</td> <td> The data class, to differentiate between the types of data pushed by the Push API. These are <span>netflow</span>, <span>radiusauth</span> and <span>radiusacct</span>. </td> </tr> <tr> <td><span>payload</span> </td> <td>array</td> <td>The array, listing the network data for an individual device.</td> </tr> <tr> <td><span>portfolioId</span> </td> <td>string</td> <td>The Infinity Classic account portfolio identifier. Contains exactly 32 characters.</td> </tr> <tr> <td><span>rat</span> </td> <td>string</td> <td> The Radio Access Technology type used by the device to connect to the internet. The following RAT types may appear: <ul> <li><span>WLAN</span> – Wireless LAN</li> <li><span>UTRAN</span> – UMTS Terrestrial Radio Access Network </li> <li><span>GERAN</span> – GSM EDGE Radio Access Network</li> <li><span>GAN</span> – Generic Access Network</li> <li><span>HSPA&#95;EVOLUTION</span> – Evolved HSPA (High Speed Packet Access)</li> <li><span>EUTRAN</span> – Evolved Universal Mobile Telecommunications System (UMTS) Terrestrial Radio Access Network</li> <li><span>VIRTUAL</span> </li> </ul> </td> </tr> <tr> <td><span>simId</span> </td> <td>string</td> <td>The SIM's unique serial number.</td> </tr> <tr> <td><span>userName</span> </td> <td>string</td> <td>The login user who started the session.</td> </tr> </tbody> </table>
