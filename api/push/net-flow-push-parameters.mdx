<p>Eseye sends customer-specific Netflow data as raw push messages to subscribed customers. Each record appears in the format described below.</p>

<h2 id="Rawpushmessagejson">Raw push message/json</h2>

<h3>Schema</h3>

<table> <col /> <col /> <col /> <thead> <tr> <th>Object name</th> <th>Type</th> <th>Description</th> </tr> </thead> <tbody> <tr> <td>destinationAddressIP</td> <td>string</td> <td>The IP address where the data was sent.</td> </tr> <tr> <td>destinationPort</td> <td>string</td> <td>The port where the network connection ended.</td> </tr> <tr> <td>duration</td> <td>string</td> <td>Reporting period, in seconds, rounded to the third decimal point.</td> </tr> <tr> <td>eventDateTime</td> <td>DateTime</td> <td> Date and time the data capture started. <br />UTC format: YYYY-MM-DDThh:mm:ss.ssssss, where T is the time separator. </td> </tr> <tr> <td>firstDateTime</td> <td>DateTime</td> <td> Date and time the reporting started. <br />UTC format: YYYY-MM-DDThh:mm:ss.ssssss, where T is the time separator. </td> </tr> <tr> <td>imei</td> <td>string</td> <td> International Mobile Equipment Identity. The unique 15-digit International Mobile Equipment Identity serial number that identifies the device on the cellular network. </td> </tr> <tr> <td>imsi</td> <td>string</td> <td>  International Mobile Subscriber Identity number. Identifies a cellular network subscriber. The network operator that provided the IMSI uses it to authenticate a SIM (and the IoT device containing the SIM) on the network.  <span><strong>Note: </strong>Contains between 15 and 16 numeric digits.</span> </td> </tr> <tr> <td>inBytes</td> <td>string</td> <td>The amount of data transferred to the SIM during the reported session, in bytes.</td> </tr> <tr> <td>inPackets</td> <td>string</td> <td>The total number of IP packets received by the SIM during the reported session.</td> </tr> <tr> <td>lastDateTime</td> <td>DateTime</td> <td> Date and time the reporting ended. <br />UTC format: YYYY-MM-DDThh:mm:ss.ssssss, where T is the time separator. </td> </tr> <tr> <td>match</td> <td>string</td> <td> Describes where the data originated, either <span>source</span> or <span>destination</span>. </td> </tr> <tr> <td>meId</td> <td>string</td> <td>Unused.</td> </tr> <tr> <td>outBytes</td> <td>string</td> <td>The amount of data transferred from the SIM during the reported session, in bytes.</td> </tr> <tr> <td>outPackets</td> <td>string</td> <td>The total number of IP packets sent by the SIM during the reported session.</td> </tr> <tr> <td>portfolioId</td> <td>string</td> <td>The SIAM account portfolio identifier. Contains exactly 32 characters.</td> </tr> <tr> <td>protocol</td> <td>string</td> <td>The transport protocol used.</td> </tr> <tr> <td>simId</td> <td>string</td> <td>The SIM's unique serial number.</td> </tr> <tr> <td>sourceAddressIP</td> <td>string</td> <td>The IP address where the data originated.</td> </tr> <tr> <td>sourcePort</td> <td>string</td> <td>The port where the network connection originated.</td> </tr> </tbody> </table>
