<p>Eseye sends customer-specific Radius Accounting data as raw push messages to subscribed customers. Each record appears in the format described below.</p>

<h2 id="Rawpushmessagejson">Raw push message/json</h2>

<h3>Schema</h3>

<table> <col /> <col /> <col /> <thead> <tr> <th>Object name</th> <th>Type</th> <th>Description</th> </tr> </thead> <tbody> <tr> <td>acctStartDateTime</td> <td>DateTime</td> <td> Date and time the session started. <br />UTC format: YYYY-MM-DDThh:mm:ss.ssssss, where T is the time separator. </td> </tr> <tr> <td>acctStopCause</td> <td>string</td> <td> The reason the session terminated: <table> <col /> <col /> <col /> <thead> <tr> <th>Code</th> <th>Termination cause</th> <th>Description</th> </tr> </thead> <tbody> <tr> <td>1</td> <td>User request</td> <td>User initiated the termination by logging off.</td> </tr> <tr> <td>2</td> <td>Lost Carrier</td> <td>The port or modem dropped the Data Carrier Detect (DCD) signal.</td> </tr> <tr> <td>3</td> <td>Lost Service</td> <td>The service is unavailable for continued provision. Connection interruptions are the most likely cause.</td> </tr> <tr> <td>4</td> <td>Idle Timeout</td> <td>The idle timer expired for the user or port.</td> </tr> <tr> <td>5</td> <td>Session Timeout</td> <td>The subscriber reached the maximum continuous time limit allowed for the service or session.</td> </tr> <tr> <td>6</td> <td>Admin Reset</td> <td>The system administrator reset the port or session.</td> </tr> <tr> <td>7</td> <td>Admin Reboot</td> <td>The system administrator reset the hardware required to continue the connection, for example, by rebooting the Network Access Server (NAS).</td> </tr> <tr> <td>8</td> <td>Port Error</td> <td>The NAS detected an error on the port, which required ending the session.</td> </tr> <tr> <td>9</td> <td>NAS Error</td> <td>The NAS detected an error (other than on the port), which required ending the session.</td> </tr> <tr> <td>10</td> <td>NAS Request</td> <td>The NAS ended the session for a non-error reason.</td> </tr> <tr> <td>11</td> <td>NAS Reboot</td> <td>The NAS ended the session unexpectedly to reboot.</td> </tr> <tr> <td>12</td> <td>Port Unneeded</td> <td>The NAS ended the session because the resource usage was beneath the low threshold. For example, the bandwidth-on-demand algorithm determined that the port was no longer needed.</td> </tr> <tr> <td>13</td> <td>Port Preempted</td> <td>The NAS ended the session to allocate the port for higher-priority use.</td> </tr> <tr> <td>14</td> <td>Port Suspended</td> <td>The NAS ended the session to suspend a virtual session.</td> </tr> <tr> <td>15</td> <td>Service Unavailable</td> <td>The NAS was unable to provide the requested service.</td> </tr> <tr> <td>16</td> <td>Callback</td> <td>The NAS ended the session in order to perform callback for a new session.</td> </tr> <tr> <td>17</td> <td>User Error</td> <td>A client input error ended the session.</td> </tr> <tr> <td>18</td> <td>Host Request</td> <td>The login host ended the session. While this usually indicates normal termination of a login session, the remote host may have failed or become unreachable.</td> </tr> </tbody> </table> </td> </tr> <tr> <td>acctStopDateTime</td> <td>DateTime</td> <td> Date and time the session ended. "" indicates the session is ongoing and has not ended. <br />UTC format: YYYY-MM-DDThh:mm:ss.ssssss, where T is the time separator. </td> </tr> <tr> <td>acctType</td> <td>string</td> <td> There are three accounting types, indicating the session state on the network: <br /><span>AccountingStart</span> – when the session starts with an authentication request. <br /><span>AccountingInterim</span> – when the network operator sends updates, such as session length, number of bytes and so on. The first interim request is usually sent 30 minutes after a start request. You may observe multiple <span>AccountingInterim</span> accounting types, or none at all, depending on the session length. <br /><span>AccountingStop</span> – when the device disconnects from the network. <span><strong>Note: </strong>You can subscribe to one or more accounting types. </span> </td> </tr> <tr> <td>apn</td> <td>string</td> <td>The Access Point Name the device used to access the IP network.</td> </tr> <tr> <td>bytes</td> <td>string</td> <td>The total amount of data sent and received by the device during the reported session, in bytes.</td> </tr> <tr> <td>eventDateTime</td> <td>DateTime</td> <td> Date and time the data capture started. <br />UTC format: YYYY-MM-DDThh:mm:ss.ssssss, where T is the time separator. </td> </tr> <tr> <td>expiryDateTime</td> <td>DateTime</td> <td> The timeout value configured for the Radius Accounting cache by the network. <br />UTC format: YYYY-MM-DDThh:mm:ss.ssssss, where T is the time separator. </td> </tr> <tr> <td>iccId</td> <td>string</td> <td>The unique Integrated Circuit Card Identifier of the specific SIM used by the device.</td> </tr> <tr> <td>imsi</td> <td>string</td> <td>  International Mobile Subscriber Identity number. For more information, see: <a href="https://en.wikipedia.org/wiki/Mobile_country_code">Mobile network codes listed by country</a>. </td> </tr> <tr> <td>messageSequenceNumber</td> <td>string</td> <td>A network-generated unique reference for the payload.</td> </tr> <tr> <td>mnoId</td> <td>string</td> <td>The Mobile Network Operator Identifier.</td> </tr> <tr> <td>msisdn</td> <td>string</td> <td>The unique primary SIM phone number (Mobile Station International ISDN Number).</td> </tr> <tr> <td>notificationClass</td> <td>string</td> <td> The data class, to differentiate between the types of data pushed by the Push API. These are <span>netflow</span>, <span>radiusauth</span> and <span>radiusacct</span>. </td> </tr> <tr> <td>outputOctets</td> <td>string</td> <td>The total amount of data sent in sets of eight bits by the device during the reported session.</td> </tr> <tr> <td>payload</td> <td>array</td> <td>The array, listing the network data for an individual device.</td> </tr> <tr> <td>portfolioId</td> <td>string</td> <td>The SIAM account portfolio identifier. Contains exactly 32 characters.</td> </tr> <tr> <td>simId</td> <td>string</td> <td>The SIM's unique serial number.</td> </tr> <tr> <td>userName</td> <td>string</td> <td>The login user who started the session.</td> </tr> </tbody> </table>
