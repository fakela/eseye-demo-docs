<p>Read AnyNet Secure SIM files using <span>AT+CRSM</span>. For more information, see <a href="/sim-only/at-commands/sim-only-crsm">AT+CRSM – reading files from the SIM</a>.</p>

<h2 id="Fileformat">File format</h2>

<p>All files follow the same basic format:</p>

<p><span>&lt;length&gt;&lt;checksum&gt;&lt;package&gt;</span> </p>

<p>where:</p>

<table> <col /> <col /> <col /> <thead> <tr> <th>Variable</th> <th>Parameter</th> <th>Description</th> </tr> </thead> <tbody> <tr> <td><span>length</span> </td> <td> 2-byte unsigned 16-bit integer</td> <td><span>package</span> byte length, part of the file header</td> </tr> <tr> <td><span>checksum</span> </td> <td>4-byte unsigned 32-bit integer, stored by most significant byte first</td> <td><span>package</span> CRC-32 checksum, part of the file header</td> </tr> <tr> <td><span>package</span> </td> <td>Binary data</td> <td>The data field. Encryption keys and certificates may require some post-processing. For information about file location and post-processing requirements, see <a href="#Availabl">Available files and sizes</a>.</td> </tr> </tbody> </table>

<h2 id="Availablefilesandsizes">Available files and sizes</h2>

<p>Eseye delivers the following files to your AnyNet Secure SIM. <span>File Max Size</span> indicates the maximum amount of data available. The file header describes the length of the actual data.</p>

<p><span>Usable Bytes</span> refers to the number of bytes available after the header is accounted for. For more information, see <a href="/sim-only/sim-only-basic-read-process#Reading">Reading the file header</a>.</p>
