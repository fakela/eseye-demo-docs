<p>Repeat the basic read data process for each file that Eseye delivers to the AnyNet Secure SIM. For more information, see <a href="/sim-only/sim-only-reading-files#Availabl">Available files and sizes</a>.</p>

<p>Use the following process actions: </p>

<p>Read file header using the block read function</p>

<p>If ((HeaderLength!= 0) and (HeaderLength!= 0xFFFF) and (HeaderChecksum!= 0) and (HeaderChecksum!= 0xFFFFFFFF)) then </p>

<p>&#123; </p>

<p>// FILE ready to read </p>

<p>Read file data in chunks() // using calls to block read function() </p>

<p>Verify checksum () </p>

<p>File Format Conversion () // convert DER to PEM as required </p>

<p>&#125; </p>

<p>else </p>

<p>&#123; </p>

<p>// FILE not found – random back-off and try again </p>

<p>&#125;</p>

<h2 id="Readblockfunction">Read block function</h2>

<Tip>We recommend that you implement a single function to request a block read of file data from the AnyNet Secure SIM.</Tip>

<Tip>For a description of the parameters, see <a href="/sim-only/at-commands/sim-only-crsm">AT+CRSM – reading files from the SIM</a>.</Tip>

<p>Use <span>AT+CRSM</span> to achieve a block read, with the following format:</p>

<p>AT+CRSM=&lt;command&gt;,&lt;fileid&gt;,(&lt;Offset&gt;/256),(&lt;Offset&gt; MOD 256),<br />&lt;readlength&gt;,&lt;data&gt;,&lt;pathid&gt;</p>

<p>where <span>data</span> is unused and therefore empty.</p>

<p>This command returns:</p>

<p>+CRSM: 144,0,"&lt;RETURNEDDATA&gt;"</p>

<p>where <span>"&lt;RETURNEDDATA&gt;"</span> is an ASCII hex string that you must return to binary form. </p>

<h4>Example</h4>

<p>To read the <span>Thing Name</span> file header into a buffer called <span>returned&#95;data</span>, call the read block function as follows: </p>

<p><span>output&#95;data</span> (byte pointer) = <span>returned&#95;data</span></p>

<ul> <li><span>Thing Name</span> <span>fileid</span>: <span>28640</span></li> <li><span>Offset</span>: <span>0</span></li> <li><span>readlength</span>: <span>6</span> bytes</li> </ul>

<p>AT+CRSM=176,28640,0,0,6,,"3F007FEE"</p>

<p>+CRSM: 144,0,"4D4150"</p>

<p>where <span>"4D4150"</span> is the <span>&lt;RETURNEDDATA&gt;</span> string for <span>"MAP"</span>. </p>

<Note>The first character in the data is the upper nibble of the first data byte. The second character is the lower nibble of the first data byte. You must convert each nibble into its hexadecimal data byte value.</Note>

<h3>Converting the returned data into binary</h3>

<p>You must convert the <span>"RETURNEDDATA"</span> ASCII hexadecimal string into binary. </p>

<h4>Example</h4>

<p>The following example processes the returned string, reading two ASCII characters and converting them to binary. The resultant byte value is stored at the location defined by <span>output&#95;data</span>:</p>

<p>index = 0;</p>

<p>while (index &lt; &lt;readlength&gt;)</p>

<p>&#123;</p>

<p>sscanf( &amp;data[index], “%02X”, &#42;output&#95;data++);</p>

<p>index = index + 2</p>

<p>&#125;</p>

<h2 id="Readingthefileheader">Reading the file header</h2>

<p>The following example uses the read block function to read the <span>Thing Name</span> header, returning 6 bytes from offset 0. 6 bytes is the header size.</p>

<p>Call the read block function with the following parameters:</p>

<p><span>output&#95;data</span> = <span>returned&#95;data</span></p>

<ul> <li><span>Thing Name</span> <span>fileid</span>: <span>28640</span></li> <li><span>offset</span>: <span>0</span></li> <li><span>readlength</span>: <span>6</span> bytes</li> </ul>

<Note>If you use the suggested read block functionality, then the returned data is in binary format, not the ASCII hex returned by the modem.</Note>

<p>After the response data is received, you can extract the file header length (2 bytes) and checksum (4 bytes). For more information, see <a href="/sim-only/sim-only-reading-files#File">File format</a>.</p>

<p>Both values are big-endian. You may need to reverse the byte order.</p>

<p>datalength = &#42;(unsigned short &#42;)&amp;returned&#95;data[0] // read 16 bit length</p>

<p>checksum = &#42;(unsigned int &#42;)&amp;returned&#95;data[2] // read 32 bit checksum</p>

<p>If <span>datalength</span> is not equal to 0 and 0xFFFF, and the <span>checksum</span> is not equal to 0 and 0xFFFFFFFF, you can read the rest of the file.</p>

<h2 id="Readingthefileinchunks">Reading the file in chunks</h2>

<p>After reading the file header, you can read the data content, starting at offset 6. Read up to 255 bytes at a time. </p>

<p>Check the modem command manual, as shorter length limits per read may apply.</p>

<h3>Example</h3>

<p>The following example reads the <span>Root CA</span> file, of length 1239 bytes. </p>

<p>file&#95;data = malloc(1240); // allocate storage for the file read assembly</p>

<p>offset=6; // skip the header bytes</p>

<p>while (length)</p>

<p>&#123;</p>

<p>if (length &gt; 255)</p>

<p>bytes&#95;to&#95;read = 255;</p>

<p>else </p>

<p>bytes&#95;to&#95;read = length;</p>

<p>// Wait for the read block response and process the data nibbles</p>

<p>read&#95;block&#95;function(&amp;file&#95;data[offset – 6], 28640, offset, bytes&#95;to&#95;read)</p>

<p>length = length – bytes&#95;to&#95;read</p>

<p>offset = offset + bytes&#95;to&#95;read</p>

<p>// repeat if more data to read</p>

<p>&#125;</p>

<p>For a file with a data record of 927 bytes, this code results in the following call sequence:</p>

<p>read&#95;lock&#95;function(&amp;file&#95;data[0], 28641, 6, 255)</p>

<p>read&#95;lock&#95;function(&amp;file&#95;data[255], 28641, 261, 255)</p>

<p>read&#95;lock&#95;function(&amp;file&#95;data[510], 28641, 516, 255)</p>

<p>read&#95;lock&#95;function(&amp;file&#95;data[765], 28641, 771, 255)</p>

<p>read&#95;lock&#95;function(&amp;file&#95;data[1020], 28641, 1026, 219)</p>

<p>which in turn will issue the following AT commands:</p>

<p>AT+CRSM=176,28641,0,6,255,,”3F007FEE”</p>

<p>AT+CRSM=176,28641,1,5,255,,”3F007FEE”</p>

<p>AT+CRSM=176,28641,2,4,255,,”3F007FEE”</p>

<p>AT+CRSM=176,28641,3,3,255,,”3F007FEE”</p>

<p>AT+CRSM=176,28641,4,2,219,,”3F007FEE”</p>

<h2 id="Fileverification">File verification</h2>

<p>After all the data bytes are read from the AnyNet Secure SIM, you must verify them. Perform a CRC32 calculation for the data and compare it to the checksum in the header. If the sum matches, the file read is complete. Otherwise randomly backoff to wait until the file update completes, and then try again.</p>

<p>For more information, see <a href="/sim-only/crc32code-example">CRC-32 code example</a>.</p>

<h2 id="Fileformatconversion">File format conversion</h2>

<p>The following files are transferred in Binary DER-encoded / ASN.1 BER-encoded format:</p>

<ul> <li><span>Root CA</span> <span>fileid</span>: 28641</li> <li><span>Public signed certificate</span> <span>fileid</span>: 28642</li> <li><span>Private key</span> <span>fileid</span>: 28643</li> </ul>

<p>You may need to convert these into PEM or base64 format for use with the SSL/TLS client. Use base64 to encode the data and add header and footer lines.</p>

<h4>Example</h4>

<p>For the <span>Public signed certificate</span> file:</p>

<p>certificate = sprint( "-----BEGIN CERTIFICATE-----&#92;n")</p>

<p>offset = strlen( certificate);</p>

<p>cert&#95;offset = 0;</p>

<p>while (cert&#95;offset &lt; cert&#95;length )</p>

<p>&#123;</p>

<p>Length = (cert&#95;length – cert&#95;offset);</p>

<p>If (length &gt; 64) length = 64;</p>

<p>base64encode(encoded, certificate[cert&#95;offset], length);</p>

<p>cert&#95;offset += length;</p>

<p>Memcpy( certificate[offset], encoded, length);</p>

<p>Offset += length;</p>

<p>certificate[offset++] = '&#92;n';</p>

<p>&#125;</p>

<p>Memcpy( certificate[offset], "-----END CERTIFICATE-----&#92;n" )</p>
