<h2 id="ATCRSMreadingfilesfromtheSIM">AT+CRSM – reading files from the SIM</h2>

<p>Read files from the AnyNet Secure SIM. For precise formats and response details, refer to the modem provider documentation.</p>

<Note>We recommend that you use <span>AT+CRSM</span> over the generic SIM access <span>AT+CSIM</span>. <span>AT+CRSM</span> provides easier access to the SIM file system, as the modem facilitates the SIM interface locking and file selection routines.</Note>

<Note>The following command describes only those fields that are used and required for Eseye file extraction. Refer to <a href="ftp://www.3gpp.org/tsg_t/TSG_T/TSGT_04/Docs/PDFs/TP-99118.pdf">3GPP TS 27.007</a> for a complete definition.</Note>

<table> <col /> <col /> <col /> <thead> <tr> <th>Type</th> <th>Syntax</th> <th>Returned Result</th> </tr> </thead> <tbody> <tr> <td>Write</td> <td> <span>AT+CRSM=&lt;command&gt;[,&lt;<var>fileid</var>&gt;[,&lt;P1offset&gt;,&lt;P2offset&gt;,&lt;P3readlength&gt;[,&lt;<var>data</var>&gt;[,"&lt;<var>pathid</var>&gt;"]]]]</span>  <br />where <ul> <li><span>command</span> is the command passed on to the SIM. Use <span>176</span> – <span>READ BINARY</span> to extract security and identity information.</li> <li> <span>fileid</span> is the file identifier on the SIM, of type integer. <br />For information about which files are delivered, see <a href="/sim-only/sim-only-reading-files#Availabl">Available files and sizes</a>. </li> <li> <span>P<var>n</var>...</span> parameters are used for the <span>READ BINARY</span> command. <br /><span>AT+CRSM</span> cannot return more than 255 bytes in a single read. These fields enable the file to be read in chunks. <br />You can use the following arithmetic operators: (+, -, /, &#42;) <ul> <li><span>P1offset</span> is the integer value contained in the top 8 bits of the 16 bit file offset value, from which to start the file read.</li> <li><span>P2offset</span> is the integer value contained in the bottom 8 bits of the 16 bit file offset value, from which to start the file read. <span>P2</span> can include a modulus (MOD) operator.</li> <li> <span>P3readlength</span> is the read data length as an integer. Limit the read to 255 bytes per block. </li> </ul> </li> <li><span>data</span> is unused for <span>READ BINARY</span>. Leave empty.</li> <li><span>pathid</span> is the elementary file path on the SIM/UICC in hexadecimal format. Use <span>"3F007FEE"</span> to read files from the AnyNet Secure SIM.</li> </ul> </td> <td> +CRSM: &lt;sw1&gt;,&lt;sw2&gt;[,"&lt;RETURNEDDATA&gt;"]  <br />or <br />+CME ERROR: &lt;err&gt; <br />where: <ul> <li><span>sw1</span>,<span>sw2</span> contain information from the AnyNet Secure SIM about the command execution. These may include a success indication or specific file read errors.</li> <li><span>"RETURNEDDATA"</span> contains the requested data for a successful read, in ASCII hexadecimal string format. For information about converting this string into binary, see <a href="/sim-only/sim-only-basic-read-process#Converti">Converting the returned data into binary</a>.</li> <li><span>err</span> is the error result code response when the command cannot pass to the SIM. To understand the value, refer to the <span>CME ERROR</span> response in the modem provider documentation.</li> </ul> </td> </tr> </tbody> </table>
