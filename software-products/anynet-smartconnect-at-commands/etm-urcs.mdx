<p>AnyNet SMARTconnect™ adds the following Eseye URCs to the module. </p>

<p>You will continue to observe other URCs from the module. </p>

<Note>For information about all other URCs, see the relevant module documentation. </Note>

<table> <col /> <col /> <thead> <tr> <th>URC</th> <th>Description</th> </tr> </thead> <tbody> <tr> <td> +ETM:EMQRDY </td> <td> ETM has entered MQTT mode and is ready to accept any <span>AT+EMQ...</span> commands. Prior to this, only <span>AT+ETM... </span>commands are accepted. </td> </tr> <tr> <td> +ETM:IDLE </td> <td> The modem has started up and is ready for commands. </td> </tr> <tr> <td> +ETM:REBOOTING </td> <td> The modem will shortly restart. Wait for <span>+ETM:IDLE</span> before sending commands. Configure automatic rebooting using <span>update&#95;autoreboot</span> in the configuration file. For more information, see <a href="/software-products/configuration-file">Using the AnyNet SMARTconnect™ configuration file</a>. </td> </tr> <tr> <td> +ETM: REBOOT REQUIRED </td> <td> The modem has received an update and needs to restart. While operation can continue as normal, the host should issue <span>AT+ETMRESET</span> at a convenient time. This restarts the device and applies the update. <br />Configure automatic rebooting using <span>update&#95;autoreboot</span> in the configuration file. For more information, see <a href="/software-products/configuration-file">Using the AnyNet SMARTconnect™ configuration file</a>. </td> </tr> <tr> <td> +ETMSTATE:&lt;StateID&gt; <br />where <span>&lt;StateID&gt;</span> is the current connectivity state of the module. </td> <td> Enable this response using the <span>AT+ETMSTATE</span> write command. For more information, see <a href="/software-products/anynet-smartconnect-at-commands/etmstate">ETMSTATE – check current state</a>. </td> </tr> <tr> <td> +ETM:SYSSTART </td> <td>Indicates that AnyNet SMARTconnect™ application has started.</td> </tr> </tbody> </table>
