<p>Checks the current state of the LTE IoT 2 click.</p>

<table> <col /> <col /> <col /> <thead> <tr> <th>Type</th> <th>Syntax</th> <th>Response</th> </tr> </thead> <tbody> <tr> <td>Test</td> <td> <span>AT+ETMSTATE=?</span>  </td> <td> +ETMSTATE: <br />0: Idle <br />1: Awaiting keys <br />2: Network connecting <br />3: Establishing TLS <br />5: Establishing data-link <br />6: Connected <br />7: Subscribed <br />8: UDP active <br />9: Error <br />OK <br />where the modem: <br /><span>0</span> – is ready to receive commands. <br /><span>1</span> – has registered to the network and is waiting for certificate delivery to complete on the SIM. <br /><span>2</span> – is waiting to establish a network connection. <br /><span>3</span> – is establishing end-to-end security of data, using the Transport Layer Security protocol. <br /><span>4</span> – this state is currently unused. <br /><span>5</span> – is establishing a connection to AWS in order to transfer data. <br /><span>6</span> – has connected to the MQTT broker. <br /><span>7</span> – is subscribed to MQTT topics. <br /><span>8</span> – <span>User Datagram Protocol</span> is ready for use. <br /><span>9</span> – the device or connections have failed. </td> </tr> <tr> <td>Read</td> <td> <span>AT+ETMSTATE?</span>  </td> <td> +ETMSTATE: <var>&lt;state&gt;</var> <br /> <br />  <br />OK <br />or <br />ERROR </td> </tr> <tr> <td> Write </td> <td> <span>AT+ETMSTATE=&lt;<var>cmd</var>&gt;</span>  <br />where <span>&lt;<var>cmd</var>&gt;</span> is the command, either: <br /><span>0</span> – disable state change URC <br /><span>1</span> – enable state change URC <br /><span>"powerdown"</span> – turn off modem when it is not in the <span>busy</span> state <br />For information about the <span>busy</span> state, see <a Use this to cleanly shutdown before power off. </td> <td> OK <br /> After <span>AT+ETMSTATE="stop"</span>, AnyNet SMARTconnect™ returns to idle and the following response occurs:  <br />+ETM: IDLE <br />or <br />+ERROR: &lt;errormessage&gt; </td> </tr> </tbody> </table>
