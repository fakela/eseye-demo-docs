title: "+EMQ Unsolicited Response Codes (URCs)"
description: "When the MQTT broker publishes data to your thing, AnyNet SMARTconnect™ forwards the data through the AT command interface using an appropriate URC."

<p>When the MQTT broker publishes data to your thing, AnyNet SMARTconnect™ forwards the data through the AT command interface using an appropriate URC.</p>

<p>If the data buffer contains only printable characters, it is presented unmodified. This makes transfer of JSON and ASCII text transparent. If non-printable characters are contained in the buffer, the entire buffer is converted to ASCII-hex and sent within quotes.</p>

Indicates data received on a single topic. For information about singletopic, see <a href="/software-products/anynet-smartconnect-at-commands/emq">EMQ – publish a message to singletopic</a>. </li> <li><span>&lt;id&gt;</span> is the subscribed topic index number, not for singletopic use</li> <li><span>&lt;len&gt;</span> is the length of the incoming data </li> <li> <span>&lt;data&gt;</span> is the received data in either ASCII-hex format or as text, depending on both the <span>urcautoformat</span> setting in the configuration file, and if <span>&lt;data&gt;</span> appears within quotes. <br />For example, if the <span>urcautoformat</span> setting = <span>1</span>, then: <ul> <li><span>&lt;id&gt;,&lt;len&gt;&#92;r&#92;n"&lt;data&gt;"</span> reports data in ASCII-hex format</li> <li><span>&lt;id&gt;,&lt;len&gt;&#92;r&#92;n&lt;data&gt;</span> reports data in text format</li> </ul> <br />For more information, see <a href="/software-products/configuration-file#urcautof">Using the AnyNet SMARTconnect™ configuration file</a>. </li> </ul> </td> </tr> <tr> <td> +EMQPERSIST: &lt;len&gt; <br />  <br />&lt;JSON&gt; </td> <td> Reports a message from the cloud service indicating something in the persistence service has changed. <br />For <span>simple</span> AWSSHADOW mode, only the <span>'&#123;"state":&#123;...'</span> parameters that have changed in the AWS delta are listed. <br />For example: <br />+EMQPERSIST: &#123;"key1":"value1","key2":"value2"&#125; <br />where <span>value1</span> and <span>value2</span> are any type of value, including strings, integers, decimals and negative numbers. <br />For <span>complete</span> AWSSHADOW mode, the entire JSON is listed. <br />For example: <br />+EMQPERSIST:&#123;"version":186201,"timestamp":1573232068,"state":&#123;"key1":"value1","key2":"value2"&#125;,"metadata":&#123;"key1":&#123;"timestamp":1573232068&#125;,"key2":&#123;"timestamp":1573232068&#125;&#125;&#125; <br />where <span>value1</span> and <span>value2</span> are numeric values, including decimals and negative numbers. </td> </tr> </tbody> </table>
