---
title: "ETMHFWREAD – reads a section of the new host firmware image"
description: "Reads consecutive sections of the new firmware image. You can set the size of each section."
---

<p>Reads consecutive sections of the new firmware image. You can set the size of each section. </p>

<p>If a host firmware image was previously downloaded using <span>AT+ETMHFWGET</span>, it is stored in flash memory on the module. For information about <span>AT+ETMHFWGET</span>, see <a href="/software-products/anynet-smartconnect-at-commands/etmhfwget">ETMHFWGET – checks for new host firmware</a>.</p>

<table> <col /> <col /> <col /> <thead> <tr> <th>Type</th> <th>Syntax</th> <th>Response</th> </tr> </thead> <tbody> <tr> <td>Test</td> <td> <span>AT+ETMHFWREAD=?</span>  </td> <td> +ETMHFWREAD: (0-4294967295),(0-1024) <br />  <br />OK </td> </tr> <tr> <td>Read</td> <td> <span>AT+ETMHFWREAD?</span>  </td> <td> <span>+ETMHFWREAD: &lt;imagesize&gt;,&lt;checksum&gt;</span>  <br />  <br />OK <br />where:  <ul> <li><span>imagesize</span> is the size of the host firmware image in bytes</li> <li><span>checksum</span> is the 16-bit XOR checksum of the entire image in hexadecimal format</li> </ul> <span><strong>Note: </strong>If both imagesize and checksum are 0, no firmware image exists in the flash memory.</span> <span><strong>Note: </strong>After reading the whole image, the MCU application should run a checksum to compare its copy and the one supplied by AnyNet SMARTconnect™. This ensures the local copy was accurately copied from AnyNet SMARTconnect™.</span> <span><strong>Note: </strong>The checksum is always four characters long, with leading zeros where required.</span> <br />or <br />ERROR </td> </tr> <tr> <td> Write </td> <td> <div> <span>AT+ETMHFWREAD=&lt;offset&gt;,&lt;length&gt;</span>  <br />where: <ul> <li><span>offset</span> is the number of offset bytes (from 0 to 4294967295 bytes)</li> <li><span>length</span> is the number of bytes read (from 0 to 1024 bytes)</li> </ul> <br /><span>offset</span> and <span>length</span> are required parameters. </div> </td> <td> +ETMHFWREAD: &lt;hexdata&gt; <br />  <br />OK <br />where <span>&lt;hexdata&gt;</span> is the returned data in hexadecimal <br />or <br /><span>+ETM ERROR: &lt;errormessage&gt;</span> – returned for an AT command-specific error <br />or <br /><span>ERROR</span> – returned when a command is invalid, for example a parameter is missing or incorrect </td> </tr> </tbody> </table>

<h4>Example</h4>

<p>To read 46 bytes in 16 byte chunks: </p>

<p>AT+ETMHFWREAD=0,16</p>

<p>+ETMHFWREAD: &lt;hexdata&gt;</p>

<p> </p>

<p>OK</p>

<p>AT+ETMHFWREAD=16,16</p>

<p>+ETMHFWREAD: &lt;hexdata&gt;</p>

<p>OK</p>

<p>AT+ETMHFWREAD=32,14</p>

<p>+ETMHFWREAD: &lt;hexdata&gt;</p>

<p>OK</p>
