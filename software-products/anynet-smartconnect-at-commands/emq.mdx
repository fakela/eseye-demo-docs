<p>Publish a message to the system <span>singlepubtopic</span>. <span>singlepubtopic</span> is a predefined topic for single-subscription systems, where the host is not expected to register a publish topic with <span>+EMQPUBOPEN</span>. The publish topic (<span>singlepubtopic</span>) and QoS (<span>singlepubtopicqos</span>) are defined in the configuration file. For more information, see the [MQTT] section, <a href="/software-products/configuration-file">Using the AnyNet SMARTconnect™ configuration file</a>. The index is not required.</p>

<table> <col /> <col /> <col /> <thead> <tr> <th>Type</th> <th>Syntax</th> <th>Returned Result</th> </tr> </thead> <tbody> <tr> <td>Test</td> <td><span>AT+EMQ=?</span> </td> <td>  +EMQ: &lt;pubdata&gt; <br />  <br />OK  </td> </tr> <tr> <td>Read</td> <td><span>AT+EMQ?</span> </td> <td> OK <br />or <br />ERROR </td> </tr> <tr> <td>Write</td> <td> <span>AT+EMQ=&lt;pubdata&gt;</span>  <br />where <br /><span>&lt;pubdata&gt;</span> – is the published message. If <span>&lt;pubdata&gt;</span> is not contained within quotes, it is sent as text with no modification. The maximum payload length is 1000 characters. All characters must be printable.  <br />If <span>"&lt;pubdata&gt;"</span> is contained within quotes, it is handled as ASCII-hex. If <span>"&#123;&lt;pubdata&gt;"</span> includes an open brace immediately after the inital quote, it is handled as a JSON string.  <span><strong>Note: </strong>AnyNet SMARTconnect™ does not verify the JSON string validity.</span> <br />AnyNet SMARTconnect™ handles <span>"</span><span>&lt;pubdata&gt;</span><span>"</span> as ASCII-hex if it contains an even number of valid ASCII-hex characters (<span>0</span>-<span>9</span>, <span>a</span>-<span>f</span>, <span>A</span>-<span>F</span>). ASCII-hex is converted to binary for transmission. <br />For JSON messages, <span>&#92;</span> (escape characters) are removed.  </td> <td> <span>OK</span> – command result <br />  <br /><span>:SEND OK</span> – subsequent URC that is sent when publish occurs. This will not happen immediately if AnyNet SMARTconnect™ is offline. <br />or <br /><span>+ETM ERROR: &lt;errormessage&gt;</span> – returned for an AT command-specific error. Check that <span>[MQTT] singlepubtopic</span> is configured in the configuration file. For more information, see <a href="/software-products/configuration-file#singlepubtopic">singlepubtopic</a>. <br />or <br /><span>ERROR</span>  – returned when a command is invalid, for example a parameter is missing or incorrect  </td> </tr> </tbody> </table>

<h4>Example</h4>

<p>AT+EMQ="&#123;&#92;"BatteryPower&#92;": &#92;"Low&#92;"&#125;"</p>
