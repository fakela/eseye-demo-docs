title: "Sending data from your thing to the cloud"
description: "To test that your thing can publish information to the cloud:"
---

<div> <h3>Before you begin</h3> <ul> <li> Using a terminal emulator, send an <span>AT&lt;CR&gt;</span> command to ensure that the smart terminal can receive AT commands. For more information, see <a href="/software-products/general-at-commands/at-commands">General AT Commands</a>. <span><strong>Note: </strong>For information about connecting a terminal emulator to the modem, see Connecting to the LTE IoT 2 click.</span> </li> <li> Send an <span>AT+ETMINFO=version</span> command to verify that the version number is <span>AnyNet SMARTconnect™ V0.99&#95;ma</span> or higher. <br />If the version number is not in this range, you will need to update the modem software. For more information, see <a href="/software-products/over-the-air-updates">Over-the-air (OTA) updates</a>. </li> <li> Send an <span>AT+ETMSTATE="startmqtt"</span> command to enable the MQTT protocol </li> <li> Ensure you know the thing name that you set up in the cloud. </li> </ul> </div>

<Note>The example below uses AWS. The current AWS interface may differ slightly from the one we used in the example.</Note>

<p>To test that your thing can publish information to the cloud:</p>

<ol> <li> Create two publish topics in the module. <ol> <li> Using a terminal emulator, type:  <br /><span>at+emqpubopen=0,"PublishToCloud0"</span>&lt;CR&gt; <br /><span>at+emqpubopen=1,"PublishToCloud1"</span>&lt;CR&gt; </li> <li> <div> Check that the first two index numbers are assigned a topic each. Type: <br /><span>at+emqpubopen?</span>&lt;CR&gt; </div> <br />A list of index numbers and their assigned topics appears. <br /><img src="/images/resources/images/emqpubopenbg96.png" alt="Emqpubopenbg96" />  </li> </ol> </li> <li> Subscribe to the newly created publish topics using your cloud provider's console.  <ol> <li> Using<span>AWS IoT</span>, in the left hand menu, select <span>Test</span> to open the <span>MQTT test client</span>, then ensure you are on the <span>Subscribe to a topic</span> tab. <br /><img src="/images/resources/images/aws_subscribe1.png" alt="Aws Subscribe1" />  </li> <li> In the<span> Topic filter</span> box, type: <br /><span>+/&lt;<i>ThingName</i>&gt;</span>  <br />This subscribes AWS to all topics related to your thing. </li> <li> Select <span>Subscribe</span>. <br /><img src="/images/resources/images/aws_subscribe2.png" alt="Aws Subscribe2" />  <br />A subscription appears, listed in the Subscriptions panel. </li> </ol> </li> <li> Publish information to the topics you created in the module. <span><strong>Note: </strong>You can send a maximum payload of 1000 characters to AWS.</span> <ol> <li> Using the terminal emulator, type: <br /><span>at+emqpublish=0,1,"&#123;&#92;"Temperature&#92;": 24&#125;"</span>&lt;return&gt; <br /><span>at+emqpublish=1,1,"&#123;&#92;"BatteryPower&#92;": &#92;"Low&#92;"&#125;"</span>&lt;return&gt; <br />The emqpublish command uses the following syntax: <br /><span>at+emqpublish=<i>IndexNumber</i>,<i>QoS</i>,</span>  <br /><span>"&#123;<i>PublishDataInJSON</i>&#125;"</span>  <br /><img src="/images/resources/images/emqpublish.png" alt="Emqpublish" />  <br />These messages instantly appear in the cloud. </li> </ol> </li> <li> View the published information in the cloud. <ol> <li> In the <span>Subscriptions</span> panel, select <span>+/&lt;<i>thingName</i>&gt;</span> to view all published messages. <br /><img src="/images/resources/images/subscriptionmessages.png" alt="Subscriptionmessages" />  </li> </ol> </li> </ol>

<Tip>If you can see the messages in the cloud, then your thing can successfully publish data into the cloud through the module.</Tip>

<p>Next, test that the cloud can publish messages to your thing. </p>
