<p>AT commands (Hayes command set) are instructions for controlling modules. The commands consist of a series of short text strings for a range of uses, including: </p>

<ul> <li>Configuring a module</li> <li>Establishing the module's network connection</li> <li>Discovering module and connection status information, to ensure they are working correctly</li> <li>Managing the module</li> <li>Encrypting and transporting data to and from the module</li> <li>Directly accessing the cloud</li> </ul>

<p>Your software must send test, read and write AT commands to your module.</p>

<Tip>Search the module supplier documentation for a full set of the AT commands you can use on your module. For example, Quectel supply the <a href="https://www.quectel.com/wp-content/uploads/2021/03/Quectel_BG95BG77BG600L_Series_AT_Commands_Manual_V2.0-3.pdf" target="_blank">BG95&amp;BG77&amp;BG600L Series AT Commands Manual (PDF)</a>. (Note, this link may not point to the latest version).</Tip>

<p>AnyNet SMARTconnect™ AT commands extend the available AT commands on your AnyNet SMARTconnect™-enabled module.</p>

<div> <h3>Before you begin</h3> <p>Before you use any AT commands on the module, ensure that it is ready to receive them. </p> <p>To test that the module is ready to receive AT commands:</p> <ul> <li> Using a terminal emulator that is connected to your module, type: <br /><span>at</span>&lt;return&gt; <br />The terminal emulator will return any of the following: <ul> <li><span>OK</span> – the module and port are connected and ready to communicate</li> <li><span>ERROR</span> – the module and port cannot communicate. Contact the modem supplier.</li> <li>Nothing – ensure you have set the correct baud rate in your code. For more information, see <a href="/quectel/rs232-serial-connection" target="_blank">Connecting to the Quectel module using a terminal emulator</a>.</li> </ul> </li> </ul> <p>If you are using AnyNet SMARTconnect™, ensure that it is correctly installed on the modem. For more information, see <a href="/quectel/installing-smartconnect-bg9x" target="_blank">Installing AnyNet SMARTconnect™ on a Quectel BGxx module</a>. </p> <p>To test that AnyNet SMARTconnect™ is ready to receive AnyNet SMARTconnect™ AT commands:</p> <ul> <li> Using a terminal emulator that is connected to your module, type: <span>at+etminfo=version</span>&lt;return&gt; <br />The terminal emulator will return either of the following: <ul> <li> <span>AnyNet SMARTconnect™ V&lt;<var>version</var>&gt;</span>, where &lt;<var>version</var>&gt; is the current software version – the module is ready to receive AnyNet SMARTconnect™ AT commands.  <br />Verify that the version number is <span>AnyNet SMARTconnect™ V0.99&#95;ma</span> or higher. </li> <li> <span>ERROR</span> – AnyNet SMARTconnect™ software has not yet initialised. Try again in 5 seconds.  </li> </ul> </li> </ul> </div>

<h2 id="ATCommandsyntax">AT Command syntax</h2>

<p>Use the following syntax:</p>

<p>AT+&lt;COMMAND&gt;&lt;CR&gt;</p>

<p>where: </p>

<ul> <li><span>AT</span> is in upper or lowercase</li> <li><span>&lt;COMMAND&gt;</span> is a test, read or write command in upper or lower case</li> <li> <span>&lt;CR&gt;</span> is the end-of-line character marking the end of a command line (alias &#92;r <span>– carriage return)</span> <br />The modem will execute the command line after receiving the end-of-line character. </li> <li><span>&lt;LF&gt;</span> is the line feed, which will move the cursor to the next line</li> </ul>

<Note>This document displays commands only. <span>&lt;CR&gt;&lt;LF&gt;</span> after a command is intentionally omitted.</Note>

<p>AT commands are usually followed by a response that includes:</p>

<p>&lt;CR&gt;&lt;LF&gt;&lt;RESPONSE&gt;&lt;CR&gt;&lt;LF&gt;</p>

<p>where <span>&lt;RESPONSE&gt;</span> is the command response</p>

<Note>This document displays responses only. <span>&lt;CR&gt;&lt;LF&gt;</span> is intentionally omitted.</Note>

<p>The response may include:</p>

<ul> <li><span>OK</span> – indicates the command executed with no errors</li> <li><span>ERROR</span> – indicates an invalid command, or that the command line was too long</li> </ul>

<h2 id="TypesofATCommandsandresponses">Types of AT Commands and responses</h2>

For example, see <a href="/software-products/general-at-commands/ccid">CCID – request unique SIM number (ICCID)</a>. </td> </tr> </tbody> </table>
