<p>When AnyNet SMARTconnect™ is subscribed to a topic and receives new MQTT messages, it can perform either of the following functions:</p>

<ul> <li> Immediately forward the messages on the AT port in URC format – <span>+EMQ: &lt;id&gt;,&lt;len&gt;,&lt;data&gt;</span> <br />where:  <br /><span>id</span> – is the message index number <br /><span>len</span> – is the message length in bits <br /><span>data</span> – is the forwarded MQTT message </li> <li> If the <span>MQTT Rx Queue</span> is enabled, store up to three MQTT messages in a queue in the module’s volatile memory. <br />If AnyNet SMARTconnect™ receives further messages, it overwrites the existing messages, starting with the oldest. <br />Messages are stored in the following format – <span>+EMQREAD: &lt;id&gt;&lt;len&gt;&#92;r&#92;n&lt;data&gt;</span> <span><strong>Note: </strong>Use the <span>[mqtt] mqttenablerxqueue</span> parameter in the configuration file to enable or disable the <span>MQTT Rx Queue</span>. For more information, see <a href="/software-products/configuration-file#mqttenablerxqueue">mqttenablerxqueue</a>.</span> </li> </ul>

<h2 id="StatusFlags">Status Flags</h2>

<p>If the <span>MQTT Rx Queue</span> is enabled, you can use the following status flags to obtain information about the internal status of AnyNet SMARTconnect™:</p>

<ul> <li><span>MQTT RX</span> – set when messages appear in the <span>MQTT Rx Queue</span>, cleared when no messages exist.</li> <li><span>MQTT RXov</span> – set when the oldest received message is overwritten because of overflow, cleared when the status flags are read.</li> <li> <span>MQTT TX Internal error</span> – set when AnyNet SMARTconnect™ is unable to save the MQTT message in the non-volatile memory, cleared when the status flags are read. </li> <li><span>MQTT TXov</span> – set when the oldest transmitted message is overwritten because of overflow, cleared when the status flags are read.</li> <li> <span>MQTT TX OK</span> – set when the transmitted message is successfully published, cleared when the status flags are read. </li> <li> <span>MQTT TX Refused</span> – set when the MQTT broker rejects the transmitted MQTT message, or closes the SSL connection five consecutive times after each publication attempt. Cleared when the status flags are read. </li> <li> <span>Reboot required</span> – set when a reboot is required in order to apply an update, cleared when AnyNet SMARTconnect™ resets. </li> <li> <span>Update fail</span> – set when a host firmware, configuration, or AnyNet SMARTconnect™ update fails, cleared when the status flags are read. </li> <li><span>Update success</span> – set when an update completes successfully for host firmware, configuration, AnyNet SMARTconnect™, or AnyNet SMARTconnect™ and modem firmware, cleared when the status flags are read.</li> </ul>

<Note>Check the status flags using the <span>AT+ETMINFO</span> command. For more information, see <a href="/software-products/anynet-smartconnect-at-commands/etminfo">ETMINFO – displays AnyNet SMARTconnect™ and device information</a>.</Note>
