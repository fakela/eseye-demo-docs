<p>AnyNet SMARTconnectâ„˘ is a suite of software tools that establishes and maintains an IoT device's connection to a cellular network, and simplifies securely connecting the device to cloud services. This includes:</p>

<ul> <li>Persistent storage of published data until connection is established and data delivery completes</li> <li>Bi-directional encryption and transportation of data</li> <li>End-to-end connection monitoring and active network management between the device and the cloud</li> <li>Network performance and reliability measured at the data transport layer</li> <li>Simplified over-the-air (OTA) updates to customer software or firmware, the modem firmware, and AnyNet SMARTconnectâ„˘</li> <li> Seamless migration if you need to change cloud providers </li> </ul>

<Note>AnyNet SMARTconnectâ„˘ supports the MQTT protocol. </Note>

<h2 id="AnyNetSMARTconnectfeatures">AnyNet SMARTconnectâ„˘ features</h2>

<h3>Cellular connection management</h3>

<ul> <li>Automatic network selection</li> <li>Automatic network establishment</li> <li> Connection monitoring and maintenance, which includes: <ul> <li> Combined Session, Network and Physical Layer connection monitoring </li> <li> Alternate path selection in conjunction with the AnyNet SIM </li> <li> Proactive intervention to restore network connection </li> <li> Network Friendly Mode operation </li> </ul> </li> </ul>

<h3>Cloud connection management</h3>

<ul> <li>Automatic cloud provider configuration based on service endpoint URL</li> <li>MQTT data transport publish and subscribe with up to 8 topics each</li> <li> Data storage/queue for MQTT QOS 1 messages, including: <ul> <li>Configurable ON /OFF to allow host to queue</li> <li>Data persisted over power cycle and network re-establishment</li> <li>Data queued until MQTT ACK (acknowledgement) received, or persistent error</li> </ul> <ul> <li>Automatically delivered or queued subscribed data</li> <li>Auto-connect mode for configured single publish/subscribe topic solutions</li> </ul> </li> </ul>

<h3>Data encryption</h3>

<ul> <li>Transport Layer Security (TLS) encryption offload, data is encrypted on the module</li> <li>Certificate storage on the module</li> <li>AnyNet Secure enabled OTA certificate installation and update</li> <li>Post-build and deployment certificate provisioning</li> </ul>

<h3>Simplified OTA updates</h3>

<ul> <li>Customer application updates</li> <li>Cellular module firmware updates</li> <li>AnyNet SMARTconnectâ„˘ updates</li> <li>AnyNet SMARTconnectâ„˘ configuration updates</li> </ul>

<h3>System interface</h3>

<ul> <li>Standard URC support (interrupt mode)</li> <li>Non-Interrupt (Polled) mode data receive and status notification support</li> <li>Configurable input/output for interrupt/wakeup signal</li> </ul>
