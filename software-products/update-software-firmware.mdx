<h2 id="AboutAnyNetSMARTconnectupdate">About AnyNet SMARTconnect‚ĄĘ update</h2>

<p>Each AnyNet SMARTconnect‚ĄĘ software version is developed for a specific Quectel BG95 module firmware version. When an AnyNet SMARTconnect‚ĄĘ update occurs, the older version remains stored in the <span>../datatx</span> folder. </p>

<p>At each boot, AnyNet SMARTconnect‚ĄĘ checks that the currently running AnyNet SMARTconnect‚ĄĘ version is correct for the current Quectel BG95 module firmware version. If it is not correct, the older version is used instead. If no older version is found, AnyNet SMARTconnect‚ĄĘ uses the available version.</p>

<h2 id="UpdatingtheAnyNetSMARTconnectapplicationandQuectelBG95modulefirmware">Updating the AnyNet SMARTconnect‚ĄĘ application and Quectel BG95 module firmware</h2>

<p>For information about the AT commands used in this procedure, see <a href="/software-products/anynet-smartconnect-at-commands/management-at-commands">Management AT commands</a>.</p>

<Accordion title="To update AnyNet SMARTconnect‚ĄĘ application and Quectel BG95 module firmware at the same time:">
 <ol> <li> Set the URL from which to download AnyNet SMARTconnect‚ĄĘ application. Send: <br />AT+ETMCFG="application","updateurl","&lt;<var>http(s)://ETMappURL</var>&gt;" <br />where <span>&lt;<var>http(s)://ETMappURL</var>&gt;</span> is the absolute URL for downloading <span>etm&#95;application.ota</span>. <br />OK </li> <li> Set the URL from which to download the Quectel BG95 module update package. Send: <br />AT+ETMCFG="application","fotaurl","&lt;<var>QuectelDFOTAfirmwareURL</var>&gt;" <br />where <span>&lt;<var>QuectelDFOTAfirmwareURL</var>&gt;</span> is the absolute URL for downloading <span>&lt;<var>QuectelDFOTAfirmware</var>&gt;.bin</span>. <br />OK </li> <li> Save the configuration changes. Send: <br />AT+ETMCFG="save" <br />OK </li> <li> Trigger AnyNet SMARTconnect‚ĄĘ application and Quectel BG95 module firmware download and update. Send: <br />AT+ETMFWCHECK <br />AnyNet SMARTconnect‚ĄĘ downloads the new application, then performs a CRC32 check to ensure the file is valid and the new AnyNet SMARTconnect‚ĄĘ version is different from the existing AnyNet SMARTconnect‚ĄĘ version. <span><strong>Note: </strong>If the file is not valid, or AnyNet SMARTconnect‚ĄĘ version is the same as a previously stored version, no AnyNet SMARTconnect‚ĄĘ update will occur.</span> <br />If the download completes successfully and the versions are different, then AnyNet SMARTconnect‚ĄĘ updates the <span>datatx/oem&#95;app&#95;path.ini</span> file to apply the new application. <br />The Quectel BG95 module firmware then downloads. <br />The following response occurs: <br />+ETMFWCHECK: complete <br />After the download completes, the Quectel BG95 module firmware is updated. <br />If the update is successful, the Quectel BG95 module will automatically reboot. <br />If only AnyNet SMARTconnect‚ĄĘ is successfully updated, the modem will reboot according to the configuration file <span>[operation] update&#95;autoreboot</span> parameter setting: <ul> <li><span>0</span> ‚Äď The system requires a manual reboot</li> <li><span>1</span> ‚Äď The system reboots automatically</li> </ul> </li> </ol> <p>For information about how AnyNet SMARTconnect‚ĄĘ application is stored and used in the datatx folder, see <a href="/software-products/anynet-smartconnect-intro#About">About AnyNet SMARTconnect‚ĄĘ</a>.</p> 
</Accordion>

<h2 id="UpdatingtheAnyNetSMARTconnectapplicationonly">Updating the AnyNet SMARTconnect‚ĄĘ application only</h2>

<p>For information about the AT commands used in this procedure, see <a href="/software-products/anynet-smartconnect-at-commands/management-at-commands">Management AT commands</a>.</p>

<Accordion title="To update AnyNet SMARTconnect‚ĄĘ application:">
 <ol> <li> Clear the current URL configuration for the Quectel BG95 module firmware update. Send: <br />AT+ETMCFG="remove","application","fotaurl" <br />OK </li> <li> Set the URL from which to download AnyNet SMARTconnect‚ĄĘ application. Send: <br />AT+ETMCFG="application","updateurl","&lt;<var>http(s)://ETMappURL</var>&gt;" <br />where <span>&lt;<var>http(s)://ETMappURL</var>&gt;</span> is the absolute URL for downloading <span>etm&#95;application.ota</span>. <br />OK </li> <li> Save the configuration changes. Send: <br />AT+ETMCFG="save" <br />OK </li> <li> Trigger AnyNet SMARTconnect‚ĄĘ application to download and update AnyNet SMARTconnect‚ĄĘ application. Send: <br />AT+ETMFWCHECK <br />AnyNet SMARTconnect‚ĄĘ downloads the new application, then performs a CRC32 check to ensure the file is valid and the new AnyNet SMARTconnect‚ĄĘ version is different from the existing AnyNet SMARTconnect‚ĄĘ version. <span><strong>Note: </strong>If the file is not valid, or AnyNet SMARTconnect‚ĄĘ version is the same as a previously stored version, no AnyNet SMARTconnect‚ĄĘ update will occur.</span> <br />If the download completes successfully and the versions are different, then AnyNet SMARTconnect‚ĄĘ updates the <span>datatx/oem&#95;app&#95;path.ini</span> file to apply the new application. <br />The following response occurs: <br />+ETMFWCHECK: complete <br />If the configuration file <span>[operation] update&#95;autoreboot</span> parameter is set to <span>1</span>, then AnyNet SMARTconnect‚ĄĘ sends the <span>+ETM: REBOOTING</span> URC and will reboot automatically, otherwise AnyNet SMARTconnect‚ĄĘ sends the <span>+ETM: REBOOT REQUIRED</span> URC. </li> <li> If required, reboot the system. Send: <br />AT+ETMRESET </li> </ol> 
</Accordion>

<p>Downloading the device host firmware</p>

<p>For information about the AT commands used in this procedure, see <a href="/software-products/anynet-smartconnect-at-commands/management-at-commands">Management AT commands</a>.</p>

<Accordion title="To download device host firmware:">
 <ol> <li> Set the URL from which to download the device host firmware. Send:  <br />AT+ETMCFG="host","updateurl","&lt;<var>http(s)://HostFirmwareURL</var>&gt;" <br />where <span>&lt;<var>http(s)://HostFirmwareURL</var>&gt;</span> is the absolute URL for downloading the host firmware. <br />OK </li> <li> Save the configuration changes. Send: <br />AT+ETMCFG="save" <br />OK </li> <li> Trigger the host firmware download. Send: <br />AT+ETMHFWGET <br />The response is either: <ul> <li><span>+ETMHFWGET: available</span> ‚Äď AnyNet SMARTconnect‚ĄĘ successfully downloaded the file.</li> <li> <span>+ETMHFWGET: none</span> ‚Äď the file download was not successful. Check the URL is correct and accessible, then try <span>AT+ETMHFWGET</span> again. </li> </ul> </li> </ol> <p>To read the file, use the <span>AT+ETMHFWREAD</span> command. For more information, see <a href="/software-products/anynet-smartconnect-at-commands/etmhfwread">ETMHFWREAD ‚Äď reads a section of the new host firmware image</a>.</p> <p>To delete the downloaded file from datatx folder, use the <span>AT+ETMHFWCONF</span> command. For more information, see <a href="/software-products/anynet-smartconnect-at-commands/etmhfwconf">ETMHFWCONF ‚Äď confirms the new host firmware is applied</a>.</p> 
</Accordion>
