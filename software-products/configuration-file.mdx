<p>Eseye-enabled module operating parameters are contained in a module-specific configuration file. </p>

<p>The Eseye-enabled Quectel module operating parameters are contained in the configuration file, which you can find here: <span>/datatx/eseyeconfig.ini</span>. </p>

<p>You can use the configuration file to set the following parameters:</p>

<ul> <li><a href="#Operatio">Operational</a> </li> <li><a href="#Network" target="_blank">Network</a> </li> <li><a href="#Applicat">Application</a> </li> <li><a href="#Host">Host</a> </li> <li><a href="#[config]">Config</a> </li> <li><a href="#MQTT">MQTT</a> </li> <li><a href="#UDP">UDP</a> </li> <li><a href="#SMS">SMS</a> </li> <li><a href="#Location">Location</a> </li> </ul>

<Note>If this file does not exist, a backup configuration file is used instead: <span>/datatx/eseyeconfig.bak</span>. If neither file exists,AnyNet SMARTconnect™ uses internal default configuration parameters.</Note>

<p>To display the contents of the current configuration file:</p>

<ul> <li>Send: <span>AT+ETMCFG="show"</span></li> </ul>

<p>To save the current configuration: </p>

<ul> <li>Send: <span>AT+ETMCFG="save"</span></li> </ul>

<p>For more information, see <a href="/software-products/anynet-smartconnect-at-commands/etmcfg">ETMCFG – read and write configuration file values</a>.</p>

<Note>To update the configuration file over the air, see <a href="/quectel/awsio-t-jobs">Updating the modem firmware using AWS IoT jobs – Quectel module</a>.</Note>

<p>You can deploy the configuration file using any of the following network protocols: <span>http</span>, <span>https</span>, or <span>tftp</span>.</p>

<p>You can trigger an update using either SMS, MQTT, or AT command.</p>

<p>The.ini file is divided into sections, denoted by square brackets. Each section has parameters listed underneath it. </p>

<Tip>To view an example configuration file, see <a href="#Example">Example content – AnyNet SMARTconnect™ configuration file</a>.</Tip>

<h2 id="ExamplecontentAnyNetSMARTconnectconfigurationfile">Example content – AnyNet SMARTconnect™ configuration file</h2>

<p>The following configuration example is for a battery operated device that connects to the network at random times, when in use.</p>

<ul> <li>The microcontroller (MCU) has very limited programming capacity.</li> <li>It uses elliptic-curve cryptography (ECC).</li> <li>The device data is sent to AWS.</li> <li> Device updates and so on are fetched from AWS. In line with best practice, the device initiates all communication with AWS. <br />For more information, see <a href="/connectivity/device-configuration">Device configuration best practices</a>. </li> </ul>

<Note>The comments within the example do not exist in the configuration file on the device.</Note>

<p> </p>

<p>[operation]</p>

<p>mode=mqtt</p>

<p>; Operating in MQTT mode by default</p>

<p>irqmode=none</p>

<p>irqpin=26</p>

<p>apptrace=1</p>

<p>nvqueuetrace=1</p>

<p>teletrace=1</p>

<p>attrace=1</p>

<p>nvqueuemaxsize=2048</p>

<p>nvqueue&#95;enable=1</p>

<p>update&#95;autoreboot=0</p>

<p>; Disables autoreboot. The device is not designed to handle the </p>

<p>; modem disconnecting and connecting in an uncontrolled way.</p>

<p>; By disabling autoreboot, then using AT+ETMSTATE="powerdown" and</p>

<p>; waiting for a POWERED DOWN URC, the device can process an OTA</p>

<p>; without disrupting the normal flow of operation.</p>

<p>; To set this up, use the AT+ETMSTATE=1 command to enable state change URC.</p>

<p>time&#95;format=%F,%T%z</p>

<p>time&#95;urc=none</p>

<p>enable&#95;time&#95;zone=0</p>

<p>time&#95;zone=0</p>

<p>; Sets the time and timezone to match where the device physically exists.</p>

<p>; The timezone value is passed to a server-side platform for further processing.</p>

<p>ntp&#95;urls=pool.ntp.org,time.nist.gov,time.google.com</p>

<p>[host]</p>

<p>singlepubtopic=dev/a01/j261a/pub/2/&#36;t</p>

<p>; Sets singletopic publish and subscribe. Limited device memory restricts the</p>

<p>; capacity to make multiple subscriptions and check that they have connected</p>

<p>; properly and so on. This makes device coding easier.</p>

<p>singlesubtopicqos=0</p>

<p>singlepubtopicqos=1</p>

<p>; Sets the publish Quality of Service to ensure the messages from device</p>

<p>mqttsettingstopic=settings</p>

<p>; information such as RSSI, MNC, MCC, IMSI, and so on.</p>

<p>; Required for ECC to work with AWS IoT Core.</p>
