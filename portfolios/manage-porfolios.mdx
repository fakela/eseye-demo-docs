<p>You can use <span>Integra</span> to view and manage parent and child (sub-)portfolios. Sub-portfolios allows for separate billing and management of different customer entities, such as different regions or departments.</p>

<p>To view all portfolios:</p>

<h3>Filtering the displayed portfolios</h3>

<p>You can filter the displayed portfolios by <span>Portfolio</span> or <span>Billing</span> information. This helps users manage and locate specific portfolios efficiently.</p>

<h4>Filtering by Portfolio information</h4>

<p>To filter the Portfolios list by Portfolio information:</p>

<ol> <li>On the <span>Portfolios</span> page, select the <span>Portfolio</span> tab.</li> <li> Use any of the following filters to constrain the available list and find a specific portfolio: <ul> <li><span>Portfolio Name</span> – type the full portfolio name.</li> <li><span>Portfolio ID</span> – type the full portfolio ID.</li> <li><span>Parent Portfolio ID</span> – type the full portfolio ID for the parent portfolio.</li> <li><span>Portfolio Status</span> – select the required status from the drop-down list.</li> <li><span>Portal</span> – select the required portal from the down-down list.</li> </ul> </li> <li>Select <span>Search</span> to filter the displayed portfolios.</li> <li>Alternatively, select <span>Reset</span> to clear the filter constraints.</li> </ol>

<h4>Filtering by Billing information</h4>

<p>To filter the Portfolios list by Billing information:</p>

<ol> <li>On the <span>Portfolios</span> page, select the <span>Billing</span> tab.</li> <li> Use any of the following filters to constrain the available list and find a specific portfolio: <ul> <li><span>Portfolio Currency</span> – type or select the portfolio currency.</li> <li><span>Invoicing Entity</span> – select the invoicing entity from the drop-down list.</li> </ul> </li> <li>Select <span>Search</span> to filter the displayed portfolios.</li> <li>Alternatively, select <span>Reset</span> to clear the filter constraints.</li> </ol>

<h2 id="Creatingaportfolio">Creating a portfolio</h2>

<p>Create a portfolio for a new customer.</p>

<ul> <li>A new portfolio allows diversification of assets, products, or services, reducing risk by spreading offerings across different areas or customer segments. </li> <li>Parent and sub-portfolios help manage financial operations efficiently across departments, projects, or subsidiaries. </li> <li>You can arrange portfolios in a hierachy. A parent portfolio consolidates charges from sub-portfolios into a single invoice. Sub-portfolios can have different billing cycles, payment methods, or pricing structures, such as subscriptions or single purchases. </li> </ul>

<p>To create a new portfolio:</p>

<Tip>Depending on your permissions, you can view, create, update and delete portfolios using Integra. For more information, see<a href="/permissions/about-user-roles" target="_blank">About user roles</a>.</Tip>

<ol> <li> On the <span>Portfolios</span> page, select <span>Actions</span> &gt; <span>New Portfolio</span> to display the <span>New portfolio</span> dialog. <br /><img src="/images/resources/images/integra/newportfoliodialog.png" alt="Newportfoliodialog" />  </li> <li> Provide the following information: <table> <col /> <col /> <thead> <tr> <th>Field</th> <th>Description</th> </tr> </thead> <tbody> <tr> <td><span>Name</span> </td> <td> A friendly name for identifying the portfolio, usually the company name followed by a geographic region or project. </td> </tr> <tr> <td><span>Title</span> </td> <td> A short unique label (no spaces) for identifying the portfolio. <span><strong>Tip: </strong>The title can have a maximum of 32 characters without any spaces.</span> </td> </tr> <tr> <td><span>Business Code</span> </td> <td> The customer's geographic location in relation to the legal billing entity (where the company is billed from), either: <ul> <li><span>DOMESTIC</span> – Customer is local to the legal billing entity. E.g. if the legal billing entity is in the UK, the customer is also in the UK.</li> <li><span>EU</span> – Customer is based in the European Union.</li> <li><span>EXPORT</span> – Customer is based in the rest of the world. E.g. if the legal billing entity is in the UK, the customer is non-UK and non-EU.</li> </ul> <span><strong>Note: </strong>You can create new business codes for the same customer to cover different regions.</span> </td> </tr> <tr> <td><span>Tax Code</span> </td> <td> The tax code for the company assigned to the portfolio, which informs the VAT rates that are applied to the invoice. Current supplied tax codes are: <ul> <li><span>STANDARD</span> – The company is local to the legal billing entity (where the company is billed from), standard tax rates apply.</li> <li><span>ESZ</span> – The company is based within the European Union, EU tax rates apply.</li> <li><span>ZERO</span> – The company is neither local, nor in the European Union, and is therefore tax exempt.</li> </ul> </td> </tr> <tr> <td><span>Invoicing Entity</span> </td> <td> The unique invoicing entity identifier for the company that generates invoices and collects payment settlements.  </td> </tr> <tr> <td><span>Currency</span> </td> <td> The unique currency identifier. The currency includes the ISO 4217 alphabetic code and formatting for the selected currency, such as digit and decimal separators and decimal places. </td> </tr> <tr> <td><span>Payment Method</span> </td> <td>A friendly name for the payment method. If the portfolio is active, certain attributes can only be edited by users with the necessary permissions.</span> <ul> <li><a href="#Settings">Settings tab</a> </li> <li><a href="#Portfoli">Portfolio attributes</a> </li> <li><a href="#Addresse">Addresses tab</a> </li> <li><a href="#Users">Users tab</a> </li> <li><a href="#Packages">Packages tab</a> </li> </ul> </li> <li> Provide values or choose from the respective drop-down list for the <a href="#About2">attributes</a> in the <span>Edit Settings page</span>.  <span><strong>Tip: </strong>An attribute will not be displayed on the <a href="#Attribut">Attributes page</a> if its value is not set. If the portfolio is active, certain attributes can only be edited by users with the necessary permissions.</span> </li> <li> Select <span>Save</span> for the changed attribute values to reflect or select <span>View</span> to return to the <span>Main page</span>. </li> </ol>
