<h3>User actions and corresponding subscription states</h3>

<p>A user performs the following actions for the system to dictate the transaction state.</p>

<h6>System automatically marks the transaction as:</h6>

<ol> <li><span>Setup</span> – When the user creates a new subscription to store the information. </li> <li><span>Requested</span> – When information is added to the new subscription and the user requests approval for the newly added subscription.</li> <li> <span>Active</span> – When the user requests <span>Approve</span> for the <span>Requested</span> subscription to appear on the next invoice.  <span><strong>Note: </strong>Note that for <span>Active</span> subscriptions, a batch task transfers the appropriate monthly changes to an invoice.</span> </li> <li>[CDATA[ ]]<span>Suspended</span> – When the <span>Approved</span> subscription is no longer charged.</li> <li><span>Deleted</span> – When the subscription has appeared on the invoice.</li> </ol>

<h3>Reason for changing subscription states</h3>

<table> <col /> <col /> <thead> <tr> <th> State Change</th> <th>Reason</th> </tr> </thead> <tbody> <tr> <td> Single subscription: <span>Setup</span> &gt; <span>Request for Approval</span> <br />Multiple subscriptions: <span>Setup</span> &gt; <span>Request Approval for Selected</span> </td> <td> <ul> <li>Evaluate the credibility and reliability of the subscription vendor.</li> <li>Review and understand the terms and conditions, including cancellation policies and renewal terms.</li> </ul> </td> </tr> <tr> <td>Single subscription: <span>Request for Approval</span> &gt; <span>Approve</span><br />Multiple subscriptions: <span>Request Approval for Selected</span> &gt; <span>Approve</span></td> <td> <ul> <li>Combine subscriptions into a single transaction to streamline financial processes and reduce administrative overhead.</li> <li>Provide an opportunity to negotiate better terms or discounts for bundled services.</li> </ul> </td> </tr> <tr> <td>Single subscription: <span>Approve</span> &gt; <span>Delete</span></td> <td> <ul> <li>Prevent the billing system from becoming cluttered.</li> </ul> </td> </tr> </tbody> </table>

<h2 id="Deletingasubscription">Deleting a subscription</h2>

<p>A subscription is deleted to prevent the billing system from becoming cluttered.</p>

<p>To delete a subscription:</p>

<Note>Only the <span>Billing Senior</span> Infinity role can delete <span>Requested</span> or <span>Setup</span> subscriptions. For more information, see Infinity roles. <span>Active</span> subscriptions cannot be deleted by the <span>Billing Senior</span>; however, the system can mark these as <span>Deleted</span>.</Note>

<ol> <li> Using customised filters, <a href="#To">find the specific subscription</a>. </li> <li>In the <span>Search Results</span> table, select <img src="/images/resources/images/integra/expand-icon.png" alt="Expand Icon" />alongside the relevant approved subscription. </li> <li> In the <span>View Transaction details</span> window of the selected subscription, select <span>Delete</span>. <br /><img src="/images/resources/images/integra/deletesubscription.png" alt="Deletesubscription" />  </li> <li> In the <span>Delete Transaction</span> dialog, select <span>Confirm</span> to delete the selected approved subscription. <span><strong>Tip: </strong>Wait until you receive a green notification that <span>The subscription has been deleted</span>.</span> </li> </ol>
