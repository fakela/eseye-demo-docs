<h3>User actions and corresponding single purchase states</h3>

<p>A user performs the following actions for the billing system to dictate the transaction state.</p>

<h6>System automatically marks the single purchase as:</h6>

<ol> <li><span>Setup</span> – When the user creates a new transaction to store the information. </li> <li><span>Requested</span> – When information is added to the <span>Setup</span> transaction and the user requests approval for the newly added transaction.</li> <li><span>Active</span> – When the user requests <span>Approve</span> for the <span>Requested</span> transaction to appear on the next invoice. </li> <li>[CDATA[ ]]<span>Exported</span> – When the <span>Approved</span> transaction appears on an invoice.</li> <li><span>Deleted</span> – When the transaction is identified as incorrect.</li> </ol>

<h3>Reason for changing purchase states</h3>

<table> <col /> <col /> <thead> <tr> <th> State Change</th> <th>Reason</th> </tr> </thead> <tbody> <tr> <td>Single transaction: <span>Setup</span> &gt; <span>Request for Approval</span><br />Multiple transactions: <span>Setup</span> &gt; <span>Request Approval for Selected</span></td> <td> <ul> <li>Ensure budgetary control by verifying compliance with contract limits.</li> <li>Ensure adherence to internal policies and external regulations to prevent legal and financial penalties.</li> <li>Review transaction details to verify accuracy and completeness.</li> </ul> </td> </tr> <tr> <td>Single transaction: <span>Request for Approval</span> &gt; <span>Approve</span><br />Multiple transactions: <span>Request Approval for Selected</span> &gt; <span>Approve</span></td> <td> <ul> <li>Reduce the risk of errors, fraud, and other financial irregularities.</li> <li>Establish an approval process that creates a clear chain of responsibility, making it easier to track decisions and hold individuals accountable.</li> <li>Allow these transactions to appear on the customer's invoice.</li> </ul> </td> </tr> <tr> <td>Single transaction: <span>Approve</span> &gt; <span>Delete</span></td> <td>  Prevent faulty processing of incorrect transactions. </td> </tr> </tbody> </table>

<h2 id="Deletingasinglepurchase">Deleting a single purchase</h2>

<Note>Only the <span>Billing Senior</span> Infinity role can delete <span>Requested</span> or <span>Setup</span> transactions. For more information, see Integra Roles. <span>Active</span> transactions cannot be deleted by the <span>Billing Senior</span>; however, the billing system can mark these as <span>Deleted</span>.</Note>

<p>An incorrect transaction is deleted to prevent faulty processing.</p>

<ol> <li> Using customised filters, <a href="#To">find the specific transaction</a>. </li> <li>In the <span>Search Results</span> table, select <img src="/images/resources/images/integra/expand-icon.png" alt="Expand Icon" />alongside the relevant approved transaction. </li> <li> In the <span>View Transaction details</span> window of the selected transaction, select <span>Delete</span>. <br /><img src="/images/resources/images/integra/deletetransaction.png" alt="Deletetransaction" />  </li> <li> In the <span>Delete Transaction</span> dialog, select <span>Confirm</span> to delete the selected approved transaction. <span><strong>Tip: </strong>Wait until you receive a green notification that <span>The transaction has been deleted</span>.</span> </li> </ol>
