<h2 id="CreatingtheessentialAnyNetfiles">Creating the essential AnyNet files</h2>

<p>Each device must contain the following five files to connect with cloud services:</p>

<ul> <li><span>anynet&#95;rootca&#95;store</span> â€“ the root CA certificate</li> <li><span>anynet&#95;pubcert&#95;store</span> â€“ the public certificate of the device</li> <li><span>anynet&#95;privkey&#95;store</span> â€“ the public private key of the device</li> <li><span>anynet&#95;thingname&#95;store</span> â€“ the Device ID, defined by the user during the device creation phase in the IoT Hub or AWS portal.</li> <li><span>anynet&#95;url&#95;store</span> â€“ the URL (hostname/endpoint) that is found on the IoT Hub and AWS portal.</li> </ul>

<p>To generate the essential AnyNet files</p>

<ol> <li> Use instructions from your cloud services provider to generate a root ca, public certificate and private key for each device.  <br />AnyNet SMARTconnectâ„˘ supports both PEM and DER formats. </li> <li> Rename these files as follows, with no extension: <ul> <li><span>anynet&#95;rootca&#95;store</span> </li><li><span>anynet&#95;pubcert&#95;store</span> </li> <li><span>anynet&#95;privkey&#95;store</span> </li> </ul></li> <li> Use a text editor to create the following: <ul> <li>  <span>anynet&#95;thingname&#95;store</span> file â€“ must only contain the Device ID. <br />For example: MyDevice </li> <li> <span>anynet&#95;url&#95;store</span> file â€“ must only contain the host endpoint. <br />For example: <span>my-hub.azure-devices.net</span> or <span>a2fguj321joea3-ats.iot.eu-west-1.amazonaws.com</span>.  </li> </ul> </li> <li>Next, you must add a header to each file.</li> </ol>

<h3>Adding a header to AnyNet files</h3>

<p>To complete the file generation process, you must add a 6-byte binary header to the beginning of each file. The header consists of the length (2 bytes) followed by the CRC-32 (4 bytes).</p>

<p>For example, for <span>anynet&#95;thingname&#95;store</span>, the file appears as follows when opened in a text editor: </p>

<p><img src="/images/resources/images/icons/header.png" alt="Header" /> </p>

<p>The same file appears as follows in a binary editor:</p>

<p><img src="/images/resources/images/icons/headerbinaryfile.png" alt="Headerbinaryfile" /> </p>

<p>For information about calculating the checksum, see <a href="/sim-only/crc32code-example" target="_blank">CRC-32 code example</a>.</p>

<p>To add the header to an AnyNet file using the supplied Windows (PowerShell) script:</p>

<ol> <li> If you are using Windows, select <a href="https://www.dropbox.com/s/4bnivp867ibildw/windows_header_gen.ps1?dl=0">here</a> to download the <span>windows&#95;header&#95;gen.ps1</span> script. </li> <li> Using Windows Powershell, run the script: <br /><span>.&#92;windows&#95;header&#95;gen.ps1 &lt;<var>anynetfile&#95;path</var>&gt;</span>  <br />where <var>anynetfile&#95;path</var> is the AnyNet file path and filename. <br />For example: <br />.&#92;windows&#95;header&#95;gen.sh.&#92;anynetfiles&#92;anynet&#95;rootca&#95;store </li> </ol>

<p>To add the header using the supplied Linux script:</p>

<ol> <li> If you are using Linux, select <a href="https://www.dropbox.com/s/g4ar86q9rorc4iu/linux_header_gen.sh?dl=0">here</a> to download the <span>linux&#95;header&#95;gen.sh</span> script. </li> <li> Using a Linux terminal, run the script: <br /><span>./linux&#95;header&#95;gen.sh &lt;<var>anynetfile&#95;path</var>&gt;</span>  <br />where <var>anynetfile&#95;path</var> is the AnyNet file path and filename. <br />For example:  <br />./linux&#95;header&#95;gen.sh anynetfiles/anynet&#95;rootca&#95;store </li> </ol>
