<p>You can connect to a cellular network using a Quectel BG95 module and Eseye eUICC or AnyNet SIM. The following steps will help you through three stages of the network connection process:</p>

<ol> <li>Configuring automatic network operator selection.</li> <li>Checking the network registration status.</li> <li>Ensuring you can send and receive data.</li> </ol>

<Note>Eseye tested the following steps on the <span>BG95-M3</span>. Steps may vary on different BG95 module variants.</Note>

<div> <h3>Before you begin</h3> <ul> <li>Configure the device and Quectel BG95 module according to the manufacturer instructions. For example, ensure you have installed the correct drivers.</li> <li>Fully insert the Eseye SIM into your device.</li> <li>Ensure the Eseye SIM is activated on the Infinity Classic portal.</li> <li>Ensure the device is on and the Quectel BG95 module is powered up.</li> <li>Configure a terminal emulator to send commands to the Quectel BG95 module.</li> </ul> </div>

<p>To connect a Quectel BG95 module to a cellular network:</p>

<ol> <li> Using the terminal emulator, verify the device is reading the SIM correctly by requesting the ICCID and IMSI. Type: <br />AT+CCID&lt;CR&gt; <br />AT+CIMI&lt;CR&gt; <span><strong>Note: </strong><span>&lt;CR&gt;</span> is the end-of-line character marking the end of a command line (alias &#92;r <span>– carriage return)</span>.</span> <span><strong>Tip: </strong>Make a note of the returned numbers in the event you need to ask for assistance later.</span> </li> <li> Using the terminal emulator, configure the Quectel BG95 module for minimum functionality. Type: <br />AT+CFUN=0&lt;CR&gt; </li> <li> Configure automatic searching of all Radio Access Technology (RAT) types, to include GSM (2G) and LTE (Cat M1). Type: <br />AT+QCFG="nwscanmode",0,1&lt;CR&gt; </li> <li> Configure the RAT search sequence in the following order: LTE Cat M1 -&gt; GSM -&gt; LTE Cat NB1. Type: <br />AT+QCFG="nwscanseq",020103,1&lt;CR&gt; </li> <li> Configure the device to search all available frequency bands. Type: <br />AT+QCFG="band",0,100002000000000F0E189F,10004200000000090E189F&lt;CR&gt; <span><strong>Tip: </strong>The advanced user may want to limit the number of bands searched, which is beyond the scope of this topic.</span> </li> <li> Set the mobile network operator selection to <span>automatic</span>. Type: <br />AT+COPS=0 <br />The response will include the current network operator and access technology.  </li> <li> Define the PDP context to route data through Eseye's APN. Type: <br />AT+CGDCONT=&lt;cid&gt;,"&lt;PDPtype&gt;","&lt;APN&gt;"&lt;CR&gt; <br />where <span>&lt;cid&gt;</span> is the context ID, <span>&lt;PDPtype&gt;</span> is the IP connection type, and <span>&lt;APN&gt;</span> is the relevant Eseye Access Point Name.  <br />For example: <span>AT+CGDCONT=1,"IP","eseye1"</span> <span><strong>Note: </strong>For information about which Eseye APN to use, speak to your Account Manager. Also see <a href="/connectivity/anynet-apn-list">Current AnyNet APN list</a>. </span> </li> <li> Configure the module for full functionality. Type: <br />AT+CFUN=1 </li> <li> Set the mode to enable <span>+CREG</span>, <span>+CGREG</span>, and <span>+CEREG</span> unsolicited result codes. Type: <br />AT+CREG=1;+CGREG=1;+CEREG=1&lt;CR&gt; <span><strong>Note: </strong>For detailed information about <span>AT+CREG</span>, see <a href="/software-products/general-at-commands/creg" target="_blank">CREG – request network registration status</a>. For details about AT+CGREG and AT+CEREG, refer to the Quectel AT Commands manual for your module.</span> </li> <li> Check the network operator selection. Type:  <br />AT+COPS?&lt;CR&gt; <br />The response includes the operator and the access technology selected. The following example response is for Verizon in LTE Cat M1 mode: <br />+cops: 0,0,"Verizon ",8 <br />The following example response is for T-Mobile in GSM mode: <br />+cops: 0,0,"T-Mobile USA",0 </li> <li> Check the active RAT and determine the signal strength. Type: <br />AT+QCSQ&lt;CR&gt; </li> <li> Check the network registration state per network type. Type: <br />AT+CREG?;+CEREG?;+CGREG?&lt;CR&gt; <br />The response includes the mode you set earlier and the registration status, which is either: <ul> <li><span>0</span> – Not registered, the device is currently not searching for new operator.</li> <li><span>1</span> – Registered to home network.</li> <li><span>2</span> – Not registered, but the device is currently searching for a new operator.</li> <li><span>3</span> – Registration denied.</li> <li><span>4</span> – Unknown. For example, out of range.</li> <li><span>5</span> – Registered, roaming. The device is registered on a foreign (national or international) network.</li> </ul> <table> <col /> <col /> <thead> <tr> <th>Example response</th> <th>Network type</th> </tr> </thead> <tbody> <tr> <td> +CREG: 1,1 </td> <td>2G – GSM (circuit switched) </td> </tr> <tr> <td> +CGREG: 1,1 </td> <td> 2G data – GPRS (packet switched)</td> </tr> <tr> <td> +CEREG: 1,5 </td> <td>Cat M1 or NB-IoT</td> </tr> </tbody> </table> <div> Factors contributing to SIM failure to register on the network may include:  <ul> <li>Missing network coverage</li> <li>Denied network access</li> <li>No valid roaming agreement between the home network and currently available operators</li> </ul> </div> </li> <li> Open a data context. Type: <br />AT+CGATT=1&lt;CR&gt; <br />If the response is <span>OK</span>, the device can send and receive data. If <span>ERROR</span> or <span>+CME ERROR</span> is returned, the attempt to open a data context has failed. Refer to the Quectel AT Commands manual for more information. </li> <li> Check the allocated IP address. If you registered to the correct APN, this matches the IP address on the RADIUS log in Infinity Classic. Type: <br />AT+CGPADDR&lt;CR&gt; </li> <li> Use the AnyNet Ping service to ensure you can send and receive data packets. Type: <br />AT+QPING=1,"192.168.109.2"&lt;CR&gt; <br />A successful response is similar to the following: <br />OK <br />+QPING: 0,"192.168.109.2",32,744,255 <br />+QPING: 0,"192.168.109.2",32,257,255 <br />+QPING: 0,"192.168.109.2",32,234,255 <br />+QPING: 0,"192.168.109.2",32,238,255 <br />+QPING: 0,4,4,0,234,744,368 </li> </ol>
