<p>Multi-network SIM profiles need time to connect to the best local available network for a device. While single MNO SIM profiles will take less time to connect to a network, this is only helpful if the provider is constantly available in the locale.</p>

<p>For Quectel modems, you can optimise the time an AnyNet SIM takes to register on a network by using the following coding best practices:</p>

<ol> <li> Enable all URCs for <span>AT+CREG</span>, <span>AT+CGREG</span>, and <span>AT+CEREG</span>, unless you prefer to poll for this information. <span><strong>Note: </strong>These URCs will enable you to see when the modem registers and deregisters in real time.</span> <br />If you prefer to poll for <span>AT+CREG</span>, <span>AT+CGREG</span>, and <span>AT+CEREG</span>, then the following responses are for registered modems: <ul> <li><span><i>n</i>,1</span>, where <span>1</span> is registered, home network</li> <li> <span><i>n</i>,5</span>, where <span>5</span> is registered, roaming </li> </ul> <span><strong>Tip: </strong>Refer to the Quectel AT command documentation for more information.</span> </li> <li> Using the <span>AT+QCFG</span> command with radio enabled will trigger a new radio frequency (RF) scan for different available frequencies, and will disregard if the modem is already camped on a cell, waiting for registration.  <br /> As a result: <ol> <li>Registration time, and therefore power consumption, will increase because of the rescan.</li> <li>In accordance with 3GPP, the modem will look for a different available service provider and ignore the <span>LOCI</span> and <span>OPLMNwACT</span> values stored in the SIM. This may cause an unwanted switch to a different service provider.</li> </ol> <br />The rescan triggers for the following commands: <ul> <li><span>band</span> </li> <li><span>iotopmode</span> </li> <li><span>nwscan</span> </li> <li><span>nwscanmode</span> </li> <li><span>nwscanseq</span> </li> <li><span>roamservice</span> </li> <li><span>servicedomain</span> </li> </ul> <span><strong>Tip: </strong>We recommend you read the existing values to ensure they are correct for your system. </span> <span><strong>Tip: </strong>We recommend you disable NB-IoT if you're not using the technology, as scanning for NB-IoT bands can take a long time.</span> <br />If you must change any of these values: <ol> <li>Use <span>AT+CFUN=0</span> to disable radio.</li> <li> Use <span>AT+QCFG</span> commands to make your changes, for example <span>AT+QCFG=“iotopmode”,0,1</span> to disable NB-IoT. </li> <li> Use <span>AT+CFUN=1</span> to enable radio. </li> </ol> </li> <li> If you want to scan for LTE Cat M1 networks before scanning for 2G networks, use any of the following settings: <ul> <li><span>AT+QCFG=”nwscanseq”,02,1</span> </li> <li><span>AT+QCFG=”nwscanseq”,020103,1</span> </li> <li><span>AT+QCFG=”nwscanseq”,0201,1</span> </li> </ul> <span><strong>Note: </strong>Ensure you disable radio when setting the <span>AT+QCFG=”nwscanseq”</span> values (see point 2 above).</span> </li> <li> For 2G and LTE Cat M1 networks, ensure you scan for available networks for 20-30 seconds before causing changes in state, such as restarting the modem.  <span><strong>Note: </strong>The time delay must take into account the potential scan times of the LTE Cat M1 bands and the AnyNet SIM. If the time delay is too short, it is very likely that time out will occur before the device transitions to a registered status. The modem may then take additional time to restart.</span> </li> <li> If you are using <span>AT+QENG</span> for additional information, do not use it to poll for registration status.  <br /><span>AT+QENG</span> is designed to provide information about serving cells, and the information only becomes valid after the <span>AT+COPS?</span> and <span>AT+CREG</span> commands indicate successful registration.  <span><strong>Tip: </strong>If you require <span>QENG="servingcell"</span> information, request it <span>after</span> the registration status has changed.</span> </li> </ol>
